##### <a name="to-restore-files-and-folders-from-a-server-backup"></a>サーバー バックアップからファイルとフォルダーを復元するには  
  
1.  Windows Server Essentials ダッシュ ボードを開き、クリックして、**デバイス**] タブ。  
  
2.  サーバーの名前をクリックし、クリックして**、サーバーのファイルまたはフォルダーの復元**で、**タスク**ウィンドウ。  
  
     復元のファイルとフォルダーのウィザードが開きます。  
  
3.  ファイルまたはフォルダーを復元ウィザードの指示に従います。  
  
> [!WARNING]
>  バックアップとファイルとフォルダーの復元の詳細については、次を参照してください。[管理のバックアップと復元](Manage-Backup-and-Restore-in-Windows-Server-Essentials.md)します。  
  
###  <a name="BKMK_ConfigreSharedFolders"></a>サーバー上の共有フォルダーを復元します。  
 共有フォルダーがまだデータ パーティションには、またはデータ パーティションに復元された場合に、サーバー システム ドライブを復元した後は、サーバー フォルダーを認識するためにもう一度共有フォルダーを構成する必要があります。 次の手順では、前に共有されている共有フォルダーを追加する方法について説明します。  
  
##### <a name="to-add-an-existing-folder-to-the-server-shared-folders"></a>サーバー共有フォルダに、既存のフォルダーを追加するには  
  
1.  ファイル エクスプ ローラーで、ハード ドライブ上の共有フォルダーを探します。  
  
2.  共有フォルダーを右クリックし、をクリックして**プロパティ**、] をクリックして、**共有**タブをクリックし、フォルダーのアクセス許可を書き留めます。  
  
3.  Windows Server Essentials ダッシュ ボードにログオン] をクリックして、**記憶域**タブをクリックし、をクリックして**フォルダーを追加**で、**サーバー フォルダーのタスク**ウィンドウ。  
  
     追加するフォルダーの作成ウィザードが開きます。  
  
4.  共有の名前を入力、**名**ボックス。  
  
5.  をクリックして**参照**に移動*< drive\ > \\ < ServerName\ >*\ServerFolders (たとえば*d:\Contoso\ServerFolders*)、共有、およびをクリックするフォルダーを選択**OK**します。  
  
6.  をクリックして**次**します。  
  
7.  手順 2 で書き留めたアクセス許可を指定し、クリックして**フォルダーの追加**します。  
  
    > [!IMPORTANT]
    >  これらのアクセス許可では、Windows Server Essentials ダッシュ ボードを使用していないフォルダーに追加された既存のアクセス許可を置き換えます。  
  
> [!IMPORTANT]
>  共有フォルダーの一覧にフォルダーを追加が完了したら、サーバーのバックアップにフォルダーが含まれていることを確認します。 サーバーのバックアップにフォルダーを追加する方法の詳細については、次を参照してください。[設定のセットアップまたはサーバー バックアップのカスタマイズ](Set-up-or-customize-server-backup.md)します。
