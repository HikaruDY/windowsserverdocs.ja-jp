##  <a name="BKMK_WhatIs"></a>コンピューターのシステムの完全復元とは何ですか。  
 ハード ディスク ドライブまたはを使用できないか、が起動しない点にコンピューターが故障したを交換することは、コンピューターの以前のバックアップからシステムを復元できます。 システムの完全復元では、バックアップ時の状態にシステムを返します。  
  
> [!IMPORTANT]
>  同様に置き換えられるコンピューターのハードウェアではないシステム ボードなど、コンピューター ハードウェア上でシステムの完全復元を行うことはできません。 インストールされているオペレーティング システムでは、コンピューターの基になるハードウェアに密接に依存します。 ただし、置き換えられるものより大きいか等しいサイズのハード ドライブにシステムの完全復元を実行できます。  
  
 システムの完全復元を実行している場合は、すべてのアプリケーション、構成、および障害、災害、または盗難前にユーザーが使い慣れた設定と、システムを復元する特定のコンピューター バックアップを選択できます。 ボリュームを復元することもできます。

 Windows PE はワイヤレス接続をサポートしていません。 このため、復元するコンピューターを小規模ビジネス ネットワークに物理的に接続する必要があります。

 新しいコンピューター上のハード ディスク ドライブのサイズは同じ、またはよりも大きい元、ハード場合、ディスク ドライブが自動的に再フォーマットし、パーティションに再分割します。 具体的な情報については、次の図を参照してください。

|複数のパーティションを持つ 1 つのディスク|複数のパーティションと追加の領域の 1 つのディスクが割り当てられている最後のパーティション|

 Windows ServerÂ® 2012 Essentials に付属のシステム復元メディアは、コンピューターに Windows プレインストール環境 (Windows PE) をインストールします。 Windows PE は MS-DOS 環境が置き換えられます、Windows のコア プログラム ファイルが含まれています。 Windows Server Essentials では、システムを復元する 2 つのサポートされている方法があります: USB を使用してフラッシュ ドライブまたはネットワークを使用して、メディアに依存しないと、クライアント復元サービスを使用して、します。

 システム復元環境に付属して 32 ビット (x86) と 64 ビット (x64) のプログラム ファイル。 復元メディアをシステムを挿入したら、適切なバージョンのファイルを選択します。 32 ビット (x86)、既定値は、30 秒以内に選択しない場合が自動的に選択します。 完全なシステムへの更新サーバー上のプログラム ファイルを復元する場合は、更新されたファイルは自動的にコンピューターにダウンロードされます。  
  
 プレインストール環境を設定した後、システムの完全復元ウィザードを開始します。 ウィザードを使用すると、以前のバックアップからコンピューターを復元します。 また、同種のハードウェアで新しいコンピューターにバックアップを復元するのにシステム復元環境を使用することができます。  
  
 ほとんどの場合、プログラム ファイルや、システム復元環境に含まれているドライバーはすべて、新しいコンピューターまたは復元されたコンピューターを再起動するために必要です。 新しいコンピューターまたは復元されたコンピューターのハードウェアによって、システム復元環境可能性があります、記憶域のすべてが含まれますされず、ネットワーク アダプター ドライバーが復元されたコンピューターを再起動するときに必要なです。 システムの完全復元ウィザードでは、ドライバーをインストールするために必要な場合です。 ハードウェア ドライバーを見つける方法については、次を参照してください[ハードウェアのドライバーを見つける場所ですか?。](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_FindDrivers). システム復元メディアを使用する方法については、次を参照してください。[システムの完全復元ウィザードを使用して](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_Using)します。

##  <a name="BKMK_Using"></a>システムの完全復元ウィザードを使用してください。  
 正常に復元を使用した後メディア、クライアント リストア サービス、または USB フラッシュ ドライブをコンピューターを起動し、復元された、または新しいクライアント コンピューターで、システムの完全復元ウィザードのドライバーが読み込まれるすべてのハードウェアが表示されていることを確認します。 このウィザードでは、サーバー、コンピューターのバックアップと、コンピューターに復元して、実際の復元処理を実行するソース ボリュームにアクセスすることができます。  
  
> [!NOTE]
>   Windows Server Essentials では、次の復元シナリオはサポートされません。  
>   
>  -   ベースのコンピューターに、マスター ブート レコード (MBR) ディスクに、Unified Extensible Firmware Interface (UEFI) を復元します。  
> -   UEFI/GPT バックアップを BIOS システムに復元しています。  
>   
>  これらのシナリオのいずれかでデータを復元する場合、システムを起動できなきます。 さらに、できないことがありますが 2 テラバイトのサイズよりも大きいハード ドライブを使用します。  
  
 **前提条件:**  
  
-   復元処理を完全なシステムを開始する前に、ネットワーク ケーブル (ワイヤード (有線) 接続) を使用して、コンピューターをサーバーと同じネットワークに接続します。 クライアント コンピューター上のすべてのハード ドライブへのアクセスがあることを確認します。  
  
    > [!WARNING]
    >  ワイヤレス ネットワークに接続を使用しているコンピューターへの完全システム復元を実行しないでください。  
  
-   コンピューターが重要なネットワークまたは記憶装置デバイス ドライバーが見つからないことがわかっている場合を検索し、システムの完全復元処理を開始する前にそれらのドライバーをフラッシュ ドライブにコピーする必要があります。 Windows Server Essentials: CD で提供されているシステムの完全復元メディアを使用している場合、CD 抑える必要があります、ドライブのシステムの完全復元処理の開始中にします。 そのため、2 番目の CD または DVD ドライブがない限り、CD または DVD に不足しているドライバーをコピーできません必要があります。 代わりに、不足しているドライバーを USB フラッシュ ドライブにコピーします。  
  
     コンピューターのドライバーを検索する方法については、次を参照してください[ハードウェアのドライバーを見つける場所ですか?。](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_FindDrivers)

##  <a name="BKMK_FindDrivers"></a>ハードウェアのドライバーはどこで確認できますか。  
 新しいコンピューターまたは復元されたコンピューターのハードウェアによって、復元メディア可能性があります、記憶域のすべてが含まれますされず、ネットワーク アダプター ドライバーが復元されたコンピューターを再起動するときに必要なです。 どのドライバーが表示されない、既存のメディアまたは製造元の Web サイトでは、それらのドライバーを検索、フラッシュ ドライブにコピーしし、フラッシュ ドライブから、新しいにコピーまたはシステムの完全復元ウィザードを実行すると、コンピューターを復元を決定する必要があります。  
  
 コンピューターをバックアップするときは、コンピューターのドライバーがバックアップに保存されます。 リカバリ メディアはすべての必要なドライバーを含めない場合は、そのコンピューターのバックアップを開いてし、ドライバーを USB フラッシュ ドライブにコピーできます。  
  
#### <a name="to-copy-drivers-from-a-backup-to-a-usb-flash-drive"></a>バックアップからドライバーを USB フラッシュ ドライブにコピーするには  
  
1.  別のコンピューターでは、ダッシュ ボードを開きます。  
  
2.  をクリックして**デバイス**、し、[ドライバーが必要なコンピューター] をクリックします。  
  
3.  をクリックして**コンピューターのファイルまたはフォルダーの復元**します。 復元のファイルまたはフォルダーのウィザードが開きます。  
  
4.  最新の正しいバックアップをクリックし、クリックして**次**します。  
  
5.  ボリュームをクリックして開くには、クリックして**次**します。 ファイルとフォルダーのバックアップに一覧表示されたウィンドウが開きます。  
  
6.  コンピューター上の USB コネクタに、USB フラッシュ ドライブを挿入し、フォルダーのシステムの完全復元用のドライバーを USB フラッシュ ドライブにコピーします。  
  
    > [!NOTE]
    >  をクリックする必要があります**1 つのレベルは最大**システム ボリュームのルートに達するまでです。  
  
7.  フラッシュ ドライブを削除し、復元しているコンピューターに挿入します。  
  
 USB フラッシュ ドライブを使用して、復元するときに、コンピューターのドライバーをインストールすることができます。 復元のファイルまたはフォルダーのウィザードは、システムの完全復元ウィザードを使用しているときに、この USB フラッシュ ドライブ上で追加ドライバーを検索します。 必要がある可能性が最も高いドライバーは、ネットワーク アダプター ドライバーと記憶域デバイス ドライバーを示します。
