###  <a name="BKMK_Overview"></a>リモート Web アクセスの概要  
 オフィス以外の場合は、Web ブラウザーを開きますしてインターネットにアクセスできる任意の場所からリモート Web アクセスにアクセスします。 リモート Web アクセスでは、次のことができます。  
  
-   サーバーの共有ファイルとフォルダーにアクセスします。  
  
-   サーバー、ネットワーク上のコンピューターにアクセスします。 これは、オフィスで前に座っている場合と、ネットワーク上のコンピューターのデスクトップにアクセスできることを意味します。  
  
  リモート Web アクセスは既定でオンがありません。 Anywhere Access のウィザードのセットアップを実行すると、ウィザードは、ルーターおよびインターネット接続を設定しようとします。 リモート Web アクセスを有効にしたら、サーバーのドメイン名をセットアップし、リモート Web アクセスをカスタマイズできます。 設定できます、ルーターをもう一度ルーターを変更した場合。  
  
 新しいユーザー アカウントを追加するときに、リモート Web アクセスにアクセスするためのアクセス許可は自動的に付与されません。 ユーザー アカウントを追加する場合は、共有フォルダー、メディア ライブラリ、コンピューター、ホーム ページ リンク、およびサーバー ダッシュ ボードへのアクセスを許可するように選択できます。 リモート Web アクセスを使用するユーザーが許可されませんを指定することができます。  
  
 リモート Web アクセスの設定は、ユーザー アカウントごとに表示される、**ユーザー**、Windows Server Essentials ダッシュ ボードのタブ。 リモート Web アクセスの設定を変更するユーザー アカウントを右クリックし、をクリックして**アカウント プロパティの表示**します。

###  <a name="BKMK_NetworkLocation"></a>定義されているネットワークの場所  
 ネットワークの場所は、Windows がネットワークに接続するときに適用されるネットワーク設定のコレクションです。 設定は、異なるされ、使用するネットワークの種類に基づいてカスタマイズすることができます。 ネットワークの場所の設定では、(ファイルとプリンタの共有、ネットワーク探索、およびパブリック フォルダーの共有) などの特定の機能が有効かどうかまたは無効にするかどうかを決定します。 ネットワークの場所は、異なるネットワークに接続する必要がある場合に役立ちます。  
  
 たとえば、自宅でジョブを使用するラップトップ コンピューターを所有して可能性があります。 オフィスにしたら、オフィス ネットワークに接続します。 ただし、付属ホーム、使用するラップトップ コンピューターにアクセスし、ビデオ、およびホーム サーバーに格納されている音楽を再生します。 新しいネットワークに接続する場所の種類を指定すると、Windows には、その場所の種類に対して事前設定されているネットワーク プロファイルが割り当てられます。 次回からそのネットワークに接続する Windows はネットワークを認識し、正しい設定が自動的に割り当てます。 これにより、追加すると、コンピューター上の情報を保護するためのセキュリティ レイヤーとその場所に必要なネットワーク機能のみオンにします。  
  
 ネットワークの場所の 4 種類あります。  
  
-   **ホーム ネットワーク**このネットワークのホーム ネットワークまたはときに、信頼できる人と、ネットワーク上のデバイスを選択します。 ホーム ネットワーク上のコンピューターは、ホーム グループに属していることができます。 ネットワーク探索でがオンにホーム ネットワークは、コンピューターを表示するには、他のネットワーク ユーザーができ、ネットワーク上の他のコンピューターやデバイスを参照することができます。  
  
-   **社内ネットワーク**にこのネットワークを小規模オフィスまたはその他の職場のネットワークを選択します。 ネットワーク探索は、使用すると、ネットワーク上の他のコンピューターやデバイスを参照してくださいし、コンピューターを表示するには、他のネットワーク ユーザーは、既定でオンですが、作成またはホーム グループに参加することはできません。  
  
-   **パブリック ネットワーク**このネットワーク (コーヒー ショップまたは空港) などの公共の場所を選択します。 この場所は、他のコンピューターに表示されないように、コンピューターを維持し、インターネットからの悪意のあるソフトウェアからコンピューターを保護するために設計されています。 ホーム グループがパブリック ネットワークで利用できないと、ネットワーク探索は無効になります。 ルーターを使用せず、インターネットに直接接続している場合、またはモバイル ブロード バンド接続がある場合も、このオプションを選択する必要があります。  
  
-   **ドメイン**このネットワークなど職場でドメインを選択します。 この種類のネットワークの場所は、ネットワーク管理者によって制御と選択または変更することはできません。

###  <a name="BKMK_DNOverview"></a>ドメイン名の概要  
 ドメイン名は、インターネット上のサーバーを一意に識別します。 ドメイン名は、少なくとも 2 つの部分で構成されています。トップ レベルのドメイン名 (TLD) と第 2 レベル ドメイン名。 たとえば、contoso.com、com は TLD で contoso は第 2 レベル ドメイン名。  
  
 オフィス以外には、サーバー上の共有ファイルまたはネットワーク上のコンピューターにアクセスするのにドメイン名を使用できます。 離れた場所にしたら、サーバーを管理することもできます。 たとえば、サーバーの contoso.com を登録します。 オフィス以外の場合は、種類、ラップトップ上の Web ブラウザーを開くことができます**contoso.com** Windows Server Essentials で設定したリモート Web アクセスのインスタンスに接続するアドレス] テキスト ボックスにします。  
  
###  <a name="BKMK_PersonalizedNames"></a>Microsoft の個人用ドメイン名を理解します。  
 Microsoft の個人用ドメイン名には、次の機能が含まれています。  
  
-   リモート Web アクセス用のカスタム ドメイン名 (たとえば、*yourhostname*. remotewebaccess.com)。 ドメイン名は、パブリック IP アドレスに関連付けられます。  
  
-   DNS 動的更新プロトコル サービス、パブリック IP アドレスが変更された場合、ドメイン名を使用して、リモート Web アクセスは中断されません。 通常、組織のブロード バンド接続のインターネット サービス プロバイダー (Isp) は、動的のパブリック IP アドレスを変更することを提供します。  
  
-   ドメイン名に関連付けられている信頼された証明書。  
  
 Microsoft の個人用ドメイン名を統合すると、サーバーで、Microsoft アカウント (旧 Windows Live ID) が必要です。 一度に 1 つにサインアップすることができます、Microsoft アカウントを持っていない場合、[Microsoft Hotmail](https://login.live.com/) Web サイト。  
  
> [!IMPORTANT]
>  Windows Live、Microsoft アカウントのパスワード、サーバーがサポートしていない特殊文字ことができます。 Microsoft の個人用ドメインを使用する場合は、Microsoft アカウントのパスワードが、サーバーがサポートする文字のみが含まれていることを確認します。 サーバーは、$ の使用をサポートしていません/、'、および % です。

###  <a name="BKMK_Prefixes"></a>ドメイン名のプレフィックスを選択します。  
 ドメイン名は階層的なラベルで構成されます。

 **第 2 レベル ドメイン名**トップレベル ドメイン名拡張子の横にあるラベルです。 第 2 レベル ドメイン名が多くの場合、会社名、製品、またはサービスに基づいて作成されます。 たとえば、www.contoso.com で、contoso は、第 2 レベル ドメイン名は、し、会社名 Contoso pharmaceuticals から選択されています。 第 2 レベル ドメインは、関連付けられている IP アドレスを持つホスト名とも呼ばれます。  
  
 **ドメイン名のプレフィックス**、サブドメインを識別します。 サブドメイン名は、サービス、デバイス、または地域を識別するために使用します。 たとえば、Contoso Pharmaceuticals はリモート ユーザーがリモート Web アクセスにログオンできるようにしたいができるように、パブリック Web サイトにアクセスする適切なアクセス許可を持つユーザーのみを許可するサブドメインを作成するために、Web サイトがしたくないです。 Contoso Pharmaceuticals は、サブドメインとして remote.contoso.com を設定し、リモート ドメイン名のプレフィックスは、します。

###  <a name="BKMK_Extension"></a>ドメイン名拡張子を選択します。  
 Internet の Web サイトのドメイン名を選択した場合を使用するドメイン名の拡張子を指定する必要があります。 拡張機能は、以下の任意のドメイン名の最後のピリオドの文字で識別されます。 (拡張子の正式な名称はトップレベル ドメインまたは TLD) です。  
  
 使用できるドメイン拡張子の 2 つの主な種類があります: 汎用と国コードです。  
  
#### <a name="generic-top-level-domains"></a>汎用トップレベル ドメイン  
 汎用ドメイン拡張子は長さで、次の 3 つまたは複数の文字し、通常、特定の種類の組織が使用します。  
  
 **汎用トップレベル ドメインの例**  
  
|ドメイン拡張子|説明|  
|----------------------|-----------------|  
|.com|通常、商業組織によって使用しますが、だれでも使用することができます。|  
|.net|ネットワーク インフラストラクチャ サービスを提供する企業向けに設計されています。|  
|.org|最初は、非営利機関や他の汎用トップレベル ドメインの別のカテゴリに分類されない企業によって使用されます。 だれでも使用できます。|  
|.edu|教育機関用に制限されています。|  
  
#### <a name="country-code-top-level-domains"></a>国コード トップレベル ドメイン  
 これらのドメイン拡張子は、次の 2 つの文字の長さです。 国または地域コードに関連付けられている内の組織で使用するように設計されています。 一部の国コード トップレベル ドメインは、その国または地域の住民のみが使用する制限されています。 他のユーザーは、だれでも使用できます。  
  
 **国コード トップレベル ドメインの例**  
  
|ドメイン拡張子|説明|  
|----------------------|-----------------|  
|.ca|カナダの Web サイトで使用|  
|.cn|中国で Web サイトで使用|  
|.de|ドイツの Web サイトで使用|  
|。co.uk|英国の Web サイトで使用|  
  
 トップレベル ドメインの完全なリストを参照してください、[Internet Assigned Numbers Authority の Web サイト](https://go.microsoft.com/fwlink/?LinkId=117438)します。  
  
#### <a name="if-a-domain-extension-is-not-available-to-select-in-the-set-up-domain-name-wizard"></a>ドメイン拡張子がウィザードで選択設定をドメイン名を使用できない場合  
 設定 Up Domain Name ウィザードを実行すると、ウィザードは、国または地域を決定する、システム情報を検索します。 ウィザードは、お住まいの地域の提携プロバイダーをサポートするドメイン拡張子のみを表示します。 ドメイン拡張子が一覧に表示されない場合、続行する別のドメイン拡張子を選択する必要があります。 ウィザードが返される一覧から拡張子を選択します。

###  <a name="BKMK_SetNameManually"></a>ドメイン名を手動で設定します。  
 このオプションを選択する場合、サーバーは監視し、管理は、ドメイン名とことはない警告が表示、構成の問題がある場合。 次のいずれかが当てはまる場合は、このオプションを考慮する可能性があります。  
  
-   国または地域のパートナー ドメイン ネーム プロバイダーが一覧表示されません。  
  
-   表示されているパートナー ドメイン プロバイダーは、ドメイン名の拡張子をサポートしていません。  
  
-   現在、パートナーではないドメイン ネーム プロバイダーからの既存のドメイン名があるし、そのドメイン名を Windows Server Essentials のサポートされるドメイン ネーム プロバイダーに転送したくないです。  
  
-   ウィザードでは、使用するドメイン名の拡張子は表示されませんが、拡張子が現在パートナーではないドメイン ネーム プロバイダーから利用できます。  
  
 ドメイン名を手動で設定する場合は、ドメインの A レコードを作成するには、ドメイン ネーム サービス プロバイダーを使用します。  
  
##### <a name="to-create-an-a-record"></a>A レコードを作成するには  
  
1.  リモートなどのホスト名を決定します。 これは、ドメイン名のプレフィックスです。 ドメイン名のプレフィックスとドメイン名を開くには、リモート Web アクセスのログオン ページの URL を定義しますたとえば、**http://remote.contoso.com**します。  
  
2.  ダッシュ ボードでドメイン名サービス プロバイダーの構成 (通常は Web ページ) に、手順 1 で決定したホスト名の A レコードを作成します。 A レコードで指定した IP アドレスが (インターネットに面して側)、ルーターの WAN 側の IP アドレスであることを確認します。 Wan 側の IP アドレスを確認する、ルーターのドキュメントを参照してください。  
  
3.  インターネット サービス プロバイダー (ISP)、ネットワークの静的 IP アドレスを購入するを接続することをお勧めします。 これにより、IP アドレスが変わらないことと、DNS エントリが古くなるいないしません。  
  
     ISP から静的 IP アドレスを取得するためのオプションを持っていない場合、ドメイン ネーム サービス プロバイダーまたは他のサービス プロバイダーからドメイン ネーム システム (DNS) 動的更新プロトコル サービスを購入することも検討する可能性があります。 DNS 動的更新プロトコルは、IP アドレスが変更された場合でも、IP アドレスが、ドメイン名に解決できるように、ネットワークの wan 側の IP アドレスを最新の状態に保つサービスです。  
  
4.  入力が求められたら、ウィザードは、信頼された証明書をインポートします。 信頼された証明書を持っていない場合、ウィザードに表示されるサポートされているドメイン名プロバイダーの 1 つからいずれかを取得したり、好みの信頼できるプロバイダーから購入できます。 信頼された証明書の詳細については、ドメイン ネーム プロバイダーにお問い合わせください。

#### <a name="image-size"></a>イメージのサイズ  
 **ロゴのイメージ**  
  
 ロゴの画像は 32 x 32 ピクセルを使用することをお勧めします。 大きな画像は 32 x 32 を縮小しより小さな画像は 32 x 32 は、イメージがゆがむことがこれを拡大します。  
  
 **背景画像**  
  
 最良の結果を背景画像のサイズの制限がないときに、およそ 800 x 500 ピクセルの画像を使用することをお勧めします。 背景画像は、ログオン ページの中央 (水平および垂直方向) に配置されます。 [ログオン] ページで、テキストを読みやすいものために、背景画像の中心には薄い色ことがあります。

-   ビットマップ (*.bmp、\*.dib、\*.rle)  
  
-   GIF (*.gif)  
  
-   PNG (*.png)  
  
-   JPG (*.jpg)

2.  をクリックして**設定**、クリックして、**Anywhere Access** ] タブ。  
  
3.  をクリックして**修復**します。 **リモート Web アクセスの修復**ウィザードが起動します。  
  
4.  をクリックして**次**します。 ウィザードは、リモート Web アクセスが分析され、問題が特定、問題の修復が試行されます。  
  
5.  クリックして、ウィザードが完了すると、アラートを受け取った場合**再試行**もう一度、問題の修復を試みます。 アラートを受け取るには、続行する場合は、アラートの詳細については、問題とトラブルシューティングの手順を確認します。
