###  <a name="BKMK_Manage7"></a>ユーザー アカウントを概要します。  
 ユーザー アカウントは、Windows Server Essentials、これにより、個人情報は、サーバーに保存され、個々 のユーザーを作成して、ファイルと設定を管理できるようになりますにアクセスする重要な情報を提供します。 Windows Server Essentials ユーザー アカウントを持っていると、コンピューターにアクセスするアクセス許可を持っている場合、ユーザーがネットワーク上の任意のコンピューターにログオンことができます。 自分のユーザー名とパスワードで各自のユーザー アカウントをユーザーにアクセスします。  
  
 ユーザー アカウントの 2 つの主な種類があります。 各種類により、ユーザーは、コンピューターを制御のレベルが異なります。  
  
-   **標準的な**アカウントは、日常的なコンピューティング用です。 標準のアカウントでは、ユーザーがファイルを削除するか、ネットワーク設定の変更など、他のユーザーに影響する変更を加えることを防ぐことでネットワークを保護するのに役立ちます。  
  
-   **管理者**アカウントは、コンピューター ネットワーク経由で、ほとんどの制御を提供します。 管理者を割り当てる必要がありますアカウントの種類に必要な場合にのみです。

###  <a name="BKMK_Access3"></a>パスワード ポリシーについて知っておくべき新機能  
 パスワード ポリシーは、ユーザーの作成し、パスワードの使用を定義する規則のセットです。 ポリシーは、ユーザー データと、サーバーに格納されているその他の情報を不正アクセスを防ぐのに役立ちます。 パスワード ポリシーは、ネットワークにアクセスするすべてのユーザー アカウントに適用されます。  
  
 Windows Server Essentials パスワード ポリシーは、次の 3 つの主な要素のとおりから成ります。  
  
-   **パスワードの長さ**します。  長くパスワードでは、セキュリティを強化します。 空のパスワードは安全ではありません。  
  
-   **パスワードの複雑さ**します。  複雑なパスワードに大文字が含まれているし、小文字 (a ~ z、A ~ Z)、base 数字 (0 ~ 9)、およびアルファベット以外の記号 (次のように;!、@、#、_、-)。 複雑なパスワードが大幅に低下未承認のアクセスを受けやすくなります。 ユーザー名、生誕日、またはその他の個人情報が含まれているパスワードは、適切なセキュリティを提供しません。  
  
-   **パスワードの有効期間**します。  Windows Server Essentials では、ユーザーが、少なくとも 180 日ごとに 1 回についてパスワードを変更している必要があります。 オプションとして、パスワードを無期限に選択できます。  
  
 やすく、コンピューター ネットワークでパスワード ポリシーを実装するためにを Windows Server Essentials には、設定または次の 4 つの事前に定義されたポリシー プロファイルのいずれかにパスワード ポリシーを変更することができます簡単なツールが用意されています。  
  
-   **脆弱な**します。  ユーザーは、任意のパスワードは空白でないことを指定できます。  
  
-   **メディア**します。  これらのパスワードは、少なくとも 5 文字を含める必要があります。 複雑なパスワードは必要ありません。  
  
-   **強い**します。  これらのパスワードは、少なくとも 5 文字を含める必要があり、文字、数字、および記号を含める必要があります。  
  
-   **厳密な**します。  これらのパスワードは、少なくとも 7 文字を含める必要があり、文字、数字、および記号を含める必要があります。 これらのパスワードは安全性を高めるが、ユーザーにとって覚えにくい場合があります。  
  
    > [!NOTE]
    >  パスワードは、ユーザーの名前やメール アドレスを含めることはできません。  
    >   
    >  Office 365 を統合する場合、適用、**厳密な**パスワード ポリシーをし、次の要件を含めるようにポリシーを更新します。  
    >   
    >  -   パスワードは、8 16 文字を含める必要があります。  
    > -   パスワードは、スペースや Office 365 の電子メール名を含めることはできません。  
  
 既定では、サーバーのインストール既定のパスワード ポリシーを設定、**強**オプションです。

###  <a name="BKMK_Access7"></a>ネットワーク管理者のパスワードと DSRM パスワードを同期します。

###  <a name="BKMK_Access8"></a>ユーザー アカウントにリモート デスクトップ アクセス許可を付与します。

###  <a name="BKMK_Access12"></a>ユーザー アカウントの内部共有フォルダーへのアクセスを変更します。  
 タスクを使用して、サーバー上の任意の共有フォルダーへのアクセスを管理することができます、**サーバー フォルダー**ダッシュ ボードのタブ。 既定では、次のサーバー フォルダーは、Windows Server Essentials をインストールするときに作成されます。

-   **会社**します。  保存し、ネットワーク ユーザーが組織に関連するドキュメントにアクセスするために使用します。  
  
-   **ファイル履歴バックアップ**します。  既定では、Windows Server Essentials は、ファイル履歴を使用して作成されたファイルのバックアップを格納します。 このサーバー フォルダーは共有されません。  
  
-   **フォルダー リダイレクト**します。  保存し、ネットワーク ユーザーがフォルダー リダイレクトに設定されているフォルダーにアクセスするために使用します。 このサーバー フォルダーは共有されません。  
  
-   **音楽**します。  保存し、ネットワーク ユーザーが音楽ファイルにアクセスするために使用します。 メディア共有をオンにすると、このフォルダーが作成されます。  
  
-   **画像**します。  保存し、ネットワーク ユーザーが画像へのアクセスに使用されます。 メディア共有をオンにすると、このフォルダーが作成されます。  
  
-   **テレビ録画**します。  保存し、ネットワーク ユーザーが録画されたテレビ番組のアクセスに使用されます。 メディア共有をオンにすると、このフォルダーが作成されます。  
  
-   **ビデオ**します。  保存し、ネットワーク ユーザーがビデオにアクセスするために使用します。 メディア共有をオンにすると、このフォルダーが作成されます。  
  
-   **ユーザー**します。  保存し、ネットワーク ユーザーがファイルにアクセスするために使用します。 ユーザー固有のフォルダーが自動的に生成、**ユーザー**サーバー フォルダーを作成するすべてのネットワーク ユーザー アカウントにします。
