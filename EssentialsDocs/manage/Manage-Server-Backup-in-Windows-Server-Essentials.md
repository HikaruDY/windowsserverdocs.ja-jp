##  <a name="BKMK_WhichBackup"></a>バックアップを選択する必要がありますか。  
 ごく最近の正しいバックアップがあり、バックアップに重要なデータが含まれているがわかっている場合、バックアップの選択が簡単ですがあります。 古いバックアップから、サーバーまたはコンピューターに復元しようとする場合に復元する正常なバックアップを選択する必要があります調査や場合によっては、いくつかの侵害します。  
  
#### <a name="to-choose-a-backup"></a>バックアップを選択するには  
  
1.  ファイルまたはフォルダーの所有者をチェックし、日付と時刻に注意してください追加または編集します。 これらの日付と時刻を開始点として使用します。  
  
2.  **復元オプションを選択して**復元のファイルとフォルダーのウィザード ページで、[ **(詳細) を選択したバックアップから復元**します。  
  
3.  ファイルまたはフォルダーの古いまたは新しいバージョンを復元するかどうかに応じて、手順 1 で説明した日時に最適なバックアップを選択します。  
  
4.  ベスト プラクティスとして、別の場所にファイルとフォルダーを復元して使用するために、ファイルの所有者と、フォルダーは、元の場所に必要なものを移動します。 終了したらファイルと、別の場所に残っているフォルダーを削除することができます。  
  
##  <a name="BKMK_1"></a>セットアップまたはサーバー バックアップのカスタマイズ  
 インストール中に、サーバーのバックアップが自動的に構成されていません。 毎日のバックアップをスケジュールすることによって、サーバーとそのデータを自動的に保護する必要があります。 ほとんどの組織は数日間作成されたデータを失う余裕があるために、毎日のバックアップ計画を維持することをお勧めします。 詳細については、次を参照してください。[設定のセットアップまたはサーバー バックアップのカスタマイズ](Set-up-or-customize-server-backup.md)します。  
  
##  <a name="BKMK_2"></a>進行中のサーバー バックアップを停止します。  
 サーバーのバックアップが、定期的にスケジュールされた時刻に開始する、またはサーバー バックアップを手動で開始するかどうかは、進行中のバックアップを停止することができます。  
  
#### <a name="to-stop-a-backup-in-progress"></a>進行中のバックアップを停止するには  
  
1.  ダッシュ ボードを開きます。  
  
2.  ナビゲーション バーで、クリックして**デバイス**します。  
  
3.  コンピューターの一覧で、[サーバー] をクリックし、をクリックして**停止サーバーのバックアップの**で、**タスク**ウィンドウ。  
  
4.  をクリックして**はい**、操作を確定します。

##  <a name="BKMK_5"></a>サーバー バックアップの設定について詳しく知る  
 サーバーのバックアップはサーバーのセットアップ中に有効になりません。  
  
> [!NOTE]
>  サーバーのバックアップを構成するときに、バックアップ先のハード ドライブとして使用するサーバーに少なくとも 1 つの外部ハード ドライブを接続する必要があります。  
  
###  <a name="BKMK_Target"></a>バックアップ先ドライブ  
 バックアップの場合、複数の外部の記憶域ドライブを使用して、オンサイトとオフサイトの記憶域の場所をローテーションすることができます。 これは、障害対策計画を支援オンサイトのハードウェアで物理的破損が発生した場合、データを回復することによってを改善できます。  
  
 サーバー バックアップ用の記憶域ドライブを選択する場合は、次を考慮してください。  
  
-   データを格納するための十分な領域が含まれているドライブを選択します。 記憶域ドライブには、少なくとも 2.5 倍の記憶域容量をバックアップするデータを含める必要があります。 ドライブは、サーバー データの将来の成長に対応するのに十分な大きさもする必要があります。  
  
-   バックアップ先ドライブにオフライン ドライブが含まれている場合、バックアップの構成は成功しません。 構成が完了し、バックアップ先を選択する場合、オフラインになっているドライブを除外するチェック ボックスをオフにします。  
  
-   バックアップ先として以前のバックアップが含まれているドライブを選択した場合、ウィザードでは、以前のバックアップを保持することができます。 バックアップを保持する場合、ウィザードは、ドライブをフォーマットしません。  
  
-   外部記憶域ドライブを再利用するときに、ドライブが空か不必要なデータしか含まれていることを確認します。  
  
-   Windows Server Essentials を実行するコンピューターで、バックアップ ドライブがサポートされていることを確認する外部の記憶域ドライブの製造元の Web サイトにアクセスする必要があります。  
  
> [!CAUTION]
>  バックアップのセットアップ サーバー ウィザードは、バックアップ用に構成するとき、記憶域ドライブをフォーマットします。  
  
### <a name="server-backup-schedule"></a>サーバー バックアップのスケジュール  
 毎日のバックアップをスケジュールすることによって、サーバーとそのデータを自動的に保護する必要があります。 ほとんどの組織は数日間作成されたデータを失う余裕があるために、毎日のバックアップ計画を維持することをお勧めします。  
  
 Windows Server Essentials のセットアップ サーバーのバックアップ ウィザードを使用すると、1 日に複数回サーバー データをバックアップすることもできます。 ウィザードは差分ベースのバックアップをスケジュール設定ため、バックアップが迅速に実行され、サーバーのパフォーマンスが大幅に影響はありません。 既定では、**設定をサーバー バックアップ**午後 12時 00分と午後 11時 00分に毎日実行するバックアップをスケジュールします。 ただし、組織のニーズに合わせて、バックアップのスケジュールを調整できます。 状況によって、バックアップ計画の効果を評価、必要に応じて計画を変更する必要があります。  
  
> [!NOTE]
>  Windows Server Essentials の既定のインストール、サーバーが自動的に毎週 1 回最適化を実行するように構成します。 これは、結果、Microsoft 以外のイメージング ソフトウェアを使用する場合、バックアップが通常よりも大きい。 場合は、定期的にサーバーを最適化する必要はありません、する次の手順を実行最適化のスケジュールを無効にします。  
>   
>  1.  Windows キー + を開くには W キーを押します**検索**します。  
> 2.  [検索] ボックスで、次のように入力します。**最適化**します。  
> 3.  結果のセクションで [**を最適化するドライブのデフラグと**します。  
> 4.  **ドライブ最適化**] ページで、ドライブを選択してクリックして**設定を変更する**します。  
> 5.  **最適化のスケジュール**] ウィンドウで、クリア、**(推奨) スケジュールに従って実行**チェック ボックスをオンし、をクリックして**OK**変更を保存します。  
  
### <a name="items-to-be-backed-up"></a>バックアップする項目  
 既定では、すべてのオペレーティング システム ファイルとフォルダーはバックアップの選択します。 サーバーで、すべてのハード_ディスク、ファイル、およびフォルダーをバックアップまたは個々 のハード_ディスク、ファイル、またはフォルダーをバックアップする] を選択することができます。 バックアップの項目を追加または削除、次のいずれかの操作を行います。  
  
-   サーバー バックアップのデータ ドライブは、隣接するチェック ボックスを選択します。  
  
-   サーバー バックアップからデータ ドライブを除外する、隣接するチェック ボックスをオフにします。  
  
> [!NOTE]
>  除外する場合、**オペレーティング システム**項目、バックアップする必要がありますまずをオフにする、**システム バックアップ (推奨)**チェック ボックスをオンします。  
  
 サーバー バックアップを使用するバックアップ用ハード_ディスク領域の量を最小限に抑える、有益または特に重要な考慮しないファイルが含まれるフォルダーを除外する可能性があります。  
  
 たとえば、大量のハード ドライブの空き領域を使用するフォルダーを含む録画したテレビ番組があります。 通常見終わった後に削除するために、これらのファイルをバックアップしないように選択することがあります。 または、保持する目的としない一時ファイルを格納するフォルダーがあります。  
  
##  <a name="BKMK_6"></a>サーバーのハード ドライブのパーティションを再作成します。  
 Windows Server Essentials サーバー上で、書式設定されていない内部ハード ディスク ドライブが検出されると、ハード ドライブの新規作成ウィザードの追加へのリンクを含む正常性アラートが生成されます。 追加のハード ドライブの新規作成ウィザードでは、ハード ドライブをフォーマットのさまざまなオプションについて説明します。 ウィザードが終了したら、ときに 1 つまたは複数のドライブのサイズに応じて、論理ハード フォーマットされたドライブがハード ドライブ上に作成し、NTFS としてフォーマットします。  
  
 ハード ディスク ドライブのパーティションを再作成が必要になった場合は、次の手順に従います。  
  
#### <a name="to-repartition-a-hard-disk-drive"></a>ハード ディスク ドライブのパーティションを再作成するには

2.  [コンピューターの管理] をクリックして**記憶域**、順にダブルクリック**ディスクの管理**します。  
  
3.  パーティション分割をやり直します] をクリックするドライブを右クリックして**ボリュームの削除**、] をクリックし、**はい**します。  
  
    > [!NOTE]
    >  ハード ディスク ドライブ上のパーティションごとに、この手順を繰り返します。  
  
4.  右クリックし、**未割り当て**ハード ディスク ドライブ、およびクリック**新しいシンプル ボリューム**します。  
  
5.  新しいシンプル ボリューム ウィザードで作成し、16 tb (16,000, 000 MB) のボリュームをフォーマットまたは小さくします。  
  
    > [!NOTE]
    >  この手順を繰り返して、ハード ディスク ドライブ上のすべての未割り当て領域を使用します。
