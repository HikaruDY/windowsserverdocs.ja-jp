# <a name="create-a-server-recovery-dvd-for-remotely-administered-servers"></a>リモートで管理されるサーバーのサーバー回復 DVD を作成します。

>Windows Server 2016 Essentials、Windows Server 2012 R2 Essentials での Windows Server 2012 Essentials を適用対象:

##  <a name="BKMK_HeadlessRecovery"></a>リモートで管理されるサーバーのサーバー回復 DVD の作成します。  
 工場出荷時のリセットとサーバーの回復のための 2 つのモデルがあり、顧客に納入したハードウェアによって異なります。  
  
 **リモートで管理されるサーバー**  
  
 このサーバーの回復オプションは、同じネットワーク上のクライアント コンピューターからプロセスを実行する必要があります。 工場出荷時リセットにより、ハードウェア固有のイメージをサーバーと共に出荷することが必要であるため、パートナーは、サーバー回復 DVD を作成する必要があります。  
  
 **ローカルで管理されるサーバー**  
  
 これは、従来のモデル、サーバーのコンソールで、サーバーを管理する場所です。 サーバーのインストール メディアを使用すると、回復を実行します。 これは、サーバーには DVD リーダーに加え、ビデオ出力を表示する機能が付属する必要があります。 お客様は、そのサーバーのインストール メディアから起動し、適切な回復方法を選択します。 ローカルで管理されているサーバーのサーバー回復 DVD を作成する必要はありません。  
  
> [!NOTE]
>  ローカルで管理されるサーバーのモデル、お客様は、工場出荷時の新規インストールを実行することでリセットを行う可能性があります。 ただし、カスタマイズ、パートナーが失われます。  
  
 サーバーの回復の 2 つの種類があります。  
  
 **工場出荷時リセット**  
  
 この回復では、元の状態に、サーバーは工場から出荷されたときに、サーバーを返します。 工場出荷時リセットでは、次を初めてオンにした、したのと同じように、サーバーの初期構成を実行するよう求められます、すべての設定とカスタマイズは失われます。 これは、別名 Day 0 ですか?。 工場出荷時リセットにより、ハードウェア固有のイメージをサーバーと共に出荷することが必要であるため、パートナーは、サーバー回復 DVD を作成する必要があります。  
  
 **ベアメタル復元**  
  
 この回復では、サーバーのバックアップが構成されていること、およびサーバー バックアップを完了が正常にサーバーの障害発生前に少なくとも 1 回想定しています。 ベア メタル復元 (BMR) は、以前のサーバー バックアップからのみシステムとブート ドライブの回復をサポートします。  
  
 BMR の後、サーバーが、復元に使用されるバックアップの時点の状態に返されます。 これは、通常、最新のバックアップが、場合によっては、以前のバックアップがあります。 データの回復は、復元のファイルとフォルダーのウィザードを使用して、システムを復元した後に行われます。 BMR が優先サーバーの回復方法すべての設定と構成が返されたため、工場出荷時リセットが Day 0 状態に、サーバーを返します。  
  
### <a name="remotely-administered-server-recovery"></a>サーバーの回復をリモートで管理されます。  
 このセクションでは、パートナーを実行する必要がありますが必要なカスタマイズし、最終的なメディアを各サーバーと共に出荷する必要がありますについて説明します。 詳細を詳しく調べる前にカスタマー エクスペリエンスについて見てみましょう。  
  
 このシナリオでは、お客様に"™ のサーバーが動作していません。 これは、ウイルス、ハード_ディスク障害、またはその他のいくつかの原因によって発生可能性があります。 お客様は、サーバーと同じネットワークに配置されているクライアント コンピューターにサーバー回復 DVD を挿入します。 サーバー回復アプリケーションでは、そのサーバーを回復する 3 つの手順を実行します。  
  
1.  回復モードでサーバーを再起動するために使用する起動可能な USB フラッシュ ドライブを作成します。 USB フラッシュ ドライブは 8 GB である必要がありますか大きい。  
  
2.  現在回復モードであるサーバーを検出します。  
  
3.  により、お客様は工場出荷時リセットまたはベアメタル復元を選択し、稼働状態に、サーバーを返します。

####  <a name="BKMK_Updating"></a>手順 1: 更新プログラムが (省略可能) WinPE  
 ADK には、カスタマイズされた Windows PE のコピーが含まれています。 このイメージが起動したときに、クライアント回復アプリケーションで回復モードでのサーバーに接続するために使用するビーコンが自動的に起動します。  
  
 Windows PE は、ネットワークなど、ハードウェア固有のドライバーやディスク コントローラー用ドライバーを追加することで、さらにカスタマイズする必要があります。 WinPE からの起動後、システム上のハード_ディスクが認識する必要があるし、ネットワークが動作する必要があります。  
  
####  <a name="BKMK_Collecting"></a>手順 2: 工場出荷時リセット イメージと XML ファイルを収集します。  
 サーバーを工場出荷時の既定値にリセットするには、次の 2 つのイメージをキャプチャする必要があります。  
  
-   システム ドライブのイメージ  
  
-   システム予約パーティション  
  
 これらのイメージをキャプチャするには、GenDiskXML.exe ツールが提供されます。 GenDiskXML.exe はまた、回復プロセスでディスク構成を再作成に使用される disk.xml という名前のファイルを収集します。  
  
1.  Sysprep に続いて、64 ビット バージョンの Windows PE を使用して、システムを再起動します。  
  
2.  .Xml ファイルと .wim ファイルを外部ソースに出力するには、実行`GenDiskXML /outputdir:<path>`、.xml ファイルと .wim ファイルを任意の外部ソースに出力します。 ファイルは、次の手順で DVD に追加されます。  
  
    > [!NOTE]
    >  システムの .wim ファイルは、4 GB を超えていないファイルの FAT32 要件を満たす分割されます。 中、.wim ファイルのキャプチャに使用されるターゲットの必要容量は、分割プロセスに対応できるように 8 GB よりもさらに拡大する必要があります。

###### <a name="to-create-the-server-recovery-dvd"></a>サーバー回復 DVD を作成するには  
  
1.  最終 ISO を保存するための場所として使用する作業フォルダーを作成します。  
  
2.  パートナー CD からの内容をコピー、**サーバー回復**手順 1 で作成した作業フォルダーのフォルダーです。  
  
3.  GenDiskXML.exe の実行時に作成された disk.xml ファイルをコピー、**リセット**フォルダー。  
  
4.  実行したときに作成されたイメージ ファイルをコピー **GenDiskXML.exe**を**リセット**フォルダー。 ファイルが、.wim ファイルと .swm ファイル、およびファイルの数が異なる場合があります。  
  
5.  フォルダーから GenDiskXML.exe を削除します。 工場出荷時の目的にのみ使用され、顧客に出荷する DVD に含めることができません。
