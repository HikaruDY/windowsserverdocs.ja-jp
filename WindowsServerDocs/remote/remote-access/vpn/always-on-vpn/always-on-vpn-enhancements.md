# Always On VPN の機能強化

>適用対象: Windows Server (半期チャネル)、Windows Server 2016、Windows 10

& #171 です。[**前:** Always On VPN の機能について説明します](../vpn-map-da.md)<br>
& #187 です。[ **[次へ]:** Always On VPN テクノロジについて説明します](always-on-vpn-technology-overview.md)

Always On VPN では、多くのメリットを従来の Windows VPN ソリューションにします。 次のキーの機能強化は、マイクロソフトのクラウド ファースト クラウドファースト、モバイルファーストのビジョンと Always On VPN を調整します。

- **プラットフォーム統合:** Always On VPN には、Windows オペレーティング システムと高度な接続の無数のシナリオの堅牢なプラットフォームを提供するサード パーティ製のソリューションとの統合の強化。

- **セキュリティ:** Always On VPN に新しい、高度なセキュリティ機能をアプリケーションが VPN 接続を使用して、トラフィックの種類を制限する認証方法を使って、接続を開始するとします。 接続がアクティブなときほとんどの場合は、接続をセキュリティで保護する特に重要です。 詳細については、 [VPN 認証オプション](https://docs.microsoft.com/en-us/windows/security/identity-protection/vpn/vpn-authentication)を参照してください。

- **VPN 接続:** Always On VPN でデバイス トンネルの有無には、自動トリガー機能を提供します。 Always On VPN での前にユーザーまたはデバイスの認証のいずれかで自動接続をトリガーする機能できませんでした。  

- **ネットワーク制御:** Always On VPN により、管理者は詳細なレベルでのルーティング ポリシーを指定する: 個々 のアプリケーションにも-これは特殊なリモート アクセスを必要とする基幹業務 (LOB) アプリに最適です。  Always On VPN はインターネット プロトコル バージョン 4 (IPv4) の両方で完全な互換性もとバージョン 6 (IPv6)。 DirectAccess とは異なりにはありません特定の依存関係 IPv6 します。

  >[!Note]
  >開始する前に、VPN サーバーで IPv6 を有効にすることを確認します。 それ以外の場合、接続を確立できないされ、エラー メッセージが表示されます。
  
- **構成との互換性:** Always On VPN を展開し、Always On VPN 経由での他の VPN クライアント ソフトウェアのいくつかの利点をいくつかの方法を管理します。

## VPN 接続

Always On VPN 接続の主な機能強化を次に示します。

| 重要な機能強化 | 説明 |
| ---- | ---- |
| **常にオン** | アクティブな VPN プロファイルを自動的に接続し、トリガーに基づく接続を維持できる Windows 10 の機能は、常に、つまり、ユーザーのサインイン、ネットワークの状態の変更、またはアクティブなデバイス画面。 Always On はも統合、バッテリーの寿命を最大化する場合は、接続されているスタンバイ エクスペリエンスできます。 |
| **アプリケーションをトリガーします。** | 指定した一連のアプリケーションの起動時に自動的に接続する Windows 10 では、VPN プロファイルを構成できます。 デスクトップと VPN 接続をトリガーする UWP アプリケーションの両方を構成することができます。 |
| **名前ベースのトリガー** | Always On VPN での特定のドメイン名のクエリが VPN 接続をトリガーするように規則を定義できます。 Windows 10 をサポートしています名前ベースのドメインに参加しているし、ドメインに参加しているコンピューターのトリガー (以前は、ドメインに参加しているコンピューターのみがサポートされていました。 |
| **信頼されたネットワークの検出** | Always On VPN には、ユーザーが企業の境界内で、信頼されたネットワークに接続されている場合に、VPN 接続がトリガーされないことを確認するには、この機能が含まれています。 この機能は、「が必要な場合にのみ接続」のシームレスなユーザー エクスペリエンスを提供する前に述べたトリガーする方法のいずれかを組み合わせることができます。 |
| **[デバイス トンネル](../vpn-device-tunnel-config.md)** | Always On VPN では、デバイスまたはコンピューターに専用の VPN プロファイルを作成する機能を提供します。 _ユーザー トンネル_デバイスやコンピューターのログオンをユーザーにのみ接続すると、これとは異なり、_デバイス トンネル_はユーザーにサインインする前に接続を確立する VPN をできます。 デバイス トンネルとユーザー トンネルの両方の VPN プロファイルで独立して動作と同時に、接続されているおよびに応じてさまざまな認証方法とその他の VPN 構成の設定を使用できます。 |

| 重要な機能強化 | 説明 |
| ---- | ---- |
| **IPv4 と IPv6 のデュアル スタックのサポート** | Always On VPN では、デュアル スタック アプローチでは、IPv4 と IPv6 の両方の使用をネイティブにサポートします。 ありません特定の依存関係の 1 つのプロトコル経由で他のネットワークのニーズ、将来の IPv6 のサポートと組み合わせることにより、最大 IPv4 または IPv6 アプリケーションの互換性なことができます。<p>**_注: _** 開始する前に、VPN サーバーで IPv6 を有効にすることを確認します。 それ以外の場合、接続を確立できないされ、エラー メッセージが表示されます。|

## 構成との互換性 

Always On vpn の構成と互換性の機能強化の一部を次に示します。

| 重要な機能強化 | 説明 |
| ---- | ---- |
| **サード パーティ製の VPN ゲートウェイの互換性** | Always On VPN クライアントでは、動作する Microsoft ベースの VPN ゲートウェイの使用は必要ありません。 IKEv2 プロトコルのサポート、を通じて、クライアントでは、この業界標準のトンネリングの種類をサポートするサード パーティ製 VPN ゲートウェイと相互運用性が容易になります。 また、UWP の VPN プラグイン Always On VPN プラットフォームの機能とメリットを損なうことがなく、カスタムのトンネリングの種類を組み合わせたを使用して、サード パーティ製の VPN ゲートウェイと相互運用性を実現できます。<p><p>_**注: **_<br>構成と Always On VPN および IKEv2 を使用して、デバイス トンネルとの互換性、ゲートウェイまたはサード パーティ製のバックエンド アプライアンス ベンダーに問い合わせてください。 |
| **業界標準 ikev2 ベースの VPN プロトコルのサポート** | Always On VPN クライアントは、IKEv2 をサポートしている、今日のいずれかの最も広く業界標準のトンネリング プロトコルを使用します。 この互換性は、サード パーティ製の VPN ゲートウェイと相互運用性を最大化します。 |

| **さまざまな管理と展開メカニズム** | 設定を管理する VPN ( _VPN プロファイル_と呼ばれます)、Windows PowerShell、System Center Configuration Manager、Intune (またはサード パーティ製のモバイル デバイス管理 [MDM] ツール) を含む多くの管理と展開メカニズムを使用して Windows構成デザイナー。 これらのオプションは、使用するクライアント管理ツールに関係なく Always On VPN の構成を簡略化します。 |
