# 手順 2.  サーバー インフラストラクチャを構成します。

>適用対象: Windows Server (半期チャネル)、Windows Server 2016、Windows Server 2012 R2、Windows 10

& #171 です。 [**前:** 手順 1 です。Always On VPN 展開を計画します。](always-on-vpn-deploy-planning.md)<br>

この手順では、インストールして、VPN をサポートするために必要なサーバー側のコンポーネントを構成します。 サーバー側のコンポーネントには、ユーザー、VPN サーバー、および NPS サーバーによって使用される証明書を配布するための PKI の構成が含まれます。  また、IKEv2 接続および VPN 接続の認証を実行する NPS サーバーをサポートするために RRAS を構成します。

## グループ ポリシーで証明書の自動登録を構成します。
この手順でグループ ポリシーを構成、ドメイン コント ローラーでドメインのメンバーが自動的にユーザーとコンピューターの証明書を要求できるようにします。 これにより、VPN 要求し、VPN 接続を自動的に認証されたユーザーの証明書を取得できます。 同様に、このポリシーで許可サーバーを要求する NPS サーバー認証証明書を自動的にします。 

VPN サーバー上の証明書を手動で登録します。

>[!TIP]
>非 domained に参加しているコンピューターでは、[ドメイン以外の CA の構成には、コンピューターが参加している](#ca-configuration-for-non-domain-joined-computers)以下を参照してください。 RRAS サーバーがドメインに参加しているではないために、VPN ゲートウェイの証明書の登録の自動登録を使用できません。  そのため、オフラインの証明書の要求手順を使用します。 


1.  ドメイン コント ローラーは、グループ ポリシーの管理を開きます。

2.  ナビゲーション ウィンドウで、ドメイン (corp.contoso.com など) を右クリックし、[**このドメインに GPO を作成しここでそのリンク**をクリックします。

3.  [新しい GPO] ダイアログ ボックスで、**ポリシーの自動登録**を入力し、 **[ok]** をクリックします。

4.  ナビゲーション ウィンドウで、**自動登録ポリシー**を右クリックし、**編集**] をクリックします。

5.  グループ ポリシー管理エディターで、次の手順をコンピューター証明書の自動登録を構成します。

    1.  ナビゲーション ウィンドウで、**コンピューター Configuration\\Policies\\Windows Settings\\Security Settings\\Public キーのポリシー**をクリックします。

    2.  詳細ウィンドウで**証明書サービス クライアント-自動登録**を右クリックし、[**プロパティ**] をクリックします。

    3.  証明書サービス クライアント-自動登録のプロパティ] ダイアログ ボックスの**構成モデル**では、[**有効**] をクリックします。

    4.  **証明書が切れた証明書、保留中の更新、失効した証明書を削除**選択と**証明書テンプレートを使用する証明書を更新**します。

    5.  **[Ok]** をクリックします。

6.  グループ ポリシー管理エディターでは、ユーザー証明書の自動登録を構成する次の手順を実行します。

    1.  ナビゲーション ウィンドウで、**ユーザー Configuration\\Policies\\Windows Settings\\Security Settings\\Public キーのポリシー**をクリックします。

    2.  詳細ウィンドウで**証明書サービス クライアント-自動登録**を右クリックし、[**プロパティ**] をクリックします。

    3.  証明書サービス クライアント-自動登録のプロパティ] ダイアログ ボックスの**構成モデル**では、[**有効**] をクリックします。

    4.  **証明書が切れた証明書、保留中の更新、失効した証明書を削除**選択と**証明書テンプレートを使用する証明書を更新**します。

    5.  **[Ok]** をクリックします。

    6.  グループ ポリシー管理エディターを閉じます。

7.  グループ ポリシーの管理。

RRAS サーバーがドメインに参加しているではないために、VPN ゲートウェイの証明書の登録の自動登録を使用できません。  そのため、オフラインの証明書の要求手順を使用します。 

1. RRAS サーバーでファイルを生成する例の証明書ポリシーに基づいてと呼ばれる**VPNGateway.inf**が (セクション 0) 付録で提供されているを要求し、次のエントリをカスタマイズします。

   - [拡張機能] のセクションでは選択された [_顧客_] VPN エンドポイント FQDN と代わりのサブジェクト名に使用 vpn.contoso.com に置き換えます。

2. 保存または**VPNGateway.inf**ファイルを選択した場所にコピーします。

3. 特権のコマンド プロンプトから、 **VPNGateway.inf**ファイルの種類を含むフォルダーに移動します。

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

6. 特権のコマンド プロンプトから、前の手順との種類で作成した VPNGateway.req ファイルが含まれているフォルダーに移動します。 


7. 証明機関のリスト] ウィンドウでは、メッセージが表示されたら、証明書の要求を処理するための適切なエンタープライズ CA を選択します。

8. RRAS サーバーには、新しく作成した**VPNGateway.cer**出力ファイルをコピーします。 

9. 保存または**VPNGateway.cer**ファイルを RRAS サーバーで選択した場所にコピーします。

10. 特権のコマンド プロンプトから、前の手順との種類で作成した VPNGateway.cer ファイルが含まれているフォルダーに移動します。
   

12. 次のプロパティを持つ RRAS サーバー用に有効な証明書が存在することを確認します。

## VPN ユーザー、VPN サーバー、および NPS サーバー グループを作成します。

この手順では、VPN を使用して、組織のネットワークに接続を許可するユーザーを含む新しい Active Directory (AD) グループを追加できます。

このグループには、2 つの目的があります。

-   ユーザーが VPN が必要とするユーザー証明書の自動登録を許可されているを定義します。

-   NPS を VPN へのアクセスを承認するユーザーを定義します。

By using a custom group, if you ever want to revoke a user’s VPN access, you can remove that user from the group.

また、VPN サーバーを含むグループと NPS サーバーが含まれている別のグループを追加します。 そのメンバーへの証明書の要求を制限するのにには、これらのグループを使用します。

>[!NOTE] 
>VPN をお勧めします DMA/境界内に存在するサーバーできないドメインに参加しています。 ただし、VPN サーバーを管理性のドメインに参加していることを希望する場合 (グループ ポリシー、バックアップ/監視エージェントでは、管理、およびためにローカル ユーザー)、し、広告にグループを追加、VPN サーバーの証明書テンプレート。

### VPN ユーザー グループを構成します。

1.  ドメイン コント ローラーは、Active Directory ユーザーとコンピューターを開きます。

2.  コンテナーまたは組織単位を右クリックして、**新しい****グループ**をクリックします。

3.  **グループの名前**、 **VPN ユーザー**、」と入力し、 **[ok]** をクリックします。

4.  **VPN ユーザー**、右クリックし、[**プロパティ**] をクリックします。

5.  [VPN ユーザーのプロパティ] ダイアログ ボックスの [**メンバー** ] タブで、[**追加**] をクリックします。

6.  ユーザーの選択] ダイアログ ボックスで、VPN が必要なすべてのユーザーを追加するアクセスし、 **[ok]** をクリックします。

7.  Active Directory ユーザーとコンピューターを閉じます。

### VPN サーバーおよび NPS サーバー グループを構成します。

1.  ドメイン コント ローラーは、Active Directory ユーザーとコンピューターを開きます。

2.  コンテナーまたは組織単位を右クリックして、**新しい****グループ**をクリックします。

3.  [**グループ名**、 **VPN サーバー**を入力し、 **[ok]** をクリックします。

4.  **VPN サーバー**、右クリックし、[**プロパティ**] をクリックします。

5.  VPN サーバーのプロパティ] ダイアログ ボックスの [**メンバー** ] タブでは、[**追加**] をクリックします。

6.  **オブジェクトの種類**をクリックして、**コンピューター** ] チェック ボックスを選択し、 **[ok]** をクリックします。

7.  **選択するオブジェクト名を入力する**では、VPN サーバーの名前を入力し、 **[ok]** をクリックします。
