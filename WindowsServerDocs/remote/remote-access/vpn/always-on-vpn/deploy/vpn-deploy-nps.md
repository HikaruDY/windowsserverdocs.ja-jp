# 手順 4.  インストールし、構成のネットワーク ポリシー サーバー (NPS)

>   適用対象: Windows Server 2019、Windows Server (半期チャネル)、Windows Server 2016、Windows Server 2012 R2、Windows 10

この手順では、VPN サーバーによって送信された接続要求の処理のネットワーク ポリシー サーバー (NPS) をインストールします。

- ユーザーが接続するアクセス許可を持っていることを確認するための承認を実行します。
- ユーザーの身元を確認する認証を実行します。
- NPS で RADIUS アカウンティングを構成したときに選択した接続要求の側面をログに記録する計算を実行しています。

このセクションの手順では、次の項目を完了できます。

1.  コンピューターまたはサーバーに NPS サーバーが計画されているし、組織や企業ネットワークにインストールされている VM では、NPS をインストールすることができます。

   >[!TIP] 
   >ネットワークで 1 つまたは複数の NPS サーバーを既にがある場合は、NPS サーバーのインストールを実行する必要はありません - 代わりに、このトピックを使用して既存の NPS サーバーの構成を更新することができます。

>[!NOTE]  
Windows Server Core でネットワーク ポリシー サーバー サービスをインストールできます。

2.  組織や企業の NPS サーバーでは、VPN サーバーから受信した接続要求を処理する RADIUS サーバーとして実行するように NPS を構成できます。

>既定では、NPS RADIUS トラフィックをすべてインストールされているネットワーク アダプターでポート 1812、1813、1645、および 1646 をリッスンします。 NPS をインストールすると、セキュリティが強化された Windows ファイアウォールを有効にする、これらのポートのファイアウォールの例外が IPv4 と IPv6 の両方のトラフィックを自動的に作成を取得します。 NPS のインストール中に、セキュリティが強化された Windows ファイアウォールで作成した例外を削除して、ポートを使用して例外を作成する場合は、ネットワーク アクセスのサーバーは、これらの既定値以外のポート経由で RADIUS トラフィックを送信するように構成、RADIUS トラフィックします。

## NPS を構成します。

すべて認証、承認を処理するように NPS を構成するために NPS をインストールするには、接続の会計職務要求後は、VPN サーバーから受信します。

### Active Directory に NPS サーバーを登録します。

この手順では、接続要求の処理中にユーザー アカウント情報にアクセスする権限があるようにに Active Directory 内のサーバーを登録します。

**手順:**

1.  サーバー マネージャーでは、**ツール**をクリックし、**ネットワーク ポリシー サーバー**をクリックします。 NPS コンソールが開きます。

2.  NPS コンソールで、 **NPS (ローカル)** を右クリックし、 **Active directory サーバーを登録**し、選択] をクリックします。<p>ネットワーク ポリシー サーバーのダイアログ ボックスが開きます。

3.  ネットワーク ポリシー サーバー] ダイアログ ボックスで、 **[ok]** を 2 回クリックします。

NPS を登録するための代替方法は、 [Active Directory ドメインに NPS サーバーを登録](../../../../../networking/technologies/nps/nps-manage-register.md)を参照してください。

### ネットワーク ポリシー サーバー アカウンティングを構成する

この手順では、ネットワーク ポリシー サーバー アカウンティングを使用して次のログ記録の種類のいずれかを構成します。

-   **イベント ログに記録**します。 監査と接続試行のトラブルシューティングに主に使用されます。 NPS のイベント NPS コンソールで NPS サーバーのプロパティを取得することでログを構成できます。

-   **ユーザーの認証とアカウンティング要求ローカル ファイルをログに記録**します。 接続の分析と請求のために主に使用します。 攻撃の後に悪意のあるユーザーのアクティビティを追跡する方法が提供されるため、セキュリティ調査ツールとしても使用します。 アカウンティング構成ウィザードを使用してローカル ファイル ログを構成することができます。

-   **ユーザーの認証とアカウンティング要求準拠している Microsoft SQL Server の XML のデータベースをログに記録**します。 1 つのデータ ソースに NPS を実行している複数のサーバーを許可するために使用します。 また、リレーショナル データベースを使用する利点を提供します。 アカウンティング構成ウィザードを使用して、SQL Server のログを構成できます。

ネットワーク ポリシー サーバー アカウンティングを構成するには、[構成ネットワーク ポリシー サーバー アカウンティング](../../../../../networking/technologies/nps/nps-accounting-configure.md)を参照してください。

## NPS を RADIUS としての VPN 接続構成します。

この手順では、組織のネットワークで RADIUS サーバーとして NPS を構成します。 NPS では、組織や企業ネットワークにアクセスする VPN サーバーの特定のグループのユーザーのみを許可するポリシーを定義する必要があり、PEAP 認証要求で有効なユーザー証明書を使用する場合にのみです。

**手順:**

1.  NPS コンソールで、標準の構成で**ダイヤルアップ接続または VPN 接続用の RADIUS サーバー**が選択されていることを確認します。
