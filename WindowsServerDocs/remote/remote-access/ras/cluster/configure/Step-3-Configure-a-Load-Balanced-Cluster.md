# <a name="step-3-configure-a-load-balanced-cluster"></a>手順 3 が負荷分散クラスターを構成します。

>適用先:Windows Server 2016 の Windows Server (半期チャネル)

サーバー クラスターの準備ができたら、1 台のサーバーの負荷分散を構成、必要な証明書を構成およびクラスターをデプロイします。

|----|--------|

|[3.2 は、負荷分散を有効にします。](#BKMK_NLB)|単一のリモート アクセス サーバーでの負荷分散を有効にします。|  
|[3.3、IP-HTTPS 証明書をインストールします。](#BKMK_InstallIPHTTP)|クラスター内の各サーバーでは、IP-HTTPS 接続の認証にサーバー証明書が必要です。  単一のリモート アクセス サーバーから IP-HTTPS 証明書をエクスポートしは、クラスターに追加する各サーバーに展開します。 これは、機能は、非自己署名証明書を使用する場合にのみ必要です。|

## <a name="BKBK_disable"></a>3.7 が負荷分散を無効にします。

#### <a name="to-disable-load-balancing"></a>負荷分散を無効にするには  
  
1.  構成された DirectAccess サーバーで次のようにクリックします。**開始**、 をクリックし、**リモート アクセス管理**します。 **[ユーザー アカウント制御]** ダイアログ ボックスが表示されたら、表示された操作が正しいことを確認し、 **[はい]** をクリックします。  
  
2.  リモート アクセス管理コンソールで、 **[構成]** をクリックします。 **タスク**ウィンドウで、**負荷分散クラスター**、 をクリックして**負荷分散を無効にする**します。  
  
3.  **負荷分散を無効にする**ダイアログ ボックスで、をクリックして**Ok**。  
  
4.  **負荷分散を無効にする**ダイアログ ボックスで、をクリックして**閉じる**します。  
  
![Windows PowerShell](../../../../media/Step-3-Configure-a-Load-Balanced-Cluster/PowerShellLogoSmall.gif)***<em>Windows PowerShell の同等のコマンド</em>***  
  
以下の Windows PowerShell コマンドレットは、前述の手順と同じ機能を実行します。 ここでは書式上の制約のために、折り返されて複数の行にわたって表示される場合もありますが、各コマンドレットは 1 行に入力します。  
  
```  
set-RemoteAccessLoadBalancer -disable  
```  
  
負荷分散を無効にしてから削除されますリモート アクセスの設定と NLB の設定 (構成されている) 場合実行元のサーバーを除くすべてのサーバー。 このリモート アクセス サーバーでは、(が構成されている) 場合、NLB の設定は削除されますが、リモート アクセスの設定は残ります。  
  
クリックすると**構成設定を削除**はリモート アクセスと削除する NLB (構成されている) 場合、配置内のすべてのサーバーからです。  
  
> [!NOTE]  
> -   負荷分散の展開時にリモート アクセスがアンインストールされると、すべてのサーバーは、Dip に残されます。 Vip が削除されます。 これにより失敗する Vip アドレスを対象にした、企業ネットワーク内のすべてのルートです。 これは、ネットワーク ロケーション サーバー証明書のサブジェクト名などの Vip に解決された DNS エントリも影響します。 この問題を回避するには、最後のリモート アクセス サーバーで、Vip から抜けるを負荷分散、無効にして、リモート アクセスをアンインストールします。  
> -   使用した後、**セット RemoteAccessLoadBalancer**コマンドレットを負荷分散の無効化、他のコマンドレットを実行する前に 2 分間待機します。 これは後に別のコマンドレットを実行しているすべてのスクリプトでも行う必要があります、**セット RemoteAccessLoadBalancer-を無効にする**コマンドレット。  
> -   専用 IP アドレスに、クラスターの仮想 IP アドレスの負荷分散の変更を無効にします。 その結果、サーバーでキャッシュされた DNS エントリの有効期限が切れるまで、サーバーの名前をクエリするすべての操作は失敗します。 負荷分散、サーバー上のキャッシュが期限切れになるまで無効にした後、リモート アクセス PowerShell コマンドレットを実行しないことを確認します。 この問題は、別のドメイン内にある別のコンピューターからのマシンでの負荷分散を無効にしようとする場合より一般的なです。 これは、リモート アクセス管理コンソールからの負荷分散を無効にし、構成の読み込みができない可能性がある場合にも発生します。 キャッシュの有効期限が切れたまたはフラッシュされた後、構成が読み込まれます。
