# <a name="step-3-configure-the-multisite-deployment"></a>手順 3、マルチサイト展開を構成します。

>適用先:Windows Server 2016 の Windows Server (半期チャネル)

マルチサイトのインフラストラクチャを構成した後、リモート アクセスのマルチサイト展開を設定する次の手順に従います。  
  
|タスク|説明|  
|----|--------|  
|3.1. リモート アクセス サーバーを構成します。|IP アドレスを設定をドメインに参加し、リモート アクセスの役割をインストールするには、追加のリモート アクセス サーバーを構成します。|  
|3.2. 管理者のアクセスを許可|DirectAccess 管理者に追加のリモート アクセス サーバーに特権を付与します。|  
|3.3. マルチサイト展開の IP-HTTPS を構成します。|マルチサイト展開で使用される、IP-HTTPS 証明書を構成します。|  
|3.4. マルチサイト展開のネットワーク ロケーション サーバーを構成します。|マルチサイト展開で使用されるネットワーク ロケーション サーバー証明書を構成します。|

|3.6. マルチサイト展開を有効にします。|最初のリモート アクセス サーバーでマルチサイト展開を有効にします。|  
|3.7. エントリ ポイントをマルチサイト展開に追加します。|他のエントリ ポイントをマルチサイト展開に追加します。|

## <a name="BKMK_Client"></a>3.5. マルチサイト展開の DirectAccess クライアントを構成します。  
DirectAccess の Windows クライアント コンピューターは、DirectAccess の関連付けを定義するセキュリティ グループのメンバーである必要があります。 マルチサイトを有効にする前にこれらのセキュリティ グループは必要 (適切な「ダウンレベル」モードを選択した) と Windows 8 クライアントおよび Windows 7 クライアントの両方と格納できます。 マルチサイトが有効にすると 1 台のサーバー モードで、既存のクライアント セキュリティ グループに変換されますセキュリティ グループの Windows 8 のみ。 マルチサイトを有効にした後は、対応する専用 Windows 7 クライアント セキュリティ グループ (これは、特定のエントリ ポイントが関連付けられている)、Windows 7 の DirectAccess クライアント コンピューターを移動する必要があります。 または DirectAccess 経由で接続できません。 Windows 7 クライアントは、Windows 8 のセキュリティ グループになっている既存のセキュリティ グループから先に削除する必要があります。 注意:Windows 7 および Windows 8 の両方のクライアント セキュリティ グループのメンバーである Windows 7 クライアント コンピューターには、リモートの接続が失われ、SP1 がインストールされていない Windows 7 クライアントは企業接続も失われます。 そのため、すべての Windows 7 クライアント コンピューターは、Windows 8 のセキュリティ グループから削除する必要があります。  
  
#### <a name="remove--windows-7--clients-from-windows-8-security-groups"></a>Windows 7 クライアントを Windows 8 のセキュリティ グループから削除します。  
  
1.  プライマリ ドメイン コント ローラーで、次のようにクリックします。 **開始**, 、クリックして **Active Directory ユーザーとコンピューター**します。  
  
2.  セキュリティ グループからコンピュータを削除するセキュリティ グループをダブルクリックし、 **< Group_Name > プロパティ**ダイアログ ボックスで、をクリックして、**メンバー**タブ。  
  
3.  Windows 7 クライアント コンピューターを選択し、クリックして**削除**します。  
  
4.  Windows 7 クライアント コンピューターを Windows 8 のセキュリティ グループから削除するには、この手順を繰り返します。  
  
> [!IMPORTANT]  
> リモート アクセスのマルチサイト構成を有効にする場合 (Windows 7 および Windows 8) のすべてのクライアント コンピューターで、企業ネットワークに直接または VPN は、グループ ポリシーを更新して接続することができるまでにリモート接続が失われます。 これは、最初に、マルチサイトの機能を有効にする場合は true ともマルチサイト無効にするとします。  
  
## <a name="BKMK_Enable"></a>3.6. マルチサイト展開を有効にします。  
マルチサイト展開を構成するには、既存のリモート アクセス サーバー上のマルチサイトの機能を有効にします。 展開でマルチサイトを有効にする前に、次の情報を確認してください。  
  
1.  グローバル ロード バランサーの設定と IP アドレスをロードする場合は、配置内にすべてのエントリ ポイント間で DirectAccess クライアント接続を分散します。  
  
2.  セキュリティでは、Windows 7 のリモート アクセス クライアント コンピューターを有効にする場合、配置内の最初のエントリ ポイントの Windows 7 クライアント コンピューターが含まれるがグループします。  
  
3.  Windows 7 クライアント コンピューターのサポートが必要な場合、配置内の最初のエントリ ポイントの Windows 7 クライアント コンピューターに適用される既定ではないグループ ポリシー オブジェクトを使用する必要がある場合は、ポリシー オブジェクトの名前をグループ化します。

2.  リモート アクセス管理コンソールで次のようにクリックします。**構成**、し、**タスク**ウィンドウで、[] をクリック**マルチサイトを有効にする**します。
