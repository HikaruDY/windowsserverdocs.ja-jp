# <a name="get-started-with-remote-desktop-on-windows"></a>Windows 上のリモート デスクトップを概要します。

>適用対象:Windows 10、Windows 8.1、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2

Windows 用のリモート デスクトップ クライアントを使用して、Windows アプリと別の Windows デバイスからリモート デスクトップを使用することができます。

開始するには、次の情報を使用します。 チェック アウトしてください、 [FAQ](remote-desktop-client-faq.md)ご質問がある場合。

> [!NOTE]
> - Windows クライアントの新しいリリースについて興味があるでしょうか。 チェック アウト[Windows 上のリモート デスクトップの新機能についてはしますか?](windows-whatsnew.md)
> - Windows 10 の任意のバージョンでは、クライアントを実行できます。

## <a name="get-the-rd-client-and-start-using-it"></a>RD クライアントを取得し、その使用を開始

Windows 10 デバイスでリモート デスクトップを開始する次の手順に従います。

1. リモート デスクトップ クライアントからのダウンロード[Microsoft Store](https://www.microsoft.com/store/p/microsoft-remote-desktop/9wzdncrfj3ps)します。 
2. [リモート接続を受け入れるように、PC 設定](remote-desktop-allow-access.md)します。
3. リモート デスクトップ接続またはリモート リソースを追加します。 RemoteApp プログラム、セッション ベースのデスクトップ、または管理者によって公開された仮想デスクトップを使用するには、Windows PC とリモート リソースに直接接続する接続を使用します。 
4. リモート デスクトップをすばやく取得できるように、項目をピン留めします。

### <a name="add-a-remote-desktop-connection"></a>リモート デスクトップ接続を追加します。

リモート デスクトップ接続を作成するには。

1. 接続センター tap で **+ 追加**、 をタップし、**デスクトップ**します。
2. 接続するコンピューターの次の情報を入力します。
   - **PC の名前** – コンピューターの名前。 Windows コンピューター名、インターネット ドメイン名、または IP アドレスを指定できます。 PC の名前をポート情報を追加することもできます (たとえば、 **MyDesktop:3389** または **10.0.0.1:3389**)。
   - **ユーザー アカウント**– リモート PC へのアクセスに使用するユーザー アカウント。 タップ **+** を新しいアカウントを追加または既存のアカウントを選択します。 次の形式を使用するには、ユーザー名: *user_name*、 *domain \user_name*、または <em>user_name@domain.com</em>します。 選択して、接続中にユーザー名とパスワードを要求するかどうかを指定することもできます**確認を求める毎回**します。
3. タップして、追加のオプションを設定することも**詳細を表示する**:
   - **表示名**–、覚えやすい名前に接続している PC にします。 任意の文字列を使用できますが、わかりやすい名前を指定しないと、PC の名前が表示されます。
   - **グループ**– 接続を後で探しやすくグループを指定します。 新しいグループを追加するにはタップして **+** 一覧からいずれかを選択します。
   - **ゲートウェイ** – 仮想デスクトップ、RemoteApp プログラム、および社内ネットワーク上のセッション ベースのデスクトップへの接続に使用する場合、リモート デスクトップ ゲートウェイです。 システム管理者から、ゲートウェイに関する情報を取得します。
   - **管理セッションに接続**-このオプションを使用して、Windows server を管理するコンソール セッションに接続します。
   - **マウス ボタンの入れ替え** – このオプションを使用して、マウスの右ボタンをマウスの左ボタンの機能を交換します。 (これは特に便利ですが、リモート PC では、左手座標系のユーザーが構成されて、右手座標系のマウスを使用する場合)。
   - **リモート セッションの解像度を設定:** – セッションで使用する解像度を選択します。 **私にとって選択**クライアントのサイズに基づいた解決を設定します。
   - **ディスプレイ画面のサイズを変更する:** – セッションの静的な高解像度を選択すると、読みやすさを向上させるために拡大表示画面上の項目を作成するオプションがあります。 注:これは、のみ、Windows 8.1 以降に接続するときに適用されます。
   - **サイズ変更時のリモート セッションの解像度を更新**– 有効にすると、クライアントが動的に、クライアントのサイズに基づいてセッションの解像度を更新します。 注:これは、のみ、Windows 8.1 以降に接続するときに適用されます。
   - **クリップボード**– 有効にすると、テキストのコピーとイメージと、リモート PC からできます。
   - **オーディオ再生**– リモート セッション中にオーディオを使用するデバイスを選択します。 ローカルのデバイスでは、リモートの PC も、まったくサウンドを再生することができます。
   - **オーディオ録音**– 有効にすると、リモート PC にアプリケーションをローカルのマイクを使用することができます。
4. タップ **保存**します。

これらの設定を編集する必要がありますか。 オーバーフロー メニューをタップ ( **.** ) デスクトップ、および順にタップの名前の横にある**編集**します。

接続を削除しますか。 ここでも、オーバーフロー メニューをタップ ( **.** )、し、タップ**削除**します。

### <a name="add-a-remote-resource"></a>リモート リソースを追加します。
リモート リソースには、RemoteApp プログラム、セッション ベースのデスクトップおよびリモート デスクトップ サービスを使用して管理者によって公開された仮想デスクトップが。

リモート リソースに追加します。

1. 接続センター画面をタップ **+ 追加**、 をタップし、**リモート リソース**します。 
2. 入力、**フィード URL**タップして、管理者によって提供される**フィードを検索する**します。
3. 入力を求められたら、フィードを購読するを使用する資格情報を提供します。

リモート リソースが、接続センターで表示されます。


リモート リソースを削除します。

1. 接続センターで、オーバーフロー メニューをタップ ( **...** )、リモート リソースの横にあります。
2. タップ **削除**します。

### <a name="pin-a-saved-desktop-to-your-start-menu"></a>[スタート] メニューに保存されたデスクトップをピン留め

[スタート] メニューへの接続をピン留めするには、オーバーフロー メニューをタップ ( **.** ) デスクトップ、および順にタップの名前の横にある**スタートにピン留め**します。

これで、[スタート] メニューから直接、リモート デスクトップ接続を開始するにはタップすることによりします。

## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>内部の資産へのアクセス、RD ゲートウェイに接続します。

リモート デスクトップ ゲートウェイ (RD ゲートウェイ) では、どこからでも企業ネットワーク上のリモート コンピューターに接続できます。 インターネット上です。 作成および、リモート デスクトップ クライアントを使用して、ゲートウェイを管理できます。

新しいゲートウェイを設定します。

1. 接続センターでタップ**設定**します。
2. ゲートウェイの横にあるタップ **+** 新しいゲートウェイに追加します。 注:新しい接続を追加するときに、ゲートウェイを追加することもできます。
3. 次の情報を入力します。
   - **サーバー名** – ゲートウェイとして使用するコンピューターの名前。 Windows コンピューター名、インターネット ドメイン名、または IP アドレスを指定できます。 サーバー名にポート情報を追加することもできます (例。**RDGateway:443**または**10.0.0.1:443**)。
   - **ユーザー アカウント**- 選択するかに接続しているリモート デスクトップ ゲートウェイで使用するユーザー アカウントを追加します。 選択することも **デスクトップのユーザー アカウントを使用して** 、リモート デスクトップ接続に使用されるものと同じ資格情報を使用します。
4. タップ **保存**します。

- **キーボード コマンドを使用して:** -キーボード コマンドなどの場所を指定できます*WIN*または*ALT + TAB*使用されます。 既定では、セッション、接続が全画面の場合にのみ送信します。

### <a name="manage-your-user-accounts"></a>ユーザー アカウントを管理します。

デスクトップまたはリモート リソースに接続する場合からもう一度選択するためにユーザー アカウントを保存できます。 デスクトップに接続するときに、ユーザー データを保存するのではなく、クライアントでユーザー アカウントを定義することもできます。

新しいユーザー アカウントを作成します。

1. 接続センターでタップ**設定**します。
2. ユーザー アカウントの横にあるタップ **+** 新しいユーザー アカウントを追加します。
3. 次の情報を入力します。
   - **ユーザー名**-リモート接続で使用するために保存するユーザーの名前。 次の形式のいずれかでユーザー名を入力することができます: user_name の domain \user_name、またはuser_name@domain.comします。
   - **パスワード** -指定したユーザーのパスワード。 これを空白のままの接続中にパスワードを求められます。
4. タップ **保存**します。

ユーザー アカウントを削除します。

1. 接続センターでタップ**設定**します。
2. ユーザー アカウントの下の一覧から削除するアカウントを選択します。
3. ユーザー アカウントの横にある編集アイコンをタップします。
4. タップ**このアカウントを削除**下部には、ユーザー アカウントを削除します。
5. ユーザー アカウントを編集してタップできますも、**保存**します。
