## 接続、ゲートウェイ、およびネットワーク

### 接続できないリモート デスクトップを使用するかどうか。

リモート PC に接続しようとしたときに発生する可能性のある一般的な問題に対する解決策を次に示します。 これらのソリューションが機能しない場合は、 [Microsoft コミュニティの web サイト](https://go.microsoft.com/fwlink/p/?LinkId=242079)に詳細なヘルプを検索できます。

- **リモート PC が見つかりません。** 確認し、PC に適切な名前があるかを確認し、その名前を正しく入力したかどうかはします。 まだ接続できない場合は、PC の名前ではなく、リモート PC の IP アドレスを使用してみてください。
- **ネットワークに問題があります。** インターネット接続があることを確認します。 
- **リモート デスクトップのポートは、ファイアウォールによってブロックされる可能性があります。** Windows ファイアウォールを使用している場合は、次の手順に従います。

   1. Windows ファイアウォールを開きます。 
   2. **アプリを許可するか Windows ファイアウォールを介した機能**をクリックします。 
   3. **設定の変更]** をクリックします。 管理者パスワードまたは選択内容を確認するよう求められます可能性があります。
   4. [**許可されているアプリと機能**、**リモート デスクトップ**] を選択し] をタップし、または **[ok]** をクリックします。

   別のファイアウォールを使用している場合は、リモート デスクトップ (通常は 3389) のポートが開いていることを確認してください。
- **リモート PC をリモート接続を設定しない可能性があります。** これを修正するには、バックアップ スクロール[リモート デスクトップの設定、PC をどのようにかどうか。](#how-do-i-set-up-a-pc-for-remote-desktop)このトピックで質問します。

### 会社のネットワークは、どの接続方法がサポートされているかどうか。
企業ネットワーク外部から office デスクトップにアクセスする場合は、会社がリモート アクセスの手段に提供する必要があります。 現在、RD クライアントには、次のものがサポートしています。

- ゲートウェイ ターミナル サーバーまたはリモート デスクトップ ゲートウェイ
- リモート デスクトップ Web アクセス
- VPN (を通じて iOS 組み込みの VPN オプション)

### VPN は機能しません

VPN の問題は、いくつかの原因を持つことができます。 最初の手順では、PC または Mac コンピューターと同じネットワークに VPN が動作することを確認します。 PC または Mac をテストすることはできない場合、デバイスのブラウザーで会社のイントラネット web ページにアクセスしようとすることができます。

その他の点を確認するには:
- **3 G ネットワークは、ブロックまたは VPN が破損します。** ブロックまたは 3 G へのトラフィックを破損するいると思われるユーザーの世界でいくつかの 3 G プロバイダーがあります。 VPN 接続が 1 分以上のと正しく動作することを確認します。
- **L2TP または PPTP Vpn します。** L2TP または PPTP、vpn を使用している場合を設定してくださいすべてのトラフィックを送信 on VPN の構成にします。
- **VPN が正しく構成されています。** 正しく構成されていない VPN サーバーは、理由は、VPN 接続しない作業または理由しばらく作業を停止できます。 このような場合、同じネットワーク上の iOS デバイスの web ブラウザーまたは PC、Mac でのテストを確認します。

### VPN が正しく動作している場合をテストする方法は?
デバイスで VPN が有効になっていることを確認します。 VPN 接続をテストするには、内部ネットワーク上の web ページに移動では、VPN 経由で利用可能な web サービスを使用します。

### L2TP や PPTP VPN 接続を構成する方法は?
L2TP または PPTP、vpn を使用している場合に設定する**すべてのトラフィックを送信** **ON** VPN の構成で確認します。

## 特定のエラー

### 「十分な特権」エラーを取得する理由
接続するセッションにアクセスすることはできません。 最も一般的な原因は、管理者のセッションに接続しようとしていることです。 コンソールへの接続には、管理者のみが許可されています。 コンソール スイッチが無効である、リモート デスクトップの高度な設定を確認します。 問題の原因でない場合に問い合わせて、システム管理者に問い合わせてください。

### 理由は、クライアントと CAL がないことですか。
リモート デスクトップ クライアントは、リモート デスクトップのサーバーに接続するとき、サーバーは、リモート デスクトップ サービス クライアント アクセス ライセンス (RDS CAL) クライアントによって保存されたを発行します。 クライアントが接続するたびにその RDS CAL を使用してもう一度と、サーバーは別のライセンスを発行していません。 デバイス上の RDS CAL が不足しているか、破損している場合、サーバーは別のライセンスを発行します。 ライセンスを持つデバイスの最大数に達すると、サーバーは新しい RDS Cal を発行していません。 ネットワーク管理者にお問い合わせください。

### 「アクセス拒否」エラーを取得する理由だったかどうか。
「アクセス拒否」エラーは、接続の試行中にリモート デスクトップ ゲートウェイと正しくない資格情報の結果によって生成されたです。 ユーザー名とパスワードを確認します。 接続が動作する前に、最近エラーが発生した場合は、する可能性があります Windows ユーザー アカウントのパスワードを変更し、まだ更新して、リモート デスクトップの設定でします。

### 「RPC エラー 23014」または「エラー 0x59e6」意味は何ですか。
**RPC エラー 23014**または**エラー 0x59E6 が、数分間の待機した後でもう一度試して**、発生 RD ゲートウェイ サーバーはアクティブな接続の最大数に到達します。 バージョンによっては、Windows RD ゲートウェイ接続の最大数を実行するいるとは異なる:、Windows Server 2008 R2 Standard の実装が 250 への接続の数を制限します。 Windows Server 2008 R2 Foundation の実装では、50 への接続の数を制限します。 その他のすべての Windows 実装では、接続の数に制限できるようにします。

### 「NTLM チャレンジを解析できませんでした」エラーは何を意味しますか。
このエラーは、リモート PC に誤ったによって発生します。 必ず、RDP セキュリティ レベル リモート PC に設定が「クライアント互換」。 (説明、システム管理者にヘルプがこれを行う必要がある場合です。)

### 「TS_RAP ことはできません、特定のホストに接続する」どういう意味ですか?
このエラーは、ゲートウェイ サーバー上のリソース承認ポリシーは、リモート PC に接続できないユーザー名を停止したときに発生します。 これは、次のインスタンスで発生します。

- リモート PC の名前は、ゲートウェイの名前と同じです。 次に、リモート PC に接続しようとすると、接続は、ゲートウェイを代わりへのアクセス許可がない可能性がありますがします。 ゲートウェイに接続する必要がある場合は、PC の名前として外部ゲートウェイの名前を使わないでください。 代わりに"localhost"または IP アドレス (127.0.0.1) または内部サーバー名を使用します。
- 自分のユーザー アカウントでは、リモート アクセスのユーザー グループのメンバーはありません。
