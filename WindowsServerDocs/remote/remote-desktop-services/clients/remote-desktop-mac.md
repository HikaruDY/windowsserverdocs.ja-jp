## <a name="add-a-remote-desktop-connection"></a>リモート デスクトップ接続を追加します。
リモート デスクトップ接続を作成するには

1. 接続センターで次のようにクリックします。 **+**、 をクリックし、**デスクトップ**します。
2. 次の情報を入力します。
   - **PC の名前**-コンピューターの名前。
      - Windows コンピューターの名前を指定できます (で見つかった、**システム**設定)、ドメイン名または IP アドレス。
      - このようなも、この名前の末尾にこのポート情報を追加することができます*MyDesktop:3389*します。
   - **ユーザー アカウント**-リモート PC へのアクセスに使用するユーザー アカウントを追加します。
      - Active Directory (AD) に参加するコンピューターまたはローカル アカウント、これらの形式のいずれかを使用: *user_name*、 *domain \user_name*、または *user_name@domain.com*します。
      - Azure Active Directory (AAD) のでは、これらの形式のいずれかのコンピューターが参加しています。*AzureAD\user_name*または *AzureAD\user_name@domain.com*します。
      - パスワードを要求するかどうかを選択することもできます。
      - 同じユーザー名を持つ複数のユーザー アカウントを管理する場合は、アカウントを区別するためにフレンドリ名を設定します。
      - アプリの設定に保存されているユーザー アカウントを管理します。 

3. 接続のこれらのオプションの設定を設定することもできます。
   - フレンドリ名を設定します。 
   - ゲートウェイを追加します。
   - サウンド出力を設定します。
   - マウス ボタンを入れ替え
   - 管理者モードを有効にします。
   - リモート セッションにローカル フォルダーをリダイレクトします。
   - 前方のローカル プリンター
   - 前方のスマート カード
4. **[保存]** をクリックします。

接続を開始するをダブルクリックします。 リモート リソースと同じです。

## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>内部の資産へのアクセス、RD ゲートウェイに接続します。

リモート デスクトップ ゲートウェイ (RD ゲートウェイ) では、どこからでも企業ネットワーク上のリモート コンピューターに接続できます。 インターネット上です。 作成し、アプリまたはデスクトップの新しい接続を設定中に環境設定で、ゲートウェイを管理します。

環境設定で新しいゲートウェイを設定します。

1. 接続センターで次のようにクリックします。**設定 > ゲートウェイ**します。 
2. をクリックして、 **+** 次の情報」と入力テーブルの下部にあるボタンをクリックします。
  - **サーバー名** – ゲートウェイとして使用するコンピューターの名前。 Windows コンピューター名、インターネット ドメイン名、または IP アドレスを指定できます。 サーバー名にポート情報を追加することもできます (例。**RDGateway:443**または**10.0.0.1:443**)。
  - **ユーザー名** -ユーザー名とは、接続先のリモート デスクトップ ゲートウェイに使用するパスワード。 選択することも **接続の資格情報を使用して** 、リモート デスクトップ接続に使用されるものと同じユーザー名とパスワードを使用します。


## <a name="manage-your-user-accounts"></a>ユーザー アカウントを管理します。

新しいユーザー アカウントを作成します。

1. 接続センターで次のようにクリックします。**設定** > **アカウント**します。
2. クリックして**ユーザー アカウントを追加**します。
3. 次の情報を入力します。
   - **ユーザー名** のリモート接続で使用するために保存するユーザーの名前。 次の形式のいずれかでユーザー名を入力することができます: user_name の domain \user_name、またはuser_name@domain.comします。
   - **パスワード** -指定したユーザーのパスワード。 リモート接続に使用する保存するすべてのユーザー アカウントは、関連付けられているパスワードが必要です。
   - **フレンドリ名**- 別のパスワードを持つ同じユーザー アカウントを使用している場合は、それらのユーザー アカウントを区別するためのフレンドリ名を設定します。

## <a name="use-a-keyboard-in-a-remote-session"></a>リモート セッションでキーボードを使用します。

Mac のキーボード レイアウトは、Windows キーボードのレイアウトによって異なります。 

- Mac のキーボード上のコマンド キーでは、Windows キーと同じです。
- Mac で、ボタンを使用する操作を実行する Windows のコントロール ボタンを使用する必要があります (例。コピー = ctrl + c) します。
- ファンクション キーは、セッションで FN キーを押してさらに、アクティブにすることができます (例。FN ながら F1 キー)。
- Mac キーボードのスペース バーの右側に Alt キーでは、Windows の Alt Gr/右 Alt キーと同じです。

既定では、リモート セッションは、クライアントで実行する OS として、同じキーボード ロケールを使用します。 (Mac で、en を実行している場合、OS では、リモート セッションにも使用されること。 OS キーボード ロケールを使用しない場合は、リモート PC に設定し、設定を変更するキーボードを確認してください。 手動でします。 参照してください、[リモート デスクトップ クライアントに関する FAQ](remote-desktop-client-faq.md)キーボードおよびロケールの詳細についてはします。
