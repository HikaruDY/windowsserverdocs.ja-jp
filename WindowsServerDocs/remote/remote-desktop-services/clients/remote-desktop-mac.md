# Mac でリモート デスクトップを使ってみる

## リモート デスクトップ接続を追加します。
リモート デスクトップ接続を作成します。

1. 接続センターで、クリックして**+**、**デスクトップ**をクリックします。
2. 次の情報を入力します。
   - **PC 名**- コンピューターの名前です。
      - (**システム**設定にあります)、Windows コンピューター名、ドメイン名または IP アドレスを指定できます。
      - *MyDesktop:3389*と同様に、この名前の末尾にポート情報を追加することもできます。
   - **ユーザー アカウント**のリモート PC へのアクセスに使用するユーザー アカウントを追加します。
      - コンピューターまたはローカル アカウントが Active Directory (AD) に参加しているはこれらの形式のいずれかを使用:*ユーザー名*、 *domain \user_name*、または*user_name@domain.com*します。
      - Azure Active Directory (AAD) にコンピューターが参加しているはこれらの形式のいずれかを使って: *AzureAD\user_name*または*AzureAD\ user_name@domain.com*します。
      - パスワードを要求するかどうかを選択することもできます。
      - 同じユーザー名を持つ複数のユーザー アカウントを管理するときは、アカウントを区別するためにフレンドリ名を設定します。
      - アプリの基本設定の保存したユーザー アカウントを管理します。 

3. 接続のため、これらのオプション設定を設定することもできます。
   - フレンドリ名を設定します。 
   - ゲートウェイを追加します。
   - サウンドの出力を設定します。
   - スワップ マウス ボタン
   - 管理者モードを有効にします。
   - リモート セッションへのローカル フォルダーをリダイレクトします。
   - 前方ローカル プリンター
   - 転送のスマート カード
4. **[保存]** をクリックします。

開始するには、接続をダブルクリックします。 リモートのリソースの場合も同様です。

## リモート リソースを追加します。
リモートのリソースは、RemoteApp プログラム、セッション ベースのデスクトップおよび RemoteApp とデスクトップ接続を使用して公開された仮想デスクトップです。

- URL には、RemoteApp とデスクトップ接続へのアクセスを提供する RD Web アクセス サーバーへのリンクが表示されます。
- 構成済みの RemoteApp とデスクトップ接続の一覧が表示されます。

リモートのリソースを追加します。

1. 接続センターのクリック**+**、**追加のリモート リソース**をクリックします。 
2. リモートのリソースの情報を入力します。
   - **フィードの URL**の RD Web アクセス サーバーの URL。 クライアントがメール アドレスに関連付けられている RD Web アクセス サーバーを検索するように指示 – このフィールドに、企業のメール アカウントを入力することもできます。
   - **ユーザー名**- に接続している RD Web アクセス サーバーを使用するユーザー名です。
   - **パスワード**の RD Web アクセス サーバーに接続するために使用するパスワードです。
3. **[保存]** をクリックします。


リモートのリソースが接続センターで表示されます。


## 内部資産へのアクセスの RD ゲートウェイへの接続します。

リモート デスクトップ ゲートウェイ (RD ゲートウェイ) では、どこからでも、企業ネットワーク上のリモート コンピューターに接続できます。 インターネット上します。 作成して、または新しいデスクトップ接続の設定中に、アプリの基本設定で、ゲートウェイを管理することができます。

環境設定で新しいゲートウェイを設定します。

1. 接続センターで、クリックして**の基本設定 > ゲートウェイ**します。 
2. クリックして、 **+** Enter、次の情報の表の下部にあるボタン。
  - [**サーバー名**]: ゲートウェイとして使用するコンピューターの名前。 Windows コンピューター名、インターネット ドメイン名、または IP アドレスを指定できます。 サーバー名にポート情報を追加することもできます (例: **RDGateway:443**または**10.0.0.1:443**)。
  - **ユーザー名**- ユーザー名と、リモート デスクトップ ゲートウェイに接続するために使用するパスワードです。 リモート デスクトップ接続用に使用されるものと同じユーザー名とパスワードを使用する**接続の資格情報の使用**を選択することもできます。


## ユーザー アカウントを管理します。

デスクトップまたはリモートのリソースに接続するときは、もう一度から選択するユーザー アカウントを保存できます。 リモート デスクトップ クライアントを使用して、ユーザー アカウントを管理できます。

新しいユーザー アカウントを作成します。

1. 接続センターで、[**設定**] をクリックして > **アカウント**。
2. **ユーザー アカウントの追加**] をクリックします。
3. 次の情報を入力します。
   - **ユーザー名**- 使用したリモート接続を保存するユーザーの名前です。 次の形式で入力したユーザー名: ユーザー名、domain \user_name、または user_name@domain.com します。
   - **パスワード**: 指定したユーザーのパスワード。 すべてのユーザー アカウントを使ってリモート接続用に保存するには、それに関連付けられているパスワードが必要です。
   - **フレンドリ名**- で異なるパスワードは、同じユーザー アカウントを使用している場合は、それらのユーザー アカウントを区別するためにフレンドリ名を設定します。
4. **保存**するには、タップして、**設定**します。

## リモート セッションでキーボードを使用します。

Mac キーボード レイアウトは、Windows のキーボード レイアウトによって異なります。 

- Mac のキーボード コマンド キーでは、Windows キーと同じです。
- Mac でのコマンド ボタンを使用するアクションを実行するには、Windows のコントロール ボタンを使用する必要があります (例:: コピー = Ctrl + C)。
- ファンクション キーは、セッションでさらに、キーを押して FN アクティブにすることができます (例:: FN 押しながら f1 キー)。
- Mac キーボードでスペース バーの右側に、Alt キーでは、Windows では、Alt Gr/右 Alt キーと同じです。

既定では、リモート セッションはでクライアントを実行している OS として同じロケールがキーボードを使用します。 (Mac では、en が実行されている場合の OS では、リモート セッションにも使用されることです。 OS キーボード ロケールが使われていない場合、キーボード、リモート PC に設定し、設定の変更を確認手動でします。 キーボードとロケールの詳細については、[リモート デスクトップ クライアントよく寄せられる質問](remote-desktop-client-faq.md)を参照してください。
