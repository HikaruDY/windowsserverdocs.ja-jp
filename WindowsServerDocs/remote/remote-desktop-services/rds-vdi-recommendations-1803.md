#### <a name="servicing-the-operating-system-and-apps"></a>オペレーティング システムやアプリケーションのサービス

イメージの最適化プロセス中には、ある時点で、使用可能な Windows 更新プログラムを適用する必要があります。 その他の Microsoft 製品の更新プログラムをインストールする Windows 更新プログラムおよび Windows を設定できます。 これを設定するには、開く**Windows 設定**を選択し、**更新とセキュリティ**、し、**詳細オプション**します。 選択**Windows 更新時に他の Microsoft 製品の更新を与える me**に設定する**で**します。

基本イメージに Microsoft Office などの Microsoft アプリケーションをインストールしようとする場合に、適切な設定になります。 これにより Office が、イメージがサービスの配置と最新の状態です。 .NET の更新プログラムおよび Windows Update を通じて利用可能な更新プログラムのある特定のサードパーティ製コンポーネント Adobe などもあります。

### <a name="tuning-windows-10-network-performance-by-using-registry-settings"></a>レジストリ設定を使用して Windows 10 のネットワークのパフォーマンスのチューニング

これは、VDI または物理コンピューターがネットワーク ベースでは主にワークロードがある環境で特に重要です。 設定のバッファリングとモ ノのキャッシュを追加することによって、ネットワークのどちらを優先するこのセクションのバイアス パフォーマンス設定では、ディレクトリのエントリがこれになど。

このセクションでは一部の設定は*レジストリに基づいたのみ*と実稼働環境用、イメージを展開する前に基本イメージに組み込む必要があります。

次の設定が記載されて、 [Windows Server 2016 のパフォーマンス チューニング ガイドライン](https://docs.microsoft.com/windows-server/administration/performance-tuning/)については、Windows 製品グループによって Microsoft.com に公開します。

#### <a name="disablebandwidththrottling"></a>DisableBandwidthThrottling

HKLM\\システム\\CurrentControlSet\\サービス\\LanmanWorkstation\\パラメーター\\DisableBandwidthThrottling

Windows 10 に適用されます。 既定値は **0**です。 既定では、SMB リダイレクターは、ネットワーク関連のタイムアウトを回避するために、待機時間の長いネットワーク接続全体のスループットを調整します。 このレジストリ値を設定**1**ので、この設定を考慮する必要があります、待機時間の長いネットワーク接続経由でファイル転送のより高いスループットを有効にするが、このスロットル無効になります。

#### <a name="fileinfocacheentriesmax"></a>FileInfoCacheEntriesMax

HKLM\\システム\\CurrentControlSet\\サービス\\LanmanWorkstation\\パラメーター\\FileInfoCacheEntriesMax

Windows 10 に適用されます。 既定値は**64**、有効範囲は 1 に 65536 でします。 この値は、クライアントがキャッシュできるファイル メタデータの量を決定するために使用されます。 値を増やすと、トラフィックを削減し、多数のファイルがアクセスされたときにパフォーマンスを向上させることができます。 この値を増やしてください**1024**します。

#### <a name="directorycacheentriesmax"></a>DirectoryCacheEntriesMax

HKLM\\システム\\CurrentControlSet\\サービス\\LanmanWorkstation\\パラメーター\\DirectoryCacheEntriesMax

Windows 10 に適用されます。 既定値は**16**、有効な 1 ~ 4096 の範囲とします。 この値は、クライアントがキャッシュできるディレクトリ情報の量を決定するために使用されます。 この値を増やすと、大規模なディレクトリへのアクセス時にネットワーク トラフィックを削減してパフォーマンスを向上させることができます。 この値を増やすことを検討**1024**します。

#### <a name="filenotfoundcacheentriesmax"></a>FileNotFoundCacheEntriesMax

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\FileNotFoundCacheEntriesMax

Windows 10 に適用されます。 既定値は**128**、有効範囲は 1 に 65536 でします。 この値は、クライアントがキャッシュできるファイル名情報の量を決定するために使用されます。 値を増やすと、トラフィックを削減し、多くのファイル名がアクセスされたときにパフォーマンスを向上させることができます。 この値を増やすことを検討**2048**します。

#### <a name="dormantfilelimit"></a>DormantFileLimit

HKLM\\システム\\CurrentControlSet\\サービス\\LanmanWorkstation\\パラメーター\\DormantFileLimit

Windows 10 に適用されます。 既定値は**1023**します。 このパラメーターでは、アプリケーションがファイルを閉じた後に共有リソース上で開いたままにする必要があるファイルの最大数を指定します。 何千ものクライアントが SMB サーバーに接続するには、この値を減らす検討**256**します。

これらの SMB の設定の多くを使用して構成することができます、[セット SmbClientConfiguration](https://docs.microsoft.com/powershell/module/smbshare/set-smbclientconfiguration)と[セット SmbServerConfiguration](https://docs.microsoft.com/powershell/module/smbshare/set-smbserverconfiguration) Windows PowerShell コマンドレット。 次の例のように、同様に、Windows PowerShell を使用してレジストリ専用の設定を構成できます。

`Set-ItemProperty -Path "HKLM:\\SYSTEM\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters" RequireSecuritySignature -Value 0 -Force`
