# <a name="prestage-cluster-computer-objects-in-active-directory-domain-services"></a>Active Directory ドメイン サービスのクラスター コンピューターのオブジェクトを用意します。

>対象: Windows Server 2012 R2、Windows Server 2012、Windows Server 2016

このトピックでは、Active Directory ドメイン サービス (AD DS) のクラスター コンピューターのオブジェクトを用意する方法を説明します。 AD DS にコンピューターのオブジェクトを作成するのに権限があるない場合は、フェールオーバー クラスターを作成するのにには、ユーザーまたはグループを有効にするのには、この手順を使用できます。

クラスターの作成ウィザードを使用して、または Windows PowerShell を使用してフェールオーバー クラスターを作成するときに、クラスターの名前を指定する必要があります。 クラスターを作成するときに、十分なアクセス許可がある、クラスターの作成プロセスはクラスター名と一致する AD DS にコンピューターのオブジェクトを自動的に作成します。 このオブジェクトは、*クラスター名オブジェクト*または CNO と呼ばれます。 CNO] で仮想コンピューター オブジェクト (Vco) はクライアント アクセス ポイントを使用して集合の役割を構成するに自動的に作成されます。 たとえば、 *FileServer1*というクライアント アクセス ポイントの可用性ファイル サーバーを作成する場合は、CNO が対応する VCO AD DS に作成します。

>[!NOTE]
>Windows Server 2012 R2 には、場所 CNO または Vco は作成されません AD DS に、Active Directory デタッチ クラスターを作成するオプションがします。 これは、特定の種類のクラスター展開向けです。 詳細については、[展開 Active Directory-Detached クラスター](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn265970(v%3dws.11)>)を参照してください。

CNO を自動的に作成するには、フェールオーバー クラスターを作成したユーザーは組織単位 (OU) またはクラスターを構成するサーバーがあるコンテナーに**コンピューターを作成するオブジェクト**の権限が必要です。 この権限がないクラスターを作成するには、ユーザーまたはグループを有効にするには、(通常は、ドメイン管理者) の AD DS に適切な権限を持つユーザーを CNO AD DS に事前登録できます。 これはクラスターに使用される名前付け規則を制御と詳細コントロールにどの OU クラスター オブジェクトがで作成したドメイン管理者も提供します。

## <a name="step-1-prestage-the-cno-in-ad-ds"></a>手順 1: AD DS に CNO を用意します。

作業を開始する前に、次を理解していることを確認します。

- 名前をクラスターを割り当てる
- ユーザー アカウントまたはクラスターを作成する権限を付与するグループの名前
