## <a name="guidance-on-disabling-system-services-on-windows-server-2016-with-desktop-experience"></a>デスクトップ エクスペリエンス搭載の Windows Server 2016 上のシステム サービスを無効にする方法のガイダンス

適用対象: Windows Server 2016

Windows オペレーティング システムには、重要な機能を提供する多くのシステム サービスが含まれています。 さまざまなサービスのさまざまな既定のスタートアップ ポリシーいます: 一部は既定では開始 (自動)、いくつか必要になったとき (手動)、およびいくつか既定で無効になっている必要があります明示的に有効にすることができます、実行する前にします。 これらの既定値は、各サービスの一般的なお客様のパフォーマンス、機能、およびセキュリティのバランスを慎重に選択されました。

ただし、一部の企業のお客様では、Windows Pc と 1 つの攻撃対象領域を最小限が少なくなり、特定の環境で必要のないすべてのサービスを完全に無効にする必要がありますので、サーバーの複数のセキュリティに重点を置いたバランスを取ることもできます。 これらのユーザーの Microsoft® に関するサービスを安全に無効にするこの目的のため、それに付随するガイダンスを提供しています。

ガイドでは (特記のないエンド ユーザーのデスクトップの代替として使用する) にデスクトップ エクスペリエンス搭載の Windows Server 2016 用です。 システム上の各サービスは、次のように分類は。

-   **無効にする必要があります。**セキュリティに重点を置いたエンタープライズを好みますこのサービスを無効にし、その機能をしない可能性があります (詳細については、次を参照してください)。
- **[OK] を無効にする:**このサービスがすべてではなく一部の企業への便利な機能を提供し、それを使用していない企業のセキュリティに重点を置いたが安全にそれを無効にします。
- **無効にしないで:**重要な機能に影響を与えるまたは特定の役割または機能が正しく機能していることを防ぐはこのサービスを無効にするとします。 そのため、無効にすることです。
-  **(ガイダンスなし):**これらのサービスを無効にする場合の影響が完全に評価されていません。 そのため、これらのサービスの既定の構成は変更されませんする必要があります。


お客様は、グループ ポリシーのセキュリティ テンプレートを使用または PowerShell オートメーションを使用して、選択したサービスを無効にするには、Windows Pc とサーバーを構成できます。 場合によってでのガイダンスには、サービス自体を無効にする代わりに、直接サービスの機能を無効にする特定のグループ ポリシー設定が含まれています。

お客様が、次のサービスとデスクトップ エクスペリエンス搭載の Windows Server 2016 では、それぞれのスケジュールされたタスクを無効にすることをお勧めします。

サービス: 
1. Xbox Live 認証マネージャー
2. Xbox Live セーブ データ

スケジュールされたタスクの場合: 
1. \Microsoft\XblGameSave\XblGameSaveTask
2. \Microsoft\XblGameSave\XblGameSaveTaskLogon

(接続されている Microsoft Excel のスプレッドシートを表示して、この記事で詳細なすべてのサービスについての情報をアクセスすることもできます:[デスクトップ エクスペリエンス搭載の Windows Server 2016 上でシステム サービスを無効にするとに関するガイダンス](https://msdnshared.blob.core.windows.net/media/2017/05/Service-management-WS2016.xlsx))

### <a name="disabling-services-not-installed-by-default"></a>既定でインストールされていないサービスを無効にします。

既定でインストールされていないサービスを無効にするポリシーを適用するに対して Microsoft がお勧めします。
-  サービスは通常、この機能がインストールされている場合に必要です。 サービスまたは機能をインストールするには、管理者権限が必要です。 機能のインストール、サービスの開始を許可しません。
-  管理者 (または場合によっては管理者以外) をセキュリティ リスクの高いでおそらくいずれかのようなサード パーティ製同等のインストールから、ブロックして、Microsoft Windows サービスは停止しません。
-  基準またはベンチマークを既定以外の Windows サービス (W3SVC など) を無効にするがいくつかの監査役誤ってな印象を与えるテクノロジ (たとえば、IIS) が本質的にセキュリティで保護されていないを使用しないでください。
-  機能 (とサービス) がインストールされてない場合これだけで不要な一括基準におよび追加の確認作業をします。
