## <a name="overview-of-ad-fs"></a>AD FS の概要  
AD FS がクライアント コンピューターを提供する ID アクセス ソリューション \ (内部または外部のネットワークに) シームレスな SSO アクセスを保護されたのとは限らない向けのアプリケーションまたはサービスは、ユーザー アカウントとアプリケーションが完全に別のネットワークや組織内にある場合でもします。  
  
アプリケーションまたはサービスが、1 つのネットワークと、ユーザー アカウントが別のネットワークでは、通常、よう求められますセカンダリ資格情報のアプリケーションまたはサービスにアクセスしようとするとします。 これらのセカンダリ資格情報は、アプリケーションまたはサービスが存在する領域内のユーザーの ID を表します。 通常、最も適切な承認の判断できるようにするために、アプリケーションまたはサービスをホストする Web サーバーで必要とされます。  
  
AD FS では、組織はこれらの組織はユーザーのデジタル ID とアクセス権を提示信頼できるパートナーに使用できる信頼関係 \(federation trusts\) を提供することによってセカンダリ資格情報の要求をバイパスできます。 このフェデレーション環境で各組織は引き続き、独自の ID を管理が、各組織のプロジェクトし他の組織からの ID を受け入れることができますも安全にします。
