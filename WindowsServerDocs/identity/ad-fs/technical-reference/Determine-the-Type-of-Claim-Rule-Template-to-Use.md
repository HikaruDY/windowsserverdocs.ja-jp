# <a name="determine-the-type-of-claim-rule-template-to-use"></a>使用する要求規則テンプレートの種類を決定します。

>適用対象: Windows Server 2016、Windows Server 2012 R2

Active Directory フェデレーション サービス \(AD FS\) インフラストラクチャの設計の重要な部分には、要求規則の完全なセットが決定することが — 要求規則テンプレートの作成に使用する必要がありますを対応するどのと — 各パートナー組織とのフェデレーションに参加するのです。 規則を作成するには、AD FS 管理スナップインで要求規則テンプレートを使用します。  
  
要求規則を構成するのには、各セットは 1 つのフェデレーションによる信頼に関連付けられたのみできます。 つまり、1 つの信頼で規則のセットを作成および使用して、フェデレーション サービスの他の信頼ことはできません。 代わりに簡単にルールを作成できます要求からすばやくの各フェデレーション パートナーと組織の間で合意された要求の望ましいセットを生成する規則テンプレート。  
  
規則と規則テンプレートの詳細については、次を参照してください。[The Role of Claim Rules](The-Role-of-Claim-Rules.md)します。  
  
使用する必要があります、要求規則テンプレートの種類の決定を開始する前に、次の点を考慮します。  
  
-   どのような要求が、信頼される要求プロバイダーによって提供されますか。  
  
-   各要求プロバイダーからどのような要求を信頼しますか。  
  
-   どのような要求は、このフェデレーション サービスを信頼する証明書利用者が必要ですか。  
  
-   信頼性情報を公開する各証明書利用者とは  
  
-   各証明書利用者へのアクセスを付与するユーザーですか。  
  
これらの質問に答えることは、信頼性の高いを計画するのに役立ちます要求規則の設計します。 スムーズな承認を作成する際しアクセス制御戦略され、ロールアウト時に、展開チームの効率。  
  
このセクションでは、環境のビジネスに基づく選択する規則テンプレートの種類について説明の必要があります。
