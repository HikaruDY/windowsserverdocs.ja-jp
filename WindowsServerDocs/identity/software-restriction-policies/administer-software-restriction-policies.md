# <a name="administer-software-restriction-policies"></a>ソフトウェアの制限のポリシーを管理します。

>適用対象: Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

IT プロフェッショナル向けのこのトピックでは、手順と Windows Server 2008 および Windows Vista 以降のソフトウェア制限ポリシー (SRP) を使用してアプリケーション制御ポリシーを管理する方法を紹介します。

## <a name="introduction"></a>概要
ソフトウェア制限ポリシー (SRP) は、ドメイン内のコンピューターで実行されているソフトウェア プログラムを指定し、これらのプログラムを実行する機能を制御する、グループ ポリシー ベースの機能です。 厳しく制限されているしたアプリケーションに限って実行を許可する、コンピューターの構成を作成するのにには、ソフトウェアの制限のポリシーを使用します。 これらは、Microsoft Active Directory Domain Services とグループ ポリシーに統合されているが、スタンドアロン コンピューター上で構成することもあります。 SRP の詳細については、次を参照してください。、 [Software Restriction Policies](software-restriction-policies.md)します。

Windows Server 2008 R2 および Windows 7 以降、Windows AppLocker を使用できるの代わりに、または SRP と連動してアプリケーション制御戦略の一部を受け取ります。

## <a name="BKMK_Create_SRP"></a>新しいソフトウェアの制限のポリシーを作成するには

2.  **アクション**] メニューのをクリックして**新しいソフトウェアの制限のポリシー**します。

> [!WARNING]
> -   使用環境に応じて、この手順を実行するには、異なる管理者資格情報が必要です。

>     -   ローカル コンピューターでは、新しいソフトウェアの制限のポリシーを作成するかどうか: ローカルの**管理者**はこの手順を完了するために必要な最低限のグループ、またはそれと同等です。

## <a name="BKMK_Prevent_Admin"></a>ソフトウェアの制限のポリシーをローカルの管理者に適用されないようにするには

1.  ソフトウェアの制限のポリシーを開きます。

2.  詳細ウィンドウでダブルクリック**実施**します。

3.  [ **、次のユーザーにソフトウェアの制限のポリシーを適用**、] をクリックして**ローカル管理者を除くすべてのユーザー**します。

> [!WARNING]
> -   ローカルのメンバーシップ**管理者**はこの手順を完了するために必要な最低限のグループ、またはそれと同等です。
> -   されていない場合、新しいソフトウェアの制限のポリシー設定のグループ ポリシー オブジェクト (GPO) を作成する必要がある可能性があります。
> -   通常は、ユーザーは、組織内のコンピューターでローカルの Administrators グループのメンバーである場合は、このオプションを有効にする可能性がありますされません。
> -   ローカル コンピューターのソフトウェアの制限のポリシー設定を定義する場合をローカルの管理者がソフトウェア制限のポリシーを適用するを防ぐためにこの手順を使用します。 ネットワークのソフトウェアの制限のポリシー設定を定義する場合は、グループ ポリシーを使用してセキュリティ グループのメンバーシップに基づくユーザーのポリシー設定をフィルター処理します。

## <a name="BKMK_Sec_Lvl"></a>ソフトウェアの制限のポリシーの既定のセキュリティ レベルを変更するには

1.  ソフトウェアの制限のポリシーを開きます。

2.  詳細ウィンドウでダブルクリック**セキュリティ レベル**します。

3.  クリックして、既定値として設定するセキュリティ レベルを右クリックして**既定として設定**します。

> [!CAUTION]
> 特定のディレクトリに設定する既定のセキュリティ レベルを**許可しない**、オペレーティング システムに悪影響を及ぼすことができます。

> [!NOTE]
> -   異なる管理者資格情報は、ソフトウェアの制限のポリシーの既定のセキュリティ レベルを変更する環境に応じて、この手順を実行する必要があります。
> -   されていない場合、新しいソフトウェアの制限のポリシー設定をこのグループ ポリシー オブジェクト (GPO) を作成する必要がある可能性があります。
> -   詳細ウィンドウで黒丸のチェック マークが付いた現在の既定のセキュリティ レベルが示されます。 現在の既定のセキュリティ レベルを右クリックした場合、**既定として設定**メニューのコマンドは表示されません。
> -   既定のセキュリティ レベルに対する例外を指定するには、ソフトウェアの制限のポリシー規則が作成します。 既定のセキュリティ レベルに設定すると**無制限**、規則が実行を許可されていないソフトウェアを指定できます。 既定のセキュリティ レベルに設定すると**許可しない**、規則が実行を許可するソフトウェアを指定できます。
> -   インストール時に、システム上のすべてのファイルのソフトウェアの制限のポリシーの既定のセキュリティ レベル設定されて**無制限**します。

## <a name="BKMK_Apply_SRP_DLLs"></a>Dll にソフトウェアの制限のポリシーを適用するには

1.  ソフトウェアの制限のポリシーを開きます。

2.  詳細ウィンドウでダブルクリック**実施**します。

3.  [**次にソフトウェアの制限のポリシーを適用**、] をクリックして**すべてのソフトウェア ファイル**します。

> [!NOTE]
> -   この手順を実行するには、ローカルのコンピューターの Administrators グループのメンバーであるか、されている必要がありますが、適切な権限を委任します。 コンピューターがドメインに参加して、Domain Admins グループのメンバーがこの手順を実行できる可能性があります。
> -   既定では、ソフトウェア制限ポリシーではダイナミック リンク ライブラリ (Dll) はチェックされません。 Dll をチェックすると、DLL が読み込まれるたびに、ソフトウェアの制限のポリシーを評価する必要があるためにシステムのパフォーマンスが低下することができます。 ただし、Dll をターゲットにウイルスを受信に関する不安がある場合は、Dll を確認することができます。 既定のセキュリティ レベル設定されている場合**許可しない**、DLL のチェックを各 DLL の実行を許可する規則をポリシーにソフトウェアの制限を作成する必要がありますを有効にするとします。
