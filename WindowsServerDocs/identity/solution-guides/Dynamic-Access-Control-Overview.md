# <a name="dynamic-access-control-overview"></a>ダイナミック アクセス制御の概要

>適用対象: Windows Server 2012 R2、Windows Server 2012

IT プロフェッショナル向けのこの概要トピックではダイナミック アクセス制御について説明し、それに関連する要素を Windows Server 2012 および Windows 8 で導入されたします。  
  
ドメイン ベースのダイナミック アクセス制御では、アクセス制御のアクセス許可と、リソース、ジョブまたは、ユーザーの役割とこれらのリソースにアクセスするために使用するデバイスの構成の感度を含めることが明確に定義された規則に基づいて制限を適用できます。  
  
たとえば、ユーザーは、仮想プライベート ネットワークを介してポータブル コンピューターを使用しているときとオフィスのコンピューターからリソースにアクセスするときに、異なるアクセス許可があります。 または、デバイス場合にのみが、ネットワーク管理者によって定義されているセキュリティ要件を満たしているへのアクセスを許可することがあります。 ダイナミック アクセス制御を使用すると、ユーザーのアクセス許可変更動的に追加の管理者の介入なし場合は、ユーザーの業務または役割 (AD DS でユーザーのアカウント属性の変更の結果として) を変更できます。  
  
ダイナミック アクセス制御は Windows Server 2012 および Windows 8 より前の Windows オペレーティング システムでサポートされていません。 ダイナミック アクセス制御がサポートされており、サポートされていないバージョンの Windows 環境で構成されると、サポートされているバージョンのみが変更を実装します。

### <a name="BKMK_Rules"></a>集約型アクセス規則  
集約型アクセス規則は、ユーザー グループ、ユーザーの信頼性情報、デバイスの信頼性情報、およびリソースのプロパティを含む 1 つまたは複数の条件を含めることができる承認規則の式です。 複数の集約型アクセス規則は、集約型アクセス ポリシーに結合することができます。  
  
1 つ以上の集約型アクセス規則がドメインに対して定義されている場合、ファイル共有の管理者は、特定のリソースおよびビジネス要件を特定の規則と一致ことができます。  
  
### <a name="BKMK_Policies"></a>集約型アクセス ポリシー  
集約型アクセス ポリシーは、条件式を含む承認ポリシーです。 たとえば、組織が、ファイルの所有者と人事 (HR) 部門の PII 情報を表示できるはメンバーだけファイルの個人を特定できる情報 (PII) へのアクセスを制限するというビジネス要件とします。 これは、組織全体にわたって、ファイル サーバーに配置されている限り、PII ファイルに適用される組織全体のポリシーを表します。 このポリシーを実装するには、組織ことができる必要があります。  
  
-   識別し、PII を含むファイルをマークします。  
  
-   PII 情報を表示する許可されている HR のメンバーのグループを識別します。  
  
-   集約型アクセス規則に集約型アクセス ポリシーを追加し、組織全体でファイル サーバーに置かれる場所に、PII を含むすべてのファイルに集約型アクセス規則を適用します。  
  
集約型アクセス ポリシーは、組織のサーバーに適用される上位のセキュリティとして機能します。 これらのポリシーに追加されます (は置き換わるものでありません)、ローカルのアクセス ポリシーまたは随意アクセス制御リスト (Dacl) ファイルとフォルダーに適用されています。  
  
### <a name="BKMK_Claims"></a>信頼性情報  
要求は、ユーザー、デバイス、またはドメイン コントローラーによって公開されているリソースに関する情報の一部です。 ユーザーの役職、ファイル、またはコンピューターのヘルス状態の部署分類は、要求の有効な例を示します。 エンティティが 1 つ以上の要求を含むことができ、リソースへのアクセスを承認するために信頼性情報の任意の組み合わせを使用できます。 信頼性情報の次の種類はサポートされているバージョンの Windows で使用できます。  
  
-   **ユーザーの信頼性情報**特定のユーザーに関連付けられている Active Directory 属性です。  
  
-   **デバイスの信頼性情報**特定のコンピューター オブジェクトに関連付けられている Active Directory 属性です。  
  
-   **リソース属性**グローバル リソース プロパティを承認の判断で使用するためにマークされ、Active Directory で公開します。  
  
信頼性情報を使用する管理者は、式、ルール、およびポリシーのユーザー、デバイス、および組み込むことができるリソースに関する正確な組織またはエンタープライズ全体のステートメントを作成します。  
  
### <a name="BKMK_Expressions2"></a>式  
条件式は、例、グループのメンバーシップ、場所、またはデバイスのセキュリティ状態の特定の条件が満たされる場合だけリソースへのアクセス許可または拒否するアクセス制御管理を強化します。 式は、ACL エディターまたは集約型アクセス規則エディターで、Active Directory 管理センター (ADAC) のセキュリティの詳細設定] ダイアログ ボックスで管理されます。  
  
式では、管理者が複雑化するビジネス環境で柔軟な条件で機密性の高いリソースへのアクセスを管理するのに役立ちます。  
  
### <a name="BKMK_Permissions2"></a>提案されたアクセス許可  
提案されたアクセス許可は、管理者はそれらを実際に変更することがなくコントロールの設定にアクセスする潜在的な変更の影響を正確にモデルを有効にします。
