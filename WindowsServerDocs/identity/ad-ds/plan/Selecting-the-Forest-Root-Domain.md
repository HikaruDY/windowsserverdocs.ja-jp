# <a name="selecting-the-forest-root-domain"></a>フォレスト ルート ドメインを選択します。

>適用対象: Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

Active Directory フォレスト内に展開する最初のドメインは、フォレスト ルート ドメインと呼ばれます。 このドメインは AD DS の展開のライフ サイクルのフォレスト ルート ドメインに残ります。  
  
フォレスト ルート ドメインには、Enterprise Admins および Schema Admins グループが含まれています。 これらのサービス管理者グループを使用して、フォレスト レベルなどの操作、追加のドメインの削除とスキーマの変更の実装を管理できます。  
  
フォレスト ルート ドメインを選択する場合は、ドメインの設計では、Active Directory ドメインのいずれかが、フォレスト ルート ドメインとして機能または専用フォレスト ルート ドメインを展開する必要がある場合を決定する必要があります。  
  
フォレスト ルート ドメインを展開する方法の詳細については、次を参照してください。[Windows Server 2008 フォレスト ルート ドメインを展開する](https://technet.microsoft.com/library/cc731174.aspx)します。  
  
## <a name="choosing-a-regional-or-dedicated-forest-root-domain"></a>地域または専用フォレスト ルート ドメインの選択  
1 つのドメイン モデルを適用する場合、1 つのドメインは、フォレスト ルート ドメインとして機能します。 複数のドメイン モデルを適用する場合の専用フォレスト ルート ドメインを展開またはフォレスト ルート ドメインとして機能する地域ドメインを選択することもできます。  
  
### <a name="dedicated-forest-root-domain"></a>専用フォレスト ルート ドメイン  
専用フォレスト ルート ドメインは、フォレストのルートとして機能する具体的には作成されるドメインです。 フォレスト ルート ドメインのサービス管理者アカウント以外のすべてのユーザー アカウントは含まれません。 また、ドメイン構造内の任意の地理的な領域は表しません。 フォレストの他のすべてのドメインは、専用のフォレスト ルート ドメインの子です。  
  
専用のフォレストのルートを使用して次の利点を提供します。  
  
-   サービス管理者のドメインからサービス管理者のフォレストの運用上の分離します。 単一ドメイン環境でビルトインの Administrators グループ、Domain Admins のメンバーは、Enterprise Admins および Schema Admins グループのメンバーになるに標準的なツールと手順を使用できます。 フォレストにある専用フォレスト ルート ドメインを使用して、Domain Admins と地域ドメインのビルトインの Administrators グループのメンバーなることはできませんフォレスト レベルのサービス管理者グループのメンバーの標準的なツールと手順を使用しています。  
  
-   他のドメイン内の運用上の変更から保護します。 専用フォレスト ルート ドメインのドメイン構造で特定の地域を表していません。 このため、これは再編成または名前の変更またはドメインの再構築すると、その他の変更によって影響ありません。  
  
-   国または地域が表示されるようにない別の地域に下位には、中立的なルートとして機能します。 一部の組織がその 1 つ国の外観を回避することができます、または領域とは別の国または地域、名前空間内の下位します。 専用フォレスト ルート ドメインを使用して、すべての地域ドメイン、ドメインの階層内のピアを使用できます。  
  
専用のフォレストのルートが使用されている複数地域ドメイン環境でネットワーク インフラストラクチャに影響を最小限には、フォレスト ルート ドメインのレプリケーション。 これは、フォレスト ルートのみサービス管理者アカウントをホストするためです。 フォレストとその他のドメインに固有のデータのユーザー アカウントの大部分は、地域ドメインに格納されます。  
  
専用フォレスト ルート ドメインを使用する 1 つの欠点は、追加のドメインをサポートするために追加の管理オーバーヘッドが作成されることです。  
  
### <a name="regional-domain-as-a-forest-root-domain"></a>フォレスト ルート ドメインと地域ドメイン  
専用フォレスト ルート ドメインを展開しないように選択する場合は、フォレスト ルート ドメインとして機能する地域ドメインを選択してください。 このドメインは、その他の地域ドメインのすべての親ドメインであり、展開する最初のドメインになります。 フォレスト ルート ドメインは、ユーザー アカウントを含むされ、その他の地域ドメインが管理されている同じ方法で管理されます。 主な違いは、Enterprise Admins および Schema Admins グループもが含まれています。  
  
関数に地域別のドメインを選択することは作成されません追加の管理オーバーヘッドが追加のドメインを管理するフォレスト ルート ドメインはの利点を作成します。 ドメインを表す、本社または最速のネットワーク接続を持つ地域など、フォレストのルートに適切な地域ドメインを選択します。 組織フォレスト ルート ドメインに地域ドメインを選択するが難しい場合は、代わりに専用フォレスト ルート モデルを使用することもできます。  
  
## <a name="assigning-the-forest-root-domain-name"></a>フォレスト ルート ドメイン名を割り当てる  
フォレスト ルート ドメイン名は、また、フォレストの名前です。 フォレスト ルート名は、プレフィックスと prefix.suffix の形式でのサフィックスで構成されるドメイン ネーム システム (DNS) 名です。 たとえば、組織では、フォレスト ルート名 corp.contoso.com があります。この例では corp プレフィックスであり contoso.com サフィックス。  
  
ネットワーク上の既存の名前の一覧から、サフィックスを選択します。 プレフィックスを新しい名前を使用していないネットワーク上で以前を選択します。 既存のサフィックスに新しいプレフィックスを添付すると、一意の名前空間を作成します。 Active Directory ドメイン サービス (AD DS) の新しい名前空間を作成する、既存の DNS インフラストラクチャが AD DS に合わせて変更する必要がないことを確認します。  
  
### <a name="selecting-a-suffix"></a>サフィックスを選択します。

1.  AD DS をホストするネットワーク上で使用されている登録済み DNS サフィックスの一覧については、組織での DNS の所有者にお問い合わせください。 内部ネットワークで使用されるサフィックスが外部で使用されるサフィックスよりも異なる可能性がありますに注意してください。 たとえば、組織可能性があります、インターネット上の contosopharma.com でとを使用 contoso.com 企業内部ネットワーク。  
  
2.  AD DS で使用するためのサフィックスを選択する DNS の所有者を参照してください。 適切なサフィックスが存在しない場合は、機関の名前を付け、インターネットとの新しい名前を登録します。  
  
Active Directory 名前空間にインターネット機関に登録されている DNS 名を使用することをお勧めします。 登録されている名のみはグローバルに一意であることが保証されます。 別の組織を後で、同じ DNS ドメイン名を登録する (または同じ DNS 名を使用する場合は、組織にマージ、取得、または別の会社によって取得された) 場合は、2 つのインフラストラクチャが相互に通信ことはできません。
