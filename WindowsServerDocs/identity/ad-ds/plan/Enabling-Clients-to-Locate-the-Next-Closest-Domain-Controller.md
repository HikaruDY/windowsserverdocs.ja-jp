# <a name="enabling-clients-to-locate-the-next-closest-domain-controller"></a>クライアントが次の最も近いドメイン コントローラーを検索できるようにする

>適用先:Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

2008 以降、Windows Server を実行するドメイン コント ローラーがあれば、行うことができますが、Windows Vista を実行するクライアント コンピュータで、またはそれ以降または Windows Server 2008 以降のドメイン コント ローラーを有効にするとより効率的に検索する、**次をお試しください最も近いサイト**グループ ポリシー設定。 この設定では、特に多くのブランチ オフィスやサイトを持っている大企業でのネットワーク トラフィックを効率化をすることで、ドメイン コント ローラー ロケーター (DC ロケーター) が向上します。

この新しい設定のドメイン コント ローラーが配置されている順序に影響を与えるため、サイト リンクのコストを構成する方法に影響を与えることができます。 多くのハブ サイトやブランチ オフィスのある企業にとって、クライアントでフェールオーバーできる番目に近いハブ サイトに最も近いハブ サイトでドメイン コント ローラーが見つけられないことを確認して、ネットワーク上の Active Directory トラフィックを大幅に短縮できます。

一般的なベスト プラクティスとして、サイトのトポロジを簡略化する必要があり、サイト リンク コストを有効にした場合は、可能な限り、 **[次へ] の最も近いサイトを試す**設定します。 多くのハブ サイト、企業ではこれにはプランの 1 つのサイト内のクライアントで別のサイトでドメイン コント ローラーにフェールオーバーが必要になる状況を処理することをすべて簡略化できます。

既定で、 **[次へ] の最も近いサイトを試す**設定が有効になっていません。 設定が有効でない場合、DC ロケーターはドメイン コント ローラーに、次のアルゴリズムを使用します。

- 同じサイトにドメイン コント ローラーを検索しようとしてください。
- 同じサイトで使用可能なドメイン コント ローラーがない場合は、ドメイン内の任意のドメイン コント ローラーを検索してみてください。

> [!NOTE]
> これは、以前のバージョンの Active Directory で DC ロケーターが使用されているのと同じアルゴリズムです。 詳細については、この記事を参照してください。 [Active Directory の機能用に DNS サポート方法](https://go.microsoft.com/fwlink/?LinkId=108587)します。

有効にした場合、 **[次へ] の最も近いサイトを試す**設定すると、DC ロケーター アルゴリズムを使用して次のドメイン コント ローラーを検索します。

- 同じサイトにドメイン コント ローラーを検索しようとしてください。
- 同じサイトで使用可能なドメイン コント ローラーがない場合は、次に最も近いサイト内でドメイン コント ローラーを検索するお試しください。 サイトは、コストの高いサイトのリンクを含む別のサイトよりもコストの低いサイト リンクがある場合は近い。
- 次に最も近いサイトに使用可能なドメイン コント ローラーがない場合は、ドメイン内の任意のドメイン コント ローラーを検索してみてください。

**[次へ] の最も近いサイトを試す**設定は、自動サイト カバレージと連携して機能します。 たとえば、次に最も近いサイトにドメイン コント ローラーがあるない場合、DC ロケーターはそのサイトの自動サイト カバレージを実行するドメイン コント ローラーを検索しようとします。

既定では、DC ロケーターでは、次に最も近いサイトを決定する場合は、読み取り専用ドメイン コント ローラー (RODC) を含む任意のサイトは考慮されません。 さらに、クライアントは、Windows Server 2008 より前のバージョンを実行するドメイン コント ローラーからの応答を取得、ときに、DC ロケーター動作と同じ場合、設定が有効でないです。
