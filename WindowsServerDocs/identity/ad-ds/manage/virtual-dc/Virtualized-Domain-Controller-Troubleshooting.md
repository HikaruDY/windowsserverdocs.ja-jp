### <a name="BKMK_ServerCoreEvents"></a>Server Core インストール オプションと、イベント ログ  
イベント ログには、仮想化ドメイン コント ローラーの複製操作に関する有用な情報の多くが含まれています。 既定では、Windows Server 2012 コンピューター インストールは、意味のないグラフィカル インターフェイスとそのため、ローカル イベント ビューアー スナップインを実行する方法はありませんの Server Core インストールは。  
  
Server Core インストールを実行しているサーバー上のイベント ログを確認するには。  
  
-   ローカルで Wevtutil.exe ツールを実行します。  
  
-   PowerShell コマンドレット Get-winevent をローカルで実行します。  
  
-   入力方向の通信を許可する「リモート イベント ログ管理」グループ (または同等のポート) の Windows の高度なファイアウォール規則を有効にした場合は、Eventvwr.exe、wevtutil.exe、または Get-winevent を使用してリモートでは、イベント ログを管理できます。 これは、Windows PowerShell 3.0 で、NETSH.exe、グループ ポリシー、または新しい Set-netfirewallrule コマンドレットを使用して Server Core インストールで実行できます。  
  
> [!WARNING]  
> DSRM になっているコンピューターにグラフィカル シェルを追加しないでください。 Windows サービス スタック (CBS) は、セーフ モードまたは DSRM で正しく動作できません。 機能または DSRM のときに役割を追加する試みはしない完了しが通常どおり起動されるまで不安定な状態にコンピューターをそのまま使用します。 正常にブートされないほとんどの状況 DSRM での仮想化ドメイン コント ローラーの複製が正常に起動できませんので、安全にグラフィカル シェルを追加することはできません。 これを行うがサポートされていないと、サーバーが使用できなくなるする可能性があります。

### <a name="advanced-troubleshooting"></a>高度なトラブルシューティング  
このモジュールを使用して高度なトラブルシューティング学べるしようとしました。*作業*ログをサンプル、何が発生したのは、いくつか説明として。 成功の仮想化ドメイン コント ローラー操作がどのようにを理解している場合の障害は環境内で明らかになります。 これらのログの昇順で、ソースによって提供され、*期待*各ログ内の複製ドメイン コント ローラーに関連するイベント (場合でも、警告およびエラー)。  
  
#### <a name="cloning-a-domain-controller"></a>ドメイン コント ローラーの複製  
この例では、複製ドメイン コント ローラーを IP アドレスを取得する DHCP を使用して SYSVOL をレプリケート frs サービスまたは DFSR を使用して (を参照してください、適切なログ必要に応じて)、グローバル カタログでは、ブランクの dccloneconfig.xml ファイルを使用します。  
  
##### <a name="directory-services-event-log"></a>ディレクトリ サービス イベント ログ  
ディレクトリ サービス ログには、大部分イベント ベースの複製操作に関する情報にはが含まれています。 ハイパーバイザーが Vm-generation ID を変更し、NTDS サービス、ノート、RID プールを無効にし、起動 ID を変更 新しい Vm-generation ID を設定し、サーバーが Active Directory のデータの受信をレプリケートします。 DFSR サービスを停止し、SYSVOL をホストするそのデータベースを削除すると、権限のない同期を強制受信します。 USN 高基準値が調整されます。

##### <a name="dcpromolog"></a>DCPROMO します。ログ  
Dcpromo.log には、ディレクトリ サービス イベント ログで説明されていない複製の実際の昇格部分が含まれています。 ログが説明については、イベント ログ エントリの影響のレベルをサポートしていないため、モジュールのこのセクションには、追加の注釈が含まれています。  
  
複製が開始する昇格プロセスの手段、DC は現在の構成のスキャン (と同様に、IFM 昇格)、既存の AD データベースを使用して再昇格し、[DC の複製 AD の入力方向の変更デルタ SYSVOL、および複製が完了するとします。  
  
> [!NOTE]  
> ログが読みやすいように、このモジュールでは、日付列を削除することで変更されました。  
  
> [!NOTE]  
> Dcpromo.log の詳細については、の概要とトラブルシューティング AD DS 簡略化された管理では、Windows Server 2012 を参照してください。  
>   
> [https://go.microsoft.com/fwlink/p/?LinkId=237244](https://go.microsoft.com/fwlink/p/?LinkId=237244)

#### <a name="BKMK_TshhotSafeRestore"></a>ドメイン コント ローラーの安全な復元のトラブルシューティングの一般的な方法  
  
1.  安全なスナップショット復元されているのに、問題が解決しないですか。  
  
    1.  ディレクトリ サービス イベント ログを調べます  
  
        1.  スナップショットの復元にエラーがあるか。  
  
        2.  AD レプリケーション エラーがあるか。  
  
    2.  システム イベント ログを調べます  
  
        1.  通信エラーがあるか。  
  
        2.  AD エラーがあるか。  
  
2.  安全なスナップショット復元想定外ですか。  
  
    1.  ユーザーか、ロールバックの原因を判断するハイパーバイザー監査ログを調べます  
  
    2.  ハイパーバイザーのすべての管理者に連絡し、通知なしの VM をロールバックするか調査します。  
  
3.  いない安全な復元とサーバー実装 USN ロールバック保護のですか。  
  
    1.  サポートされていないハイパーバイザーまたは統合サービスのディレクトリ サービス イベント ログを調べます  
  
    2.  オペレーティング システムをチェックして、Windows Server 2012 を実行している検証ですか。
