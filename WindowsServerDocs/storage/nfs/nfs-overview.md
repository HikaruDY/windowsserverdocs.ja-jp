# ネットワーク ファイル システムの概要

>適用対象: Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

このトピックでは、ネットワーク ファイル システムの役割サービスとファイル サービスおよび記憶域サービス サーバーの役割が Windows Server に含まれている機能について説明します。 ネットワーク ファイル システム (NFS) は、ファイル共有が混在した環境 Windows と Windows 以外のコンピューターの両方が含まれている企業向けのソリューションを提供します。

## 機能の説明

NFS プロトコルを使用して、Windows および Linux または UNIX などの他の Windows 以外のオペレーティング システムを実行しているコンピューターの間でファイルを転送することができます。

Windows server NFS には、NFS と NFS クライアント サーバーが含まれます。 Windows Server を実行しているコンピューターでは、その他の Windows 以外のクライアント コンピューター向け NFS ファイル サーバーとして機能する nfs サーバーを使用できます。 Nfs クライアントでは、Windows 以外の NFS サーバーに保存されているファイルにアクセスする Windows Server を実行している Windows ベースのコンピューターができます。

## Windows および Windows Server のバージョン

Windows では、NFS クライアントとオペレーティング システムのバージョンとファミリに応じて、サーバーの複数のバージョンをサポートします。 

| オペレーティング システム | NFS サーバー バージョン |NFS クライアントのバージョン|
| ----------------- | ------------------- | ----------------- |
| Windows 7、Windows 8.1、Windows 10 | なし | Nfs V2、NFSv3 |
| Windows Server 2008、Windows Server 2008 R2 | Nfs V2、NFSv3 | Nfs V2、NFSv3 |
| Windows Server 2012、Windows Server 2012 R2、Windows Server 2016、Windows Server 2019 | Nfs V2、NFSv3、NFSv4.1  | Nfs V2、NFSv3 |

## 実際の適用例

NFS を使用する方法をいくつか示します。

- Windows NFS ファイル サーバーを使用して、複数のプラットフォーム クライアントから、同じファイル共有に SMB と NFS の両方のプロトコル経由でのマルチ プロトコルのアクセスを提供します。
- Windows 以外のクライアント コンピューター NFS ファイル共有へのアクセスを提供する、主に Windows 以外のオペレーティング システム環境で Windows NFS ファイル サーバーを展開します。
- SMB と NFS の両方のプロトコルを通じてアクセスできるファイル共有上のデータを格納することによって、アプリケーションを 1 つのオペレーティング システムから別に移行します。

## NFS バージョン 4.1

NFS バージョン 4.1 では、すべての[RFC 5661](https://tools.ietf.org/html/rfc5661)の省略可能な側面の一部だけでなく、必要な側面を実装します。

- **擬似ファイル システム**、物理的および論理的な名前空間を分離し、NFS バージョン 3 と NFS バージョン 2 と互換性のあるファイル システムです。 これは、擬似ファイル システムの一部エクスポート ファイル システムのエイリアスが提供されます。
- **複合 Rpc**では、関連する演算を組み合わせるし、読み取りを削減します。
- **セッションおよびセッション トランク**により、1 つだけセマンティックでき、継続的な可用性と NFS 4.1 クライアントと NFS サーバー間で複数のネットワークを活用してパフォーマンスを向上できます。

## NFS インフラストラクチャ

Windows Server 2012 で NFS インフラストラクチャ全体の機能強化以下で説明します。

- **リモート プロシージャ コール (RPC)/外部データ表現 (XDR)** トランスポート インフラストラクチャ、WinSock のネットワーク プロトコルによって強化されたは両方 NFS サーバーと NFS クライアントで利用できます。 向上のサポートを提供、およびスケーラビリティを向上し、受信側のスケーリング (RSS) を提供します。 このトランスポート デバイス インターフェイス (TDI) に置き換えられます。
- **RPC ポート マルチプレクサー**機能は、ファイアウォール対応 (を管理するポートが少なくなります) と NFS の展開を簡略化します。
- **キャッシュの自動調整とスレッド プール**は、動的で自動的にキャッシュを調整し、スレッド プールのワークロードに基づく新しい RPC/XDR インフラストラクチャのリソースの管理機能。 NFS が展開されるとすぐに最適なパフォーマンスを提供する、パラメーターを調整するときに完全に関連する推測が削除されます。
- **新しい Kerberos プライバシーの実装と認証のオプション**の既存の krb5 と krb5i 認証オプションと共に Kerberos プライバシー (Krb5p) のサポートを追加します。
- **Identity マッピングの Windows PowerShell モジュール**のコマンドレットをやすく id マッピングを管理、Active Directory ライトウェイト ディレクトリ サービス (AD LDS) を構成およびパスワードを UNIX と Linux とフラットなファイルを設定します。
- Nfs version 4.1 NFS 共有下にマウントされたボリュームにアクセスする、**ボリュームがマウント ポイント**できます。
- **ポートの多重化**機能をサポートすると、RPC ポート マルチプレクサー (ポート 2049) ファイアウォール フレンドリーなは、NFS 展開を簡略化します。

## NFS バージョン 3 の継続的な可用性

NFS バージョン 3 のクライアントには、複数の可用性とダウンタイムを短縮、高速かつ透過的な計画されたフェイル オーバーことができます。 フェールオーバー プロセスは、ために、NFS バージョン 3 のクライアントの高速です。

- クラスター インフラストラクチャには、ネットワーク名、共有リソースのフェールオーバー時間が大幅に向上する 1 つのリソースではなく 1 つのリソースで使用できます。
- NFS サーバー内でのフェールオーバー パスは、パフォーマンス向上のために調整されます。
- NFS サーバーでのワイルドカードの登録が必要なしなくなったと、フェールオーバーの詳細なします。
- ネットワーク状態モニター (NSM) 通知がフェールオーバー プロセスの後送信され、クライアントが、障害が発生したに再接続する TCP タイムアウトを待機する必要がなくなりましたサーバーを経由します。

NFS サーバーのみ手動で開始すると、通常計画されたメンテナンス時に、透明なフェールオーバーをサポートしているに注意してください。 予期しないフェールオーバーが発生した場合、NFS クライアントには、接続が失われます。 NFS サーバーも再開のキー フィルターの統合はありません。 これは、ローカル アプリまたは SMB セッションは、NFS クライアントに予定されたフェールオーバー後すぐにアクセスしている同じファイルにアクセスを試みると、NFS クライアントができなくなります (透明なフェールオーバーが成功はありません)、接続を意味します。

## 展開と管理性の向上

NFS を管理するための展開と、次の方法が向上します。

- 新しい 40 以上の Windows PowerShell コマンドレット容易に構成および NFS ファイル共有を管理します。 詳細については、 [Windows PowerShell コマンドレット NFS](https://docs.microsoft.com/powershell/module/nfs/?view=win10-ps)を参照してください。
- ローカルのフラットなファイルのマッピングのストア id マッピングを構成するための新しい Windows PowerShell コマンドレットと id マッピングが向上します。
- サーバー マネージャーのグラフィカル ユーザー インターフェイスが使いやすくします。
- 新しい WMI バージョン 2 のプロバイダーは、管理を容易に利用できます。
- RPC ポート マルチプレクサー (ポート 2049) がファイアウォール対応および NFS の展開を簡略化します。

## サーバー マネージャー情報

サーバー マネージャーまたはそれ以降の[Windows Admin Center](../../manage/windows-admin-center/understand/windows-admin-center.md)で、NFS サーバーの役割サービスを追加する役割の追加と機能の削除ウィザードを使用 ([ファイルおよび iSCSI サービスの役割)。 機能のインストールに関する一般的な情報は、[インストールまたはアンインストール役割、役割サービス、または機能](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831809(v=ws.11)>)を参照してください。 NFS ツール用のサーバーには、nfs サーバーを管理するネットワーク ファイル システム MMC スナップインのサービスと NFS コンポーネントのクライアントが含まれます。 スナップインを使用して、コンピューターにインストールされている NFS コンポーネント用のサーバーを管理できます。 NFS サーバーには、いくつかの Windows のコマンド ラインの管理ツールも含まれています。

- **マウント**リモート NFS 共有 (エクスポート) をローカルにマウントし、Windows クライアント コンピューター上のローカル ドライブ文字にマップします。
- **次のよう**NFS コンポーネントの NFS とクライアント用のサーバーの設定の構成を管理します。
- **Nfsshare** NFS サーバーを使用して共有されているフォルダーの NFS 共有設定を構成します。
- **Nfsstat**を表示または nfs サーバーで受信した呼び出しの統計情報をリセットします。
- **Showmount**には、nfs サーバーによってエクスポートされたマウントされているファイル システムが表示されます。
- **マウント解除**は、NFS マウントされたドライブを削除します。

NFS Windows Server 2012 では、NFS の具体的にはいくつかの新しいコマンドレットで Windows PowerShell 用の NFS モジュールを紹介します。 これらのコマンドレットは、NFS 管理タスクを自動化する簡単な方法を提供します。 詳細については、 [Windows PowerShell コマンドレット NFS](https://docs.microsoft.com/powershell/module/nfs/?view=win10-ps)を参照してください。
