# <a name="folder-redirection-offline-files-and-roaming-user-profiles-overview"></a>フォルダーのリダイレクト、オフライン ファイル、および移動ユーザー プロファイルの概要

>対象: Windows 10、Windows 8、Windows 8.1、Windows Server 2012、Windows Server 2012 R2、Windows Server 2016

このトピックでは、フォルダーのリダイレクト、オフライン ファイル (クライアント側のキャッシュまたは CSC)、および移動ユーザー プロファイル (RUP とも呼ばれます) テクノロジをなどの新機能とその他の情報がある場所について説明します。

## <a name="technology-description"></a>テクノロジの説明

フォルダー リダイレクトとオフライン ファイルを同時に使うと、フォルダーの内容をローカルでキャッシュすることで速度と可用性を向上しつつ、(ドキュメント フォルダーなどの) ローカル フォルダーのパスをネットワーク上の場所にリダイレクトできます。 移動ユーザー プロファイルは、ユーザー プロファイルをネットワーク上の場所にリダイレクトするために使われます。 これらの機能 Intellimirror として参照するために使用します。

- **フォルダーのリダイレクト**には、ユーザーと手動またはグループ ポリシーを使用して、新しい場所に既知のフォルダーのパスをリダイレクトするのには管理者が使用されます。 新しい場所には、ローカル コンピューター上のフォルダーまたはファイル共有ディレクトリを使用できます。 ユーザーは、ローカル ドライブにまだ存在している場合、リダイレクト先のフォルダー内のファイルを操作します。 たとえば、ネットワーク上の場所には通常、ローカル ドライブに保存されている [ドキュメント] フォルダーをリダイレクトすることができます。 フォルダー内のファイルは、[利用可能なユーザーにネットワーク上の任意のコンピューターからします。
- **オフライン ファイル**ネットワーク ファイル使用できるように、ユーザー場合でも、サーバーへのネットワーク接続が使用できないか、速度が遅いためです。 オンラインで作業しているとき、ファイルのアクセスのパフォーマンスは、サーバー、ネットワークの速度です。 オフラインで作業すると、ローカル アクセス速度でオフライン ファイル フォルダーからファイルが取得されます。 コンピューターがオフライン モードに切り替える場合。
  - **常にオフライン**モードを有効になっています。
  - サーバーが使用できません。
  - ネットワーク接続が設定可能なしきい値よりも低速化
  - ユーザーが手動で Windows エクスプ ローラーで、[**オフライン作業**] ボタンを使用してオフライン モードに切り替える
- **ユーザー プロファイルの移動**では、ユーザーは、複数のコンピューター上のアプリケーションの設定と同じオペレーティング システムを受信できるように、ユーザー プロファイルをファイル共有にリダイレクトします。 プロファイルのパスとファイルの共有に設定されているアカウントを使用して、コンピューターにユーザーがサインインすると、ユーザーのプロファイルがローカル コンピューターにダウンロードし、(存在する場合)、ローカルのプロファイルと結合します。 コンピューターから、ユーザーが、すべての変更など、その人のプロファイルのローカル コピーがプロファイルのサーバー側のコピーと結合されます。 一般的に、ネットワーク管理者は、ドメインのアカウントで移動ユーザー プロファイルを使用します。

## <a name="practical-applications"></a>実際の適用例

管理者は、集中ユーザー データと設定を保存して、ユーザーがオフラインまたはネットワークやサービスが停止中にデータを使用できるようにするフォルダーのリダイレクト、オフライン ファイル、および移動ユーザー プロファイルを使用できます。 特定のアプリケーションによっては、次のとおりです。

- サーバー ベースのバックアップ ツールを使用して、ユーザーのフォルダーと設定をバックアップするなどの管理タスクは、クライアント コンピューターからのデータを一元管理します。
- ネットワークまたはサーバーの停止がある場合でも、ユーザーに、ネットワーク ファイルへのアクセスを有効にします。
- 帯域幅使用量を最適化して、ファイルとオフサイトのどこにある企業のサーバーでホストされているフォルダーにアクセスする支店のユーザーのエクスペリエンスを向上します。
- 低速のネットワーク経由でオフラインで作業中にネットワーク ファイルにアクセスするモバイル ユーザーを有効にします。

## <a name="always-offline-mode"></a>常にオフライン モード

Windows 8 および Windows Server 2012 から始めて、管理者は、常にオフラインで作業する高速のネットワーク接続で接続がときにもファイルをオフラインでのユーザーの環境を構成できます。 Windows では、既定では、時間単位からバック グラウンドで同期することで、ファイルをオフライン キャッシュ内のファイルを更新します。

### <a name="what-value-does-always-offline-mode-add"></a>どのような値が常に、オフライン モードの追加ですか。

常にオフライン モードでは、次のような利点があります。

- ユーザーには、[ドキュメント] フォルダーなどのリダイレクトされたフォルダー内のファイルへのアクセスを高速化が発生します。
- 高価な WAN 接続またはモバイル 4 G ネットワークなどの従量制課金接続でコストの減少、ネットワークの帯域幅が減少します。

### <a name="how-has-always-offline-mode-changed-things"></a>常にオフライン モードで作業が変更された方法

ネットワークの可用性と低速リンク モード (接続速度が遅いモードとも呼ばれます) が有効にして、1 ミリ秒に設定する場合でも、条件によって、オンラインおよびオフライン モードの切り替えはユーザーの Windows 8 の場合は、Windows Server 2012 よりも前待機時間のしきい値。

常にオフライン モードでは、コンピューターことがない画面切り替えを設定オンライン モードに**低速リンク モードを構成する**グループ ポリシー設定が構成されているし、**待機時間**のしきい値パラメーターがミリ秒が 1 に設定します。 変更は、同期、バック グラウンドで最大 120 分ごとに、既定が**バック グラウンドの同期を構成する**グループ ポリシー設定を使用して同期が可能。

詳細については、[常にオフライン モードを提供する高速ファイルへのアクセスを有効にする](enable-always-offline.md)を参照してください。

## <a name="cost-aware-synchronization"></a>コスト対応の同期

コストに対応した同期では、Windows では、ユーザーがモバイル 4 G ネットワークとサブスクライバーは、帯域幅の制限に近いかのような従量制課金ネットワーク接続を使用して、または別のプロバイダーのネットワーク上に移動するときにバック グラウンド同期が無効にします。

>[!NOTE]
>従量制課金ネットワーク接続では、Windows 8、Windows Server 2012、および Windows Server 2016 の [オフライン (遅い) モードに移行する際の既定 35 ミリ秒待機時間の値よりもラウンドト リップ ネットワーク待ち時間通常があります。 このため、これらの接続通常移行オフライン (遅い) モードに自動的にします。

### <a name="what-value-does-cost-aware-synchronization-add"></a>コストに対応した同期はどのような値を追加しますか。

コストに対応して同期を使うと、使用量の制限のある従量制課金接続を使っているとき、または別のプロバイダーのネットワーク上に移動するときに異常なデータ使用のコストを回避するやすくなります。

### <a name="how-has-cost-aware-synchronization-changed-things"></a>コストに対応した同期が項目を変更する方法がよいですか。

Windows 8、Windows Server 2012 の前に、モバイル ネットワーク プロバイダーからツールを使用して、データの使用状況を追跡する従量制課金ネットワーク接続でファイルをオフラインでの使用中に料金を最小化したいユーザーがありました。 ユーザー手動で切り替えて、オフライン モードが移動したときの帯域幅の制限の近くにある、またはその制限を超えてします。

コストに対応した同期は、Windows は自動的に従量制課金接続時にローミングと帯域幅使用量の制限を追跡します。 ユーザーが移動の帯域幅の制限の近くにある、またはその制限を超えてと Windows オフライン モードに切り替わりますのすべての同期ができなくなります。 ユーザーが手動で同期の開始、および管理者エグゼクティブなど、特定のユーザーに関するコスト対応の同期を無効にできます。

詳細については、[従量制課金ネットワークにバック グラウンド ファイル同期を有効にする](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/jj127408(v%3dws.11))を参照してください。

## <a name="primary-computers-for-folder-redirection-and-roaming-user-profiles"></a>フォルダーのリダイレクトおよび移動ユーザー プロファイルの主のコンピューター

今すぐドメイン、ユーザーごとにすることができるフォルダー リダイレクト、ユーザー プロファイルの移動、またはその両方を使用するコンピューターはどれかを制御する主のコンピューターと呼ばれる、コンピューターのセットを指定できます。 主のコンピューターを指定することは、特定のコンピューターまたはデバイスにユーザー データと設定を関連付ける方法をシンプルで強力な管理者の管理を簡素化、データのセキュリティを強化破損からユーザー プロファイルを保護するため。

### <a name="what-value-do-primary-computers-add"></a>主のコンピューターでどのような値を追加すればよいですか。

ユーザーのプライマリ コンピューターを指定する 4 つの主な利点があります。

- 管理者は、ユーザーを使用してリダイレクトされたデータの設定にアクセスするコンピューターを指定できます。 たとえば、管理者は、ユーザー データと設定がユーザーのデスクトップおよびノートを移動して、そのユーザーが、会議室コンピューターなどの他のコンピューターにログオンするときに情報を移動しないを選択できます。
- セキュリティと、ユーザーがログオンしているコンピューターでその他の個人または会社のデータのプライバシーに関するリスクの軽減主のコンピューターを指定します。 たとえば、一時的なアクセス権の従業員のコンピューターにログオンする一般的なマネージャーしない、個人または会社のデータの背後にします。
- 主のコンピューターに管理者が正しく構成されているのリスクを軽減するためが有効にするか、それ以外の場合異なる間の移動は、破損したプロファイルが、構成システムなど、x86 ベースおよび x64 ベースのコンピューター間で。
- ユーザーのユーザー プロファイルまたはリダイレクトされたフォルダーがダウンロードされないために、ユーザーの最初のサインイン サーバーなど、プライマリ以外のコンピューター上に必要な時間が高速です。 ユーザー プロファイルに加えた変更は、共有するファイルをアップロードする必要はありませんので、サインアウト時間が短縮もします。

### <a name="how-have-primary-computers-changed-things"></a>主のコンピューターで項目が変更されて方法

主のコンピューターにユーザーの個人データをダウンロードを制限するには、フォルダー リダイレクトおよび移動ユーザー プロファイルのテクノロジは、コンピューターにサインインしたときに次のロジック チェックを実行します。

1. Windows オペレーティング システム、新しいグループ ポリシーの設定を確認 (**主のコンピューターのみ移動プロファイルをダウンロード**および**主のコンピューターのみリダイレクト フォルダー**) **msDS 台のコンピューター**がアクティブな属性を確認するにはDirectory ドメイン サービス (AD DS) ユーザーのプロファイルを移動またはフォルダーのリダイレクトを適用するかに影響を与える必要があります。
2. ポリシー設定は、主のコンピューターのサポートは可能 Windows AD DS スキーマが**msDS 台のコンピューター**の属性をサポートしていることを確認します。 そのような場合は、ユーザーのログオンしているコンピューターが次のように、ユーザーのプライマリ コンピューターとして指定されたかどうかは Windows を決定します。
    1. コンピューターが、ユーザーのプライマリ コンピューターのいずれかの場合は、Windows には、移動ユーザー プロファイルとフォルダーのリダイレクションの設定が適用されます。
    2. コンピューターが、ユーザーのプライマリ コンピューターのいずれかでない場合は、Windows がある場合は、ユーザーのキャッシュされたローカル プロファイルを読み込みか、新しいローカル プロファイルを作成します。 Windows には、既存のリダイレクトされたフォルダーでは、ローカル フォルダー リダイレクションの設定が保持される前に適用したグループ ポリシー設定によって指定された削除操作によっても削除されます。

詳細については、[フォルダーのリダイレクトおよび移動ユーザー プロファイルのプライマリ コンピューターを展開する](deploy-primary-computers.md)を参照してください。
