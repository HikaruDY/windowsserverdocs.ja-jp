## <a name="prerequisites"></a>前提条件

### <a name="hardware-requirements"></a>ハードウェア要件

フォルダー リダイレクト、x64 ベースまたは x86 ベースのコンピューターが必要です。Windows® RT. ではサポートされていません

### <a name="software-requirements"></a>ソフトウェア要件

フォルダー リダイレクトでは、次のソフトウェア要件があります。

- フォルダー リダイレクトを管理するには、Domain Administrators セキュリティ グループ、Enterprise Administrators セキュリティ グループ、または Group Policy Creator Owners セキュリティ グループのメンバーとしてサインインする必要があります。
- クライアント コンピューターには、Windows 10、Windows 8.1、Windows 8、Windows 7、Windows Server 2019、Windows Server 2016、Windows Server (半期チャネル)、Windows Server 2012 R2、Windows Server 2012、Windows Server 2008 R2、または Windows Server 2008 を実行する必要があります。
- クライアント コンピューターは、管理している Active Directory ドメイン サービス (AD DS) に参加している必要があります。
- グループ ポリシー管理および Active Directory 管理センターがインストールされたコンピューターが使用できる必要があります。
- ファイル サーバーは、リダイレクトされたフォルダーをホストできる必要があります。
    - ファイル共有で DFS 名前空間を使用している場合、ユーザーのさまざまなサーバーへの編集の競合を防ぐため、DFS フォルダー (リンク) のターゲットは 1 つである必要があります。
    - ファイル共有で DFS レプリケーションを使用して、コンテンツを別のサーバーにレプリケートする場合、ユーザーのさまざまなサーバーへの編集の競合を防ぐため、ユーザーはソース サーバーにのみアクセスできる必要があります。
    - クラスター化されたファイル共有を使用する場合は、フォルダー リダイレクトとオフライン ファイルのパフォーマンスの問題を回避するためにファイル共有の継続的な可用性を無効にします。 さらに、オフライン ファイルは、ユーザーがまだ、オフライン ファイルの常時オフライン モードを使用していないユーザーのストレスを感じる可能性、継続的に使用可能なファイル共有へのアクセスを失った後 3 ~ 6 分のオフライン モードに移行できない可能性があります。

> [!NOTE]
> フォルダー リダイレクトの一部の新しい機能は、追加のクライアント コンピューターと Active Directory スキーマの要件があります。 詳細については、次を参照してください[プライマリ コンピューターを展開する](deploy-primary-computers.md)、[を無効にするオフライン ファイル フォルダーの](disable-offline-files-on-folders.md)、[を有効にするの常時オフライン モード](enable-always-offline.md)、および[フォルダーの移動に最適化された有効にする。](enable-optimized-moving.md).

フォルダー リダイレクトの GPO を作成する方法を次に示します。

1. グループ ポリシー管理がインストールされたコンピューターでサーバー マネージャーを開きます。
2. **ツール**メニューの  **Group Policy Management**します。
3. ドメインまたはフォルダーのリダイレクトをセットアップしを選択する OU を右クリックして**このドメインに GPO を作成し、リンクする**します。

## <a name="step-4-configure-folder-redirection-with-offline-files"></a>手順 4:オフライン ファイルとフォルダー リダイレクトを構成します。

フォルダー リダイレクトの設定用の GPO を作成した後、次の手順で説明したように有効にし、フォルダー リダイレクトを構成するには、グループ ポリシー設定を編集します。

> [!NOTE]
> オフライン ファイルは、Windows クライアント コンピューターにリダイレクトされたフォルダーの既定で有効になっているし、ユーザーが変更しない限り、Windows Server を実行するコンピューターで無効になっています。 オフライン ファイルが有効になっているかどうかは、グループ ポリシー コントロールを使用して、使用、**許可、オフライン ファイル機能の使用を禁止または**ポリシー設定。
> その他のオフライン ファイル グループのポリシー設定の一部については、次を参照してください。[を有効にする高度なオフライン ファイル機能](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>)、および[オフライン ファイルのグループ ポリシーを構成する](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>)します。

グループ ポリシーでフォルダー リダイレクトを構成する方法を次に示します。

1. グループ ポリシー管理で作成した GPO を右クリックし (たとえば、**フォルダー リダイレクトの設定**)、し、**編集**します。
2. グループ ポリシー管理エディター ウィンドウに移動します**ユーザー構成**、し**ポリシー**、し**Windows 設定**、し**フォルダー。リダイレクト**します。
3. リダイレクトするフォルダーを右クリックして (たとえば、**ドキュメント**)、し、**プロパティ**します。
4. **プロパティ** ダイアログ ボックスから、**設定**ボックスで、**基本 - 全員のフォルダーを同じ場所にリダイレクト**します。

    > [!NOTE]
    > Windows XP または Windows Server 2003 を実行しているクライアント コンピューターには、フォルダー リダイレクトを適用するには、選択、**設定** タブで選択し、**もリダイレクト ポリシーを適用すると、Windows 2000、Windows 2000 Server、Windows XP、およびWindows Server 2003 オペレーティング システム**チェック ボックスをオンします。

5. **ターゲットのフォルダー場所**セクションで、**ルート パスの下には、各ユーザーのフォルダーを作成**し、**ルート パス**ボックスに、格納するファイル共有へのパスを入力たとえば、フォルダーのリダイレクト:  **\\ \\fs1.corp.contoso.com\\ユーザー $**
6. 選択、**設定** タブで、および、**ポリシー削除時** セクションで、必要に応じて  **、ポリシーが削除されたときに、ローカル ユーザー プロファイルの場所にフォルダーをリダイレクト**(この設定に役立ちます adminisitrators とユーザーの動作は予想フォルダー リダイレクト)。
7. 選択 **[ok]** 、し、[**はい**警告] ダイアログ ボックス。

## <a name="step-5-enable-the-folder-redirection-gpo"></a>手順 5:GPO のフォルダー リダイレクトを有効にします。

フォルダー リダイレクト グループ ポリシー設定の構成を完了すると、次の手順は、許可することと、影響を受けるユーザーに適用する GPO を有効にするのには。

> [!TIP]
> プライマリ コンピューターのサポートまたはその他のポリシー設定を実装する予定がある場合は、GPO を有効にする前に、ここでそれらを実行してください。 これにより、プライマリ コンピューターのサポートが有効にされる前に、ユーザー データがプライマリでないコンピューターにコピーされることを防ぎます。

フォルダー リダイレクトの GPO を有効にする方法を次に示します。

1. グループ ポリシーの管理を開きます。
2. 選択し、作成した GPO を右クリックして**リンクになっている**します。 メニュー項目の横にあるチェック ボックスが表示されます。

## <a name="step-6-test-folder-redirection"></a>手順 6:フォルダー リダイレクトをテストします。

フォルダー リダイレクトをテストするには、フォルダー リダイレクト用に構成されたユーザー アカウントでコンピューターにサインインします。 フォルダーとプロファイルがリダイレクトされることを確認します。

フォルダー リダイレクトをテストする方法を次に示します。

1. サインインにプライマリ コンピューター (プライマリ コンピューターのサポートが有効な) 場合フォルダー リダイレクトが有効にするユーザー アカウントを使用します。
2. ユーザーが以前にコンピューターにサインインしている場合、管理者特権でコマンド プロンプトを開き、次のコマンドを入力して、クライアント コンピューターに最新のグループ ポリシー設定が適用されるようにします。
    
    ```PowerShell
    gpupdate /force
    ```
3. エクスプローラーを開きます。
4. リダイレクトされたフォルダー (たとえば、マイ ドキュメント フォルダー、ドキュメント ライブラリで) を右クリックし、**プロパティ**します。
5. 選択、**場所**タブをクリックし、パスがローカル パスの代わりに指定したファイル共有が表示されることを確認します。
