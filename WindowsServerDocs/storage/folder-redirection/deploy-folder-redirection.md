>適用対象: Windows 10、Windows 7、Windows 8、Windows 8.1、Windows Server 2008 R2、Windows Server 2012、Windows Server 2012 R2、Windows Server 2016、Windows Vista

このトピックでは、Windows Server を使用して、フォルダー リダイレクトとオフライン ファイルを Windows クライアント コンピューターに展開する方法について説明します。

## 前提条件

### ハードウェア要件

フォルダー リダイレクト、x64 または x86 ベースのコンピューターが必要です。Windows® RT ではサポートされていません

### ソフトウェア要件

フォルダー リダイレクトでは、次のソフトウェア要件があります。

- フォルダー リダイレクトを管理するには、ドメイン管理者セキュリティ グループ、エンタープライズ管理者セキュリティ グループ、またはグループ ポリシーの作成者の所有者のセキュリティ グループのメンバーとして署名が必要です。
- クライアント コンピューターには、Windows 10、Windows 8.1、Windows 8、Windows 7、Windows Server 2012 R2、Windows Server 2012、Windows Server 2008 R2、または Windows Server 2008 を実行する必要があります。
- クライアント コンピューターは、Active Directory ドメイン サービス (AD DS) が管理しているに参加する必要があります。
- コンピューターは、グループ ポリシーの管理とインストールされている Active Directory 管理センターで利用可能である必要があります。
- ファイル サーバーは、リダイレクトされたフォルダーをホストできる必要があります。
    - ファイル共有は、DFS 名前空間を使用している場合、DFS フォルダー (リンク) に 1 つのターゲットをユーザーが複数のサーバーで競合する編集することを防ぐために必要があります。
    - ファイル共有では、DFS レプリケーションを使用して、別のサーバーを使ってコンテンツをレプリケートしている場合、ユーザーをユーザーが複数のサーバーで競合する編集することを防ぐために、ソース サーバーのみがアクセスできる必要があります。
    - クラスター化されたファイル共有を使用する場合は、フォルダー リダイレクトとオフライン ファイルとパフォーマンスの問題を回避するためにファイル共有の可用性を無効にします。 さらに、オフライン ファイルは、ユーザーがまだオフライン ファイルを常にオフライン モードを使用していないユーザーに不満可能性がありますが、継続的に利用可能なファイル共有へのアクセスを失った後も、3 ~ 6 分間のオフライン モードに移行されていない可能性があります。

>[!NOTE]
>フォルダー リダイレクトの一部の新しい機能では、追加のクライアント コンピューターと Active Directory スキーマの要件があります。 詳しくは、[プライマリ コンピューターの展開](deploy-primary-computers.md)、[無効にするオフライン ファイル](disable-offline-files-on-folders.md)フォルダーに、[モードを有効に常にオフラインで](enable-always-offline.md)、および[最適化されたフォルダーに移動を有効にする](enable-optimized-moving.md)を参照してください。

## 手順 1: フォルダー リダイレクトのセキュリティ グループを作成します。

環境が既に設定されていない場合、フォルダー リダイレクト、最初の手順が、フォルダー リダイレクト ポリシー設定を適用するすべてのユーザーが含まれているセキュリティ グループを作成します。

フォルダー リダイレクトのセキュリティ グループを作成する方法を次に示します。

1. Active Directory 管理センターで、コンピューターでサーバー マネージャーを開きますがインストールされます。
2. [**ツール**] メニューには、 **Active Directory 管理センター**を選択します。 Active Directory 管理センターが表示されます。
3. 適切なドメインまたは OU を右クリックし、**新規**作成] を選択し、**グループ**を選択します。
4. [**グループの作成**] ウィンドウの [**グループ**] で、次の設定を指定します。
    - [**グループ名**を入力など、セキュリティ グループの名前:**フォルダー リダイレクト ユーザー**。
    - **グループのスコープ**内では、**セキュリティ**を選択し、**グローバル**] を選択します。
5. [**メンバー** ] セクションで、**追加**を] を選択します。 [ユーザー、連絡先、コンピューター、サービス アカウントまたはグループの選択] ダイアログ ボックスが表示されます。
6. ユーザーまたはフォルダー リダイレクトを展開し、 **[ok]** を選択し、もう一度 **[ok]** を選択するグループの名前を入力します。

## 手順 3: フォルダー リダイレクトの GPO を作成します。

フォルダー リダイレクト設定用に作成された GPO があるまだない場合は、作成するのには、次の手順を使用します。

フォルダー リダイレクトの GPO を作成する方法を次に示します。

1. グループ ポリシーの管理で、コンピューターでサーバー マネージャーを開きますがインストールされます。
2. **ツール**] メニューからは、**グループ ポリシーの管理**を選択します。
3. ドメインまたはフォルダーのリダイレクトをセットアップする OU を右クリックし、**このドメインに GPO を作成しここでそのリンク**を選択します。
4. **新しい GPO** ] ダイアログ ボックスで、GPO (たとえば、**フォルダー リダイレクトの設定**) の名前を入力し、 **[ok]** を選択します。
5. 新しく作成した GPO を右クリックし、**リンクを有効になっている**チェック ボックスをオフにし、します。 これは、GPO が構成を完了するまでに適用されることを防ぎます。
6. GPO を選択します。 [**スコープ**] タブの [**セキュリティ フィルター処理**] セクションで、 **Authenticated Users**を選択し、**削除**を GPO がすべてのユーザーに適用されていることを防ぐためにを選択します。
7. [**セキュリティ フィルター処理**] セクションで**追加**を] を選択します。
8. **[ユーザー、コンピューター、またはグループ**] ダイアログ ボックスで、手順 1 (たとえば、**フォルダー リダイレクト ユーザー**) で作成したセキュリティ グループの名前を入力し、 **[ok]** を選択します。
9. **[委任**] タブを選択、[**追加**] を選択**Authenticated Users**の入力し、 **[ok]** を選択既定の読み取りアクセス許可を承認するには、もう一度 **[ok]** を選択します。
    
    この手順は、 [MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)でセキュリティの変更のために必要があります。

>[!IMPORTANT]
>[MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)、するようになりましたに加えられた変更を提供する必要がありますセキュリティ Authenticated Users グループは、フォルダー リダイレクト GPO に対する読み取りアクセス許可を委任 - それ以外の場合、ユーザーに GPO を適用しませんを取得する、またはによりが既に適用される場合、GPO が削除され、リダイレクトPC をローカル フォルダーにバックアップします。 詳しくは、[展開するグループ ポリシーのセキュリティ更新プログラム MS16-072](https://blogs.technet.microsoft.com/askds/2016/06/22/deploying-group-policy-security-update-ms16-072-kb3163622/)を参照してください。

## 手順 4: オフライン ファイルとフォルダー リダイレクトを構成します。

フォルダー リダイレクト設定用の GPO を作成した後、次の手順で説明したように有効にして、フォルダー リダイレクトを構成するには、グループ ポリシー設定を編集します。

>[!NOTE]
>オフライン ファイルが既定では、Windows クライアント コンピューターにリダイレクトされたフォルダーを有効になっているし、ユーザーによって変更された場合を除き、Windows Server を実行しているコンピューターで無効にします。 オフライン ファイルが有効になっているかどうかを制御するグループ ポリシーを使用する、**許可オフライン ファイルの機能の使用を禁止または**ポリシー設定します。
> その他のオフライン ファイルのグループ ポリシー設定の一部については、[有効にする高度なオフライン ファイルの機能](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>)、および[オフライン ファイルのグループ ポリシーの構成](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>)を参照してください。

グループ ポリシーでフォルダー リダイレクトを構成する方法を次に示します。

1. グループ ポリシーの管理 (たとえば、**フォルダー リダイレクトの設定**) を作成した GPO を右クリックして選び、**編集**します。
2. グループ ポリシー管理エディター ウィンドウで、**ユーザーの構成**では、その**ポリシー**をし、 **Windows の設定**し、[**フォルダー リダイレクト**に移動します。
3. (たとえば、**ドキュメント**など) をリダイレクトして、**プロパティ**を選択するフォルダーを右クリックします。
4. **プロパティ**] ダイアログ ボックスで、**設定**のボックスから**基本 - 全員のフォルダーと同じ場所にリダイレクト**を選択します。

    > [!NOTE]
    > フォルダー リダイレクトを Windows XP または Windows Server 2003 を実行しているクライアント コンピューターに適用し、 **[設定**] タブを選択して、選択、**もリダイレクト ポリシーの適用を Windows 2000、Windows 2000 Server、Windows XP、および Windows Server 2003 オペレーティングシステム**チェック ボックス。
5. **ターゲット フォルダーの場所**のセクションで選択**ルート パスの下には、各ユーザーのフォルダーを作成**し、**ルート パス**] ボックスに入力など、リダイレクトされたフォルダーを保存するファイル共有へのパス: **\\\fs1.corp.contoso.com\\ユーザー $**
6. **[設定**] タブを選択し、セクションでは、**ポリシーの削除**、必要に応じて **、ポリシーが削除されるローカルのユーザー プロファイルの場所に戻すフォルダー リダイレクト**(この設定を高めるフォルダー リダイレクトが複数の動作予想どおりの adminisitrators とユーザー)。
7. **[Ok]** を選択し、警告ダイアログ ボックスで **[はい]** を選択します。

## 手順 5: フォルダー リダイレクト GPO を有効にします。

フォルダー リダイレクト グループ ポリシー設定の構成が完了したら、次の手順では影響を受けるユーザーに適用することを許可する、GPO を有効にします。

>[!TIP]
>プライマリ コンピューターのサポートやその他のポリシー設定を実装する場合は、今、GPO を有効にする前に。 これにより、ユーザー データはプライマリ コンピューターのサポートを有効にする前に、プライマリ以外のコンピューターにコピーされています。

フォルダー リダイレクトの GPO を有効にする方法を次に示します。

1. グループ ポリシーの管理。
2. 作成した GPO を右クリックし、**リンクの有効化**します。 メニュー項目の横にあるチェック ボックスが表示されます。

## 手順 6: フォルダー リダイレクトをテストします。

フォルダー リダイレクトをテストするには、フォルダー リダイレクト用に構成されたユーザー アカウントを持つコンピューターにサインインします。 フォルダーとプロファイルがリダイレクトされることを確認します。

フォルダー リダイレクトをテストする方法を次に示します。

1. サインイン プライマリ コンピューター (プライマリ コンピューターのサポートが有効な) 場合フォルダー リダイレクトが有効にしたユーザー アカウントを使用します。
2. ユーザーがコンピューターにサインインした、管理者特権のコマンド プロンプトを開いてし、クライアント コンピューターに最新のグループ ポリシー設定が適用されていることを確認するには、次のコマンドを入力します。
    
    ```PowerShell
    gpupdate /force
    ```
3. ファイル エクスプ ローラーを開きます。
4. (たとえば、マイ ドキュメント フォルダーのドキュメント ライブラリに)、リダイレクトされたフォルダーを右クリックし、**プロパティ**を選択します。
5. **位置情報**] タブを選択し、パスがローカル パスではなく、指定したファイル共有を表示することを確認します。
