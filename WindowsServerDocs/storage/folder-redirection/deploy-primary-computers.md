# フォルダー リダイレクト、および移動ユーザー プロファイルの主なコンピューターを展開します。

>適用対象: Windows 10、Windows 8、Windows 8.1、Windows Server 2012、Windows Server 2012 R2、Windows Server 2016

このトピックでは、プライマリのコンピューターのサポートを有効にして、ユーザーの主なコンピューターを指定する方法について説明します。 これを行うと、フォルダー リダイレクト、および移動ユーザー プロファイルのコンピューターを使用して制御できます。

>[!IMPORTANT]
>移動ユーザー プロファイルの主なコンピューターのサポートを有効にすると、常にプライマリ コンピューターのサポートが有効フォルダー リダイレクトもします。 これは、ドキュメントとユーザー プロファイルは、プロファイルのままに小さなと回サインオンすることが高速に留まる外の他のユーザー ファイルを保持します。

## 前提条件

## ソフトウェア要件

プライマリのコンピューターのサポートには、次の要件があります。

- Active Directory Domain Services (AD DS) スキーマを更新して Windows Server 2012 スキーマの追加を含める必要があります (Windows Server 2012 のドメイン コント ローラーに自動的に更新プログラムをインストール、スキーマ)。 AD DS スキーマの更新について詳しくは、 [Adprep.exe 統合](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh472161(v=ws.11)#adprepexe-integration>)し、[実行されている Adprep.exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd464018(v=ws.10)>)をご覧ください。
- クライアント コンピューターには、Windows 10、Windows 8.1、Windows 8、Windows Server 2016、Windows Server 2012 R2、または Windows Server 2012 を実行する必要があります。

>[!TIP]
>フォルダー リダイレクトを構成する Gpo を有効にする前に、の主なコンピューターのサポートを設定することをお勧めプライマリ コンピューターのサポートには、最初に、これらのテクノロジを展開している場合は、フォルダー リダイレクトや移動ユーザー プロファイルが必要ですが、およびユーザー プロファイルをローミングします。 これには、プライマリ コンピューターのサポートを有効にする前に、プライマリ以外のコンピューターにコピーしてからユーザー データことができます。 構成については、[フォルダー リダイレクトを展開](deploy-folder-redirection.md)し、[移動ユーザー プロファイルの展開](deploy-roaming-user-profiles.md)をご覧ください。

## 手順 1: ユーザーの主なコンピューターを指定できます。

プライマリのコンピューターのサポートを展開する最初の手順が各ユーザーの主なコンピューターを指定します。 これには、関連するコンピューターの識別名を取得し、 **[PrimaryComputer**属性を設定し、Active Directory 管理センターを使用します。

>[!TIP]
>Windows PowerShell を使用すると、プライマリ コンピューターと連携して、に関するブログの投稿[Windows 8 の主なコンピューターをより深くがもう少し詳しく見ると](<https://blogs.technet.microsoft.com/askds/2012/10/23/digging-a-little-deeper-into-windows-8-primary-computer/>)を参照してください。

ユーザーの主なコンピューターを指定する方法を以下に示します。

1. Active Directory 管理センターがインストールされているコンピューターで、サーバー マネージャーを開きます。
2. [**ツール**] メニューには、 **Active Directory 管理センター**を選択します。 Active Directory 管理センターが表示されます。
3. 適切なドメイン内**のコンピューター**のコンテナーに移動します。
4. プライマリ コンピューターとして指定し、[**プロパティ**] を選択するコンピューターを右クリックします。
5. ナビゲーション ウィンドウでは、**拡張機能**を選択します。
6. **属性エディター** ] タブを選択**識別名**にスクロール、**ビュー**、いる値を右クリックして、**コピー**] を選択、 **[ok]** を選択して、[**キャンセル**] を選択します。
7. 適切なドメインの**ユーザー**コンテナーに移動、コンピューターを割り当てるユーザーを右クリックし、[**プロパティ**。
8. ナビゲーション ウィンドウでは、**拡張機能**を選択します。
9. [**属性エディター** ] タブを選択して **[PrimaryComputer**して**編集**] を選択します。 [複数値の文字列エディター] ダイアログ ボックスが表示されます。
10. テキスト ボックスを右クリックして**貼り付け**を選択し、**追加**] を選択して **[ok]** を選択し、もう一度 **[ok]** を選択しています。

## 手順 2: 必要に応じてを有効にする主なコンピューターのグループ ポリシーでフォルダー リダイレクト

次の手順では、必要に応じてフォルダー リダイレクトの主なコンピューターのサポートを有効にするグループ ポリシーを構成します。 これにより、ユーザーのフォルダーをユーザーの主なコンピューターでは、として指定するコンピューターではなく、他のコンピューターにリダイレクトできます。 コンピューターごとに、またはユーザーごとに、フォルダー リダイレクトのプライマリ コンピューターを制御できます。

フォルダー リダイレクトの主なコンピューターを有効にする方法を以下に示します。

1. グループ ポリシーの管理] で、フォルダー リダイレクトや (たとえば、**フォルダー リダイレクト設定**または**ユーザー プロファイルの設定のローミング**) の移動ユーザー プロファイルの初期構成を行うときに作成した GPO を右クリックし、**編集**を選択します。
2. コンピューター ベースのグループ ポリシーを使用してプライマリ コンピューターのサポートを有効にするには、**コンピューターの構成**に移動します。 ユーザー ベースのグループ ポリシーでは、**ユーザーの構成**に移動します。
    - コンピューター ベースのグループ ポリシーでは、コンピューターのすべてのユーザーに影響を与える、GPO が適用されるすべてのコンピューターにプライマリ コンピューターの処理が適用されます。
    - ユーザーに基づくグループ ポリシーをプライマリ コンピューターの処理をユーザーがサインオンするすべてのコンピューターに影響を与える、GPO が適用されるすべてのユーザー アカウントに適用されます。
3. [**コンピューターの構成**] または [**ユーザーの構成****ポリシー**]、 **[管理用テンプレート**]、**システム**、**フォルダー リダイレクト**を移動します。
4. **プライマリ コンピューターのみでフォルダー リダイレクト**、右クリックし、**編集**します。
5. **Enabled**] を選択し、 **[ok]** を選択します。

## 手順 3: 必要に応じてを有効にするプライマリ コンピューターのグループ ポリシーで移動ユーザー プロファイル

次の手順では、必要に応じて移動ユーザー プロファイルの主なコンピューターのサポートを有効にするグループ ポリシーを構成します。 これにより、ユーザーの主なコンピューターでは、として指定するコンピューターではなく、他のコンピューターをローミングするユーザーのプロファイルです。

移動ユーザー プロファイルの主なコンピューターを有効にする方法を以下に示します。

1. まだ行っていない場合は、フォルダー リダイレクト、プライマリ コンピューターのサポートを有効にします。
    * これは、ドキュメントとユーザー プロファイルは、プロファイルのままに小さなと回サインオンすることが高速に留まる外の他のユーザー ファイルを保持します。
2. グループ ポリシーの管理] では、(たとえば、**フォルダー リダイレクト、および移動ユーザー プロファイルの設定**) を作成した GPO を右クリックし、し**編集**を選択します。
3. **コンピューターの構成**では、し、**ポリシー**の**管理用テンプレート**、し、**システム**、し**ユーザー プロファイル**に移動します。
4. **プライマリのコンピューターのみ、移動プロファイルのダウンロード**を右クリックし、**編集**します。
5. **Enabled**] を選択し、 **[ok]** を選択します。

## 手順 4: GPO の有効化にします。

フォルダー リダイレクトと移動ユーザー プロファイルの構成が終了したら、されていない場合は、GPO を有効にします。 これにより影響を受けるユーザーとコンピューターに適用することを許可します。

フォルダー リダイレクトやユーザー プロファイルの Gpo のローミングを有効にする方法を以下に示します。

1. グループ ポリシーの管理
2. 作成した Gpo を右クリックし、**リンクを有効になっています**。 メニュー項目の横にあるチェック ボックスが表示されます。

## 手順 5: テスト コンピューターのプライマリ関数

プライマリのコンピューターのサポートをテストするにプライマリ コンピューターにサインイン、フォルダーとプロファイルがリダイレクトされる、ことを確認し、プライマリ以外のコンピューターにサインインし、フォルダーとプロファイルにリダイレクトされていないことを確認します。

コンピューターの主な機能をテストする方法を以下に示します。

1. 指定したプライマリ コンピューターのフォルダー リダイレクトや移動ユーザー プロファイルを有効にしたユーザー アカウントにサインインします。
2. Windows PowerShell セッションまたは管理者としてコマンド プロンプト ウィンドウを開きますユーザー アカウントが、以前のコンピューターに署名されている場合、次のコマンドを入力およびし、サインオフする最新のグループ ポリシー設定が適用されることを確認するメッセージが表示されたら、クライアント コンピューター。
    ```PowerShell
    Gpupdate /force
    ```
3. ファイル エクスプ ローラーを開きます。
4. (たとえば、マイ ドキュメント ライブラリ内のフォルダー、ドキュメント)、リダイレクトされたフォルダーを右クリックし、**プロパティ**] を選択します。
5. **位置情報**] タブを選択し、パスがローカル パスではなく、指定したファイル共有を表示することを確認します。 ユーザー プロファイルをローミングすることを確認するには、**コントロール パネル**を開き、[**システムとセキュリティ**]、**システム**を選択して、**高度なシステム設定**] を選択、ユーザー プロファイルのセクションで**設定**を選択および**し確認ローミング** **Type** ] 列でします。
6. ユーザーの主なコンピューターとして指定されていないコンピューターに同じユーザー アカウントでサインインします。
7. 代わりに、**ローカル**のプロファイルの種類やローカル パスを探して手順 2 ~ 5 を繰り返します。

>[!NOTE]
>各フォルダーのフォルダー リダイレクト ポリシー設定で、次の設定が構成されている場合を除き、フォルダーは、プライマリのコンピューターのサポートを有効にする前に、コンピューターにリダイレクトされた場合、フォルダーのリダイレクトされたままなります: **、ローカルに、フォルダー リダイレクトユーザー プロファイルの場所、ポリシーが削除される**します。 特定のコンピューターでローミングされた以前のプロファイル**の種類**の列は**ローミング**を表示できる同様に、ただし、**状態**] 列は、**ローカル**で表示されます。
