/MUXes SLB と RAS ゲートウェイによってアドバタイズされた VIP 論理ネットワークのルートを受信する、SDN インフラストラクチャで使用されるルーターで BGP ピアを作成する必要があります。 BGP ピアリングは、(SLB/MUX または外部の BGP ピアに RAS ゲートウェイ) から 1 つの方法で発生するだけ済みます。  ルートの最初のレイヤーの上を使用できます静的ルートまたは OSPF などの別の動的ルーティング プロトコルただし、前述の VIP 論理ネットワークの IP サブネット プレフィックスは、外部の BGP ピアに、物理ネットワークからルーティング可能である必要があります。

## <a name="default-gateways"></a>既定のゲートウェイ
物理ホストとゲートウェイ仮想マシンなど、複数のネットワークに接続するために構成されているマシンが構成されている 1 つのデフォルト ゲートウェイのみ必要です。 インターネットに接続するために使用するアダプターのデフォルト ゲートウェイを構成します。

仮想マシンの場合、デフォルト ゲートウェイとして使用するネットワークを決定するこれらの規則に従います。

1. 仮想マシンが、トランジット ネットワークに接続する場合、またはマルチホーム トランジット ネットワークやその他のネットワークにある場合は、デフォルト ゲートウェイとしてトランジット論理ネットワークを使用します。
2. 仮想マシンは、管理ネットワークにしか接続できない場合は、デフォルト ゲートウェイとして、管理ネットワークを使用します。 
3. /MUXes SLB と RAS ゲートウェイ HNV プロバイダー ネットワークを使用します。 既定のゲートウェイとして HNV プロバイダー ネットワークを使わないでください。 
4. Storage1、Storage2、パブリック VIP または VIP をプライベート ネットワークに直接仮想マシンを接続できません。

HYPER-V ホストと記憶域ノードは、デフォルト ゲートウェイとして、管理ネットワークを使用します。  記憶域ネットワークは、割り当てられている既定のゲートウェイを決してが必要です。

## <a name="extending-your-infrastructure"></a>お客様のインフラストラクチャを拡張します。  
サイズ変更とリソース、インフラストラクチャ要件は次開催を予定してテナントのワークロードの仮想マシンに依存します。 CPU、メモリ、およびインフラストラクチャの仮想マシンのディスク要件 (例: ネットワーク コント ローラーで、SLB、ゲートウェイなど) は、前の表に記載します。 これらのインフラストラクチャ仮想マシンを必要に応じてスケール アウトの詳細を追加できます。 ただし、HYPER-V ホストで実行されているテナントの仮想マシンがあるは、独自の CPU、メモリ、およびディスクの要件を考慮する必要があります。   

テナント ワークロードの仮想マシンは、物理的な HYPER-V ホスト上のリソースが多すぎるの使用を開始、ときに、追加の物理ホストを追加することで、インフラストラクチャを拡張できます。 これ行う Virtual Machine Manager で、または (最初のデプロイ方法によって、インフラストラクチャ) の PowerShell スクリプトを使用してネットワーク コント ローラーから新しいサーバーのリソースを作成します。 HNV プロバイダー ネットワークに関する追加の IP アドレスを追加する必要がある場合は、(対応する IP プール) で、ホストが使用できる新しいの論理サブネットを作成できます。
