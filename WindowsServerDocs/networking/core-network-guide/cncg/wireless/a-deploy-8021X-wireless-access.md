2. **Advanced Encryption Standard** \(AES\)は商用と政府機関データの暗号化の推奨される暗号化プロトコルです。 AES では、tkip または WEP のいずれかのより高いレベルのワイヤレスの転送のセキュリティを提供します。 AES では TKIP、WEP とは異なり、AES 標準をサポートするワイヤレス ハードウェアが必要です。 AES は、対称\-キー暗号化標準的な 3 つのブロック暗号では、AES を使用する\-128、AES\-192、AES\-256 です。

Windows Server 2016 で、次の AES\-ベースのワイヤレス暗号化メソッドは、WPA2 の認証方式を選択するときにワイヤレス プロファイルのプロパティで構成できるように\-Enterprise で、お勧めします。

1. **AES\-CCMP**します。 カウンター モード暗号ブロック チェーン メッセージ認証コード プロトコル\(CCMP\) 802.11i 標準を実装して、WEP で提供されるより高いセキュリティの暗号化を目的し、128 ビット AES 暗号化キーが使用されます。

### <a name="network-policy-server"></a>ネットワーク ポリシー サーバー
ネットワーク ポリシー サーバー \(NPS\)一元的に構成し、リモート認証ダイヤルインを使用して、ネットワーク ポリシーを管理することができます\-User Service \(RADIUS\)サーバーと RADIUS プロキシ。 802.1 X ワイヤレス アクセスを展開するときに、NPS が必要です。

    プライベート CA を展開する場合、ドメイン メンバー クライアント コンピューターでグループ ポリシーが更新されたときに CA 証明書が、現在のユーザーとローカル コンピューターの信頼されたルート証明機関証明書ストアに自動的にインストールします。 パブリック CA からサーバー証明書を展開する場合は、パブリック CA 証明書が信頼されたルート証明機関の証明書ストアに既にことを確認します。

資格情報が有効な認証が成功した場合は、NPS は接続要求の処理の承認フェーズを開始します。 資格情報が有効でない、認証に失敗した場合は、アクセス拒否メッセージを NPS に送信し、接続要求が拒否されました。

1.
