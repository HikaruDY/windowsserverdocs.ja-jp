# <a name="wireless-access-deployment-planning"></a>ワイヤレス アクセスの展開の計画

>適用対象: Windows Server (半期チャネル)、Windows Server 2016

ワイヤレス アクセスを展開する前に、次の項目を計画する必要があります。

- ワイヤレス アクセスのインストール、ネットワーク上の \(APs\) をポイントします。

- ワイヤレス クライアントの構成とアクセス

次のセクションでは、これらの計画手順の詳細を説明します。

## <a name="planning-wireless-ap-installations"></a>ワイヤレス AP のインストールの計画
ワイヤレス ネットワーク アクセス ソリューションを設計するときに、次の操作を行う必要があります。

1. どのような標準、ワイヤレス Ap をサポートする必要がありますを決定します。
2. ワイヤレス サービスを提供するカバレッジ領域を判断します。
3. ワイヤレス Ap を検索する場所を決定します。

また、ワイヤレス AP の IP アドレス スキームを計画し、ワイヤレス クライアント必要があります。 セクションを参照して**ワイヤレス AP の nps の構成を計画**下の関連の情報。

さらに、高度なセキュリティを提供する、ネットワーク、ワイヤレス Ap は、次のセキュリティ オプションをサポートする必要があります。

## <a name="planning-wireless-client-configuration-and-access"></a>ワイヤレス クライアントの構成とアクセスを計画します。

802.1X\ の展開を計画するときに認証されたワイヤレス アクセスは、クライアントに固有のいくつかの要因を考慮する必要があります。

- **複数の標準のサポートを計画**します。

    かどうか、ワイヤレス コンピューターすべてを使用している同じバージョンの Windows または別のオペレーティング システムを実行しているコンピューターの組み合わせにあるかどうかを決定します。 これらが異なる場合は、オペレーティング システムでサポートされている標準の相違点を理解することを確認します。

    かどうかのすべてのワイヤレス クライアント コンピューター上のワイヤレス ネットワーク アダプターのサポートされているすべて同じワイヤレス標準では、さまざまな標準をサポートする必要があるかどうかを決定します。 たとえば、WPA\ エンタープライズおよび TKIP のみをサポートして他のユーザーに、一部のネットワーク アダプターのハードウェア ドライバーで WPA2\ エンタープライズと AES をサポートするかどうかを決定します。

- **計画のクライアント認証モード**します。 認証モードでは、Windows クライアントがドメインの資格情報を処理する方法を定義します。 ワイヤレス ネットワーク ポリシーでは、次の 3 つのネットワーク認証モードから選択できます。  

    1. **ユーザーの再認証**します。 このモードでは、認証では、コンピューターの現在の状態に基づいてセキュリティ資格情報を使用して常に実行を指定します。 ユーザーがログオンしていないコンピューターに、コンピューターの資格情報を使用して認証が実行されます。 コンピューターにユーザーがログオン時に、認証は常にユーザーの資格情報を使用して実行します。  

    2. **コンピューターのみ**します。 コンピューターは、モード認証を指定するだけでは常に実行コンピューターの資格情報のみを使用しています。  

    3.  **ユーザー認証**します。 ユーザー認証モードでは、コンピューターに、ユーザーがログインしている場合にのみ、その認証が実行を指定します。 コンピューターにログオンしたユーザーがない場合は、認証の試行は行われません。  

- **ワイヤレス制限計画**します。 すべてのユーザー ワイヤレス同じレベルの目的のワイヤレス ネットワークへのアクセスを提供するかどうかや、ワイヤレス ユーザーの一部のアクセスを制限するかどうかを決定します。 NPS で特定のワイヤレス ユーザー グループに対しての制限を適用できます。  たとえば、定義できます特定日、および時間特定のグループで、ワイヤレス ネットワークへのアクセスが許可されています。  

- **新しいワイヤレス コンピューターを追加するための方法を計画**します。 Wireless\ 対応参加しているコンピューターをドメインに、ワイヤレス ネットワークを展開する前に、コンピューターが 802.1 X で保護されていないワイヤード (有線) ネットワークのセグメントに接続されている場合、ワイヤレス構成設定が自動的に適用されてワイヤレス ネットワークを構成した後 \ (IEEE 802.11\) ポリシーおよびワイヤレス クライアントでグループ ポリシーが更新された後、ドメイン コントローラー。  

    ドメインに参加していないコンピューターの場合は、ただし、計画する必要がある 802.1X\ に必要な設定を適用する方法-認証済みアクセスします。 たとえば、次の方法のいずれかを使用して、コンピューターをドメインに参加するかどうかを決定します。

    1.  802.1 X で保護されていないワイヤード (有線) ネットワークのセグメントにコンピューターを接続し、コンピューターをドメインに参加します。

    2.  手順とワイヤレス独自プロファイルがコンピューターをドメインに参加させることができるブートス トラップを追加するに必要な設定をワイヤレス ユーザーに提供します。

    3.  ワイヤレス クライアントをドメインに参加する IT スタッフを割り当てます。
