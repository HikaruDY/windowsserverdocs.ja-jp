# <a name="network-policy-server-management-with-administration-tools"></a>管理ツールを使用してネットワーク ポリシー サーバーの管理

>適用対象: Windows Server (半期チャネル)、Windows Server 2016

このトピックを使用すると、NPS サーバーの管理に使用できるツールについて説明します。

NPS をインストールした後は、NPS サーバーを管理することができます。

- ローカルに NPS の NPS Microsoft 管理コンソール \(MMC\) スナップインで、[管理ツール、Windows PowerShell コマンド、またはネットワーク シェル \(Netsh\) コマンドで静的 NPS コンソールを使用します。
- リモート NPS サーバーから、NPS、またはリモート デスクトップ接続の NPS MMC スナップインで、NPS 用の Netsh コマンド、Windows PowerShell コマンドを使用しています。
- リモート ワークステーション、NPS MMC や Windows PowerShell などの他のツールとの組み合わせでリモート デスクトップ接続を使用しています。

>[!NOTE]
>Windows Server 2016 では、NPS コンソールを使用して、ローカルの NPS サーバーを管理できます。 ローカルとリモートの両方の NPS サーバーを管理するには、NPS MMC スナップインを使用する必要があります。

次のセクションでは、ローカルおよびリモートの NPS サーバーを管理する方法の手順を提供します。

## <a name="use-netsh-nps-commands-to-manage-an-nps-server"></a>NPS の Netsh コマンドを使用して、NPS サーバーを管理するには

Netsh NPS コンテキストでコマンドを使用するには表示され、アカウンティング、および監査の両方で、NPS とリモート アクセス サービスを使用するデータベース認証、承認の構成の設定をします。 Netsh NPS コンテキストのコマンドを使用します。
