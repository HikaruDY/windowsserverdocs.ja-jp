# <a name="network-policy-server-management-with-administration-tools"></a>管理ツールを使ったネットワーク ポリシー サーバーの管理

>適用対象:Windows Server 2016 の Windows Server (半期チャネル)

このトピックを使用すると、管理、NPSs に使用できるツールについて説明します。

NPS をインストールした後、NPSs を管理できます。

- NPS Microsoft 管理コンソールを使用してローカルで\(MMC\)スナップインで、管理ツール、Windows PowerShell コマンド、またはネットワーク シェルで静的な NPS コンソール\(Netsh\) NPS コマンド。
- リモートの NPS、NPS、またはリモート デスクトップ接続の NPS MMC スナップインで、NPS の Netsh コマンド、Windows PowerShell コマンドを使用しています。
- リモート デスクトップ接続を使用して NPS の mmc スナップインまたは Windows PowerShell などの他のツールと組み合わせてのリモート ワークステーション。

>[!NOTE]
>Windows Server 2016 では、NPS コンソールを使用して、ローカルの NPS を管理できます。 リモートとローカルのどちらの NPSs を管理するには、NPS MMC スナップインを使用する必要があります\-でします。

次のセクションでは、ローカルおよびリモート NPSs を管理する方法についてを説明します。

## <a name="configure-the-local-nps-by-using-the-nps-console"></a>NPS コンソールを使用してローカルの NPS を構成します。

NPS をインストールした後は、NPS MMC を使用して、ローカルの NPS を管理するのにこの手順を使用することができます。

**管理者の資格情報** 

この手順を完了するには、Administrators グループのメンバーがあります。

### <a name="to-configure-the-local-nps-by-using-the-nps-console"></a>NPS コンソールを使用して、ローカルの NPS を構成するには

1. サーバー マネージャーで、クリックして **ツール**, 、クリックして **ネットワーク ポリシー サーバー**します。 NPS コンソールが開きます。

2. NPS コンソールで、NPS をクリックします。\(ローカル\)します。 詳細ペインでいずれかを選択**標準構成**または**高度な構成**、選択した内容に応じて、次のいずれかの操作を行います。
    - 選択した場合**標準構成**をリストから、シナリオを選択し、構成ウィザードを開始する指示に従います。
    - 選択した場合**高度な構成**、矢印をクリックして展開**高度な構成オプション**、し、確認し、必要な NPS の機能に基づいて使用可能なオプションの構成RADIUS サーバー、RADIUS プロキシ、またはその両方です。

## <a name="manage-an-nps-by-using-remote-desktop-connection"></a>リモート デスクトップ接続を使用して、NPS を管理します。

この手順を使用すると、リモート デスクトップ接続を使用して、リモートの NPS を管理します。

リモート デスクトップ接続を使用すると、Windows Server 2016 を実行している、NPSs をリモートで管理できます。 NPSs は、Windows 10 または Windows クライアントの以前のオペレーティング システムを実行するコンピューターからリモートで管理できます。

リモート デスクトップ接続を使用して、2 つのメソッドのいずれかを使用して複数 NPSs を管理することができます。

1. 各、NPSs にリモート デスクトップ接続を個別に作成します。
2. リモート デスクトップを使用して、1 つの NPS に接続し、そのサーバーで NPS MMC を使用して、他のリモート サーバーを管理する. 詳細については、前のセクションを参照してください。 **NPS MMC スナップインを使用して管理の複数 NPSs\-で**します。
