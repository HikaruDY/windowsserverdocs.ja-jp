# <a name="verify-configuration-after-nps-changes"></a>NPS の変更後の構成を確認します。

>適用対象:Windows Server 2016 の Windows Server (半期チャネル)

このトピックを使用すると、サーバーに IP アドレスまたは名前の変更後に、NPS 構成を確認します。

## <a name="verify-configuration-after-an-nps-ip-address-change"></a>NPS の IP アドレスの変更後の構成を確認します。

ある可能性がありますの状況、NPS または異なる IP サブネットに、サーバーを移動するときなど、プロキシの IP アドレスを変更する必要があります。 

NPS サーバーまたはプロキシの IP アドレスを変更する場合、NPS の展開の一部を再構成する必要は。 

ネットワーク アクセスの認証、承認、または NPS RADIUS サーバーと RADIUS プロキシ サーバーのネットワーク上でアカウンティングに、IP アドレスの変更が中断しないことを検証するために、次の一般的なガイドラインを使用します。

メンバーである必要がある**管理者**、またはこれらの手順を実行するのと同じです。

### <a name="to-verify-configuration-after-an-nps-ip-address-change"></a>NPS の IP アドレスの変更後の構成を確認するには

1. ワイヤレス アクセス ポイントや、NPS の新しい IP アドレスでの VPN サーバーなどのすべての RADIUS クライアントを再構成します。

2. NPS が、リモート RADIUS サーバー グループのメンバーである場合は、NPS の新しい IP アドレスで NPS プロキシを再構成します。

3. SQL Server ログを使用する NPS を構成した場合は、SQL Server と、NPS を実行しているコンピューター間の接続が引き続き正常を確認します。

4. NPS と、NPS プロキシまたは他のサーバーやデバイス間の RADIUS トラフィックをセキュリティで保護するために IPsec を展開している場合は、IPsec ポリシーまたは NPS の新しい IP アドレスを使用するセキュリティが強化された Windows ファイアウォールの接続セキュリティ規則を再構成します。

5. NPS がマルチホームに特定のネットワーク アダプターにバインドするようにサーバーを構成する場合は、新しい IP アドレスで NPS ポート設定を再構成します。

### <a name="to-verify-configuration-after-an-nps-proxy-ip-address-change"></a>プロキシの IP アドレスが変更後、NPS の構成を確認するには

1. ワイヤレス アクセス ポイントや VPN サーバー、NPS プロキシの新しい IP アドレスを持つなど、すべての RADIUS クライアントを再構成します。

2. NPS プロキシはマルチホームに特定のネットワーク アダプターにバインドするプロキシを構成した場合は、新しい IP アドレスで NPS ポート設定を再構成します。

3. プロキシ サーバーの IP アドレスを持つすべてリモートの RADIUS サーバー グループのすべてのメンバーを再構成します。 各 NPS を RADIUS クライアントとして構成されている NPS プロキシを持つ、このタスクを実行するには。

    a.  ダブルクリック**NPS (ローカル)**、 をダブルクリックします**RADIUS クライアントとサーバー**、 をクリックして**RADIUS クライアント**、詳細ウィンドウで、RADIUS クライアントをダブルクリックしました。変更します。

    b.  RADIUS クライアントで**プロパティ**の**アドレス\(IP または DNS\)**、NPS プロキシの新しい IP アドレスを入力します。

4. SQL Server ログを使用する NPS プロキシを構成した場合は、SQL Server と、NPS プロキシを実行しているコンピューター間の接続が引き続き正常を確認します。

## <a name="verify-configuration-after-renaming-an-nps"></a>NPS の名前を変更した構成を確認します。

NPS またはなど、サーバーの名前付け規則を再設計するときに、プロキシの名前を変更する必要がある場合があります。

NPS サーバーまたはプロキシ名を変更する場合、NPS の展開の一部を再構成する必要は。 

ネットワーク アクセスの認証、承認、またはアカウンティング サーバー名の変更は中断されませんを検証するために、次の一般的なガイドラインを使用します。

メンバーである必要がある**管理者**、または同等で、この手順を実行します。

### <a name="to-verify-configuration-after-an-nps-or-proxy-name-change"></a>NPS サーバーまたはプロキシの名前変更後の構成を確認するには

1. NPS リモート RADIUS サーバー グループのメンバーである IP アドレスではなく、コンピューター名でグループが構成されている場合は、新しい NPS 名前で、リモート RADIUS サーバー グループを再構成します。

2. NPS では、証明書ベースの認証方法が展開されている場合、名前の変更には、サーバー証明書が無効にします。 証明機関 (CA) の管理者から、新しい証明書を要求するか、自動登録によって、新しい証明書を取得するグループ ポリシーを更新するには、コンピューターがドメイン メンバー コンピューターとするドメインのメンバーに自動登録証明書の場合は、. グループ ポリシーを更新するには。

    a.  コマンド プロンプトまたは Windows PowerShell を開きます。

    b.  「 **gpupdate**」と入力し、Enter キーを押します。


3. 新しいサーバー証明書がある場合は後、は、CA 管理者が、古い証明書を取り消すことを要求します。 

     古い証明書が失効すると、NPS は、古い証明書の有効期限が切れるまで、それを使用する続行されます。 既定では、古い証明書は 1 つ週 10 時間の最大時間有効です。 この期間は、かどうか、証明書失効リスト (CRL) の有効期限とトランスポート層セキュリティ (TLS) キャッシュの時間の有効期限から変更されている既定値によって異なる可能性があります。 既定の CRL の有効期限は 1 週間です。既定の TLS では、時間の有効期限は 10 時間をキャッシュします。 

     新しい証明書をすぐに使用するように NPS を構成する場合は、ただし、手動で再構成できます、新しい証明書により、ネットワーク ポリシー。

4. 古い証明書の有効期限が切れる後、NPS は、新しい証明書を使用して自動的に開始されます。 

5. SQL Server ログを使用する NPS を構成した場合は、SQL Server と、NPS を実行しているコンピューター間の接続が引き続き正常を確認します。
