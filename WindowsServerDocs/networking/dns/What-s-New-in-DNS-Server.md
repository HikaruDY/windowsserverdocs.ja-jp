# <a name="whats-new-in-dns-server-in-windows-server"></a>新機能 Windows server DNS サーバーの新機能

>適用対象:Windows Server 2016 の Windows Server (半期チャネル)

このトピックでは、Windows Server 2016 で追加または変更は、ドメイン ネーム システム (DNS) サーバーの機能について説明します。  
  
Windows Server 2016 では、DNS サーバーは、次の領域におけるサポートの強化を提供します。  
  
|機能|新機能か強化された機能か|説明|  
|-----------------|-------------------|---------------|  
|DNS のポリシー|新規|DNS サーバーが DNS クエリに応答する方法を指定する DNS ポリシーを構成することができます。 DNS 応答は、クライアントの IP アドレス (場所) に基づくことができますの時間、日、その他のいくつかのパラメーター。 DNS のポリシーには、位置認識 DNS、トラフィック管理、負荷分散、スプリット ブレイン DNS、およびその他のシナリオが有効にします。|  
|回答率の制限 (RRL)|新規|DNS サーバーで応答レート制限を有効にできます。 これによりには、DNS サーバーを使用して、サービス拒否攻撃では、DNS クライアントを開始する悪意のあるシステムの可能性を回避します。|  
|名前付きエンティティ (DANE) の DNS ベースの認証|新規|TLSA (トランスポート層セキュリティ認証) レコードを使用すると、その状態からのドメイン名の証明書をはず CA はどのような DNS クライアントに情報を提供します。 これは、場所、独自の web サイトを指す DNS キャッシュが破損する可能性があります、別の CA が発行された証明書を提供とだれかが、中間の攻撃を防ぎます。|  
|不明なレコードのサポート|新規|不明なレコードの機能を使用して、Windows DNS サーバーによって明示的にサポートされていないレコードを追加することができます。|  
|IPv6 ルート ヒント|新規|ルート ヒントをサポートして、IPV6 ルート サーバーを使用して、インターネット名前解決を実行するネイティブ IPV6 を使用することができます。|  
|Windows PowerShell のサポート|強化された機能|新しい Windows PowerShell コマンドレットは、DNS サーバー使用できます。|

## <a name="response-rate-limiting"></a>応答のレート制限

サーバーは、同じクライアントを対象とするいくつかの要求を受信すると、DNS クライアントに要求に応答する方法を制御する RRL 設定を構成することができます。 これにより、送信、DNS サーバーを使用して、サービス拒否 (Dos) 攻撃から防ぐだれかができます。 たとえば、net ボットでは、要求元として 3 つ目のコンピューターの IP アドレスを使用して DNS サーバーに要求を送信できます。 RRL、なし、DNS サーバーは、大量の 3 番目のコンピューター、すべての要求に対する応答可能性があります。 RRL を使用する場合は、次の設定を構成できます。  
  
-   **1 秒あたりの応答**します。 これは、同じ応答は、1 秒以内にクライアントに与えられる時間の最大数です。  
  
-   **1 秒あたりのエラー**します。 これは、エラー応答は、1 秒以内に、同じクライアントに送信されます最大回数です。  
  
-   **ウィンドウ**します。 これは、要求が多すぎますが行われた場合、対象のクライアントに応答が中断されます秒数です。  
  
-   **リーク レート**します。 これは、DNS サーバーが応答を中断時にクエリに応答がどの程度の頻度です。 たとえば場合は、サーバーが 10 秒間、クライアントへの応答を中断し、リークが発生率が 5、サーバーはすべて 5 に送信されるクエリの 1 つのクエリに応答も。 これにより、DNS サーバーが応答速度が、サブネットや FQDN で制限を適用する場合でも、応答を取得する正当なクライアントです。  
  
-   **TC レート**します。 これは、クライアントに応答が中断されている場合の tcp 接続を試みるクライアントに指示する使用されます。 たとえば、TC 率は 3 であり、サーバーが特定のクライアントへの応答を中断する場合、サーバーは受信したすべての 3 つのクエリ用の TCP 接続の要求を発行します。 TC レートの値がクライアントに応答をリークする前に、TCP 経由で接続するオプションを提供する、リーク レートよりも小さいことを確認します。  
  
-   **最大応答数**します。 これは、応答が中断されている間、サーバーはクライアントに発行する応答の最大数です。  
  
-   **ホワイト リスト ドメイン**します。 これは、RRL 設定から除外するドメインの一覧です。  
  
-   **ホワイト リスト サブネット**します。 これは、RRL 設定から除外するサブネットの一覧です。  
  
-   **ホワイト リストのサーバー インターフェイス**します。 これは、RRL 設定から除外する DNS サーバー インターフェイスの一覧です。  
  
## <a name="dane-support"></a>DANE サポート

DANE サポートを使用する\(RFC 6394 と 6698\) DNS サーバーでホストされている DNS クライアントにどのような CA をドメイン名から発行される証明書が期待される必要がありますを指定します。 これには、一種の中間の攻撃の人が、DNS キャッシュが破損して、独自の IP アドレスを DNS 名を指定することができないようにします。  
  
たとえば、CA1 という名前のよく知られている機関から証明書を使用して、www.contoso.com に SSL を使用するセキュリティで保護された web サイトをホストします。 Www.contoso.com を異なる、not-ように-よく知られた、証明書機関の名前がこの CA2 から証明書を取得することがだれかがある可能性があります。 次に、フェイク www.contoso.com の web サイトをホストしているエンティティは、偽のサイトを www.contoto.com をポイントするには、クライアントまたはサーバーの DNS キャッシュが破損することにあります。 エンド ユーザー CA2 から証明書が表示されます可能性があります単にそれを受け入れし、偽のサイトに接続します。 DANE で、クライアントは TLSA レコードを求める contoso.com の DNS サーバーに要求を行うし、www.contoso.com の証明書が CA1 別の問題について説明します。 別の CA から証明書、接続は中止されます。
